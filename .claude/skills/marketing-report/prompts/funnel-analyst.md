## Секция III: Анализ воронки

### III.0 Вводный абзац

1-2 строки: общая картина воронок за `{{PERIOD_LABEL}}`. Ключевое отклонение или тренд из `{{FINDINGS}}` и `{{DIAGNOSTICS}}`.

### III.1 WB — Органическая воронка

Источник данных: `{{TRAFFIC}}.wb_total` — текущий и предыдущий периоды.

Поля: `card_opens`, `add_to_cart`, `orders`, `buyouts`.

Расчёт конверсий:
- CR переход→корзина = `add_to_cart / card_opens * 100`
- CR корзина→заказ = `orders / add_to_cart * 100`
- CR заказ→выкуп = `buyouts / orders * 100`

Δ в п.п. = текущий CR − предыдущий CR.

**ASCII-диаграмма (ОБЯЗАТЕЛЬНО, внутри блока кода для корректного рендеринга в Notion):**

```
┌──────────────────────────────────────────────────┐
│           ОРГАНИЧЕСКАЯ ВОРОНКА WB                │
│                                                  │
│  {card_opens}  Переходы в карточку               │
│     ▼ ({cr_open_cart}%)                          │
│  {add_to_cart}  Добавления в корзину             │
│     [{prev_cr_open_cart}% → {delta_open_cart}пп] │
│     ▼ ({cr_cart_order}%)                         │
│  {orders}  Заказы                                │
│     [{prev_cr_cart_order}% → {delta_cart_order}пп]│
│     ▼ ({cr_order_buyout}%)  ⚠️ лаг 3-21 дн.      │
│  {buyouts}  Выкупы                               │
│     [{prev_cr_order_buyout}% → {delta_buyout}пп] │
└──────────────────────────────────────────────────┘
```

*Заполнить реальными числами из `{{TRAFFIC}}.wb_total`. Формат чисел: `1 234 567`.*

**Оценка по бенчмаркам (женское нижнее бельё):**

| Шаг воронки | Текущий CR | Предыдущий CR | Δ п.п. | Норма | Оценка |
|---|---|---|---|---|---|
| Переход → корзина | % | % | п.п. | 5–15% | ✅/⚠️/🔴 |
| Корзина → заказ | % | % | п.п. | 20–40% | ✅/⚠️/🔴 |
| Заказ → выкуп | % | % | п.п. | 40–65% | ✅/⚠️/🔴 |

Шкала оценки:
- ✅ — в пределах нормы
- ⚠️ — ниже нормы
- 🔴 — критично: CR < 50% от нижней границы нормы

Специальные правила:
- Если выкуп упал > 5 п.п.: `> 🔴 Критическое падение конверсии выкупа — требует срочного анализа причин возвратов`
- При DEPTH=day: `> ⚠️ Выкуп — лаговый показатель (задержка 3-21 дней). Интерпретировать с осторожностью`
- При DEPTH=week: добавить оговорку "данные по выкупам могут быть неполными"
- При DEPTH=month: выкуп интерпретировать без оговорок (достоверный показатель)

**Вывод по органической воронке WB (2-3 строки):** какой шаг является узким местом, связь с `{{DIAGNOSTICS}}`, рекомендация.

### III.2 WB — Рекламная воронка

Источник данных: `{{ADVERTISING}}.wb_ad_daily_series` — суммы по периоду.

Поля: `views` (показы), `clicks` (клики), `to_cart` / `atbs` (корзина), `orders` (заказы).

Расчёт конверсий:
- CTR = `clicks / views * 100`
- CR клик→корзина = `to_cart / clicks * 100`
- CR корзина→заказ = `orders / to_cart * 100`
- Полная CR = `orders / clicks * 100`

**ASCII-диаграмма (ОБЯЗАТЕЛЬНО):**

```
┌──────────────────────────────────────────────────┐
│             РЕКЛАМНАЯ ВОРОНКА WB                 │
│                                                  │
│  {views}  Показы                                 │
│     ▼ CTR {ctr}%                                 │
│  {clicks}  Клики                                 │
│     [{prev_ctr}% → {delta_ctr}пп]                │
│     ▼ ({cr_click_cart}%)                         │
│  {to_cart}  Добавления в корзину                 │
│     [{prev_cr_click_cart}% → {delta_click_cart}пп]│
│     ▼ ({cr_cart_order}%)                         │
│  {orders}  Заказы                                │
│     [{prev_cr_cart_order}% → {delta_cart_order}пп]│
└──────────────────────────────────────────────────┘
Полная конверсия: {full_cr}%  [{prev_full_cr}% → {delta_full_cr}пп]
```

**Оценка по бенчмаркам:**

| Шаг воронки | Текущий CR | Предыдущий CR | Δ п.п. | Норма | Оценка |
|---|---|---|---|---|---|
| CTR (показы → клики) | % | % | п.п. | 3–7% | ✅/⚠️/🔴 |
| Клик → корзина | % | % | п.п. | 15–25% | ✅/⚠️/🔴 |
| Корзина → заказ | % | % | п.п. | 25–40% | ✅/⚠️/🔴 |
| Полная CR (клик → заказ) | % | % | п.п. | 2–5% | ✅/⚠️/🔴 |

> ⚠️ Если CTR < 1,5% или полная CR < 1% — рекламные материалы требуют пересмотра. Указать конкретные кампании из `{{DIAGNOSTICS}}`.

**Вывод по рекламной воронке WB (2-3 строки):** ключевой провал CR, качество трафика, рекомендация (ставки / креативы / таргетинг).

### IV.4 Ключевые инсайты (3-5 строк)

Обязательно ответить на вопросы:

1. **Тренд органики:** растёт или снижается доля органических переходов? Назвать направление и причину из `{{DIAGNOSTICS}}` (SEO, позиции, контент карточки, отзывы).

2. **Влияние на ДРР:** если органика растёт → снижение зависимости от рекламы → потенциал снижения ДРР. Оценить в п.п. ДРР: `если органика +X% переходов → экономия ≈ Y ₽ на рекламе`.

3. **Связь с внешней рекламой:** если в предыдущем периоде были отключены/снижены внешние кампании (из `{{DIAGNOSTICS}}`) — оценить, как это повлияло на органический трафик (лаг блогеров 3-7 дней).

> 💡 Если доля органики выросла > 5 п.п. — отметить как позитивный тренд: бренд снижает зависимость от платного трафика.

> ⚠️ Если доля органики упала > 5 п.п. — отметить риск: рост ДРР без роста органической базы. Рекомендовать SEO-оптимизацию карточек.

## Критические правила (10)

1. **ASCII-диаграммы ОБЯЗАТЕЛЬНЫ** — это ключевая визуальная особенность маркетингового отчёта. Пропуск недопустим
2. **ASCII внутри блоков кода** (``` ```) — обязательно для корректного рендеринга в Notion
3. **Каждый CR показывать с тремя значениями:** текущий + предыдущий + Δ в п.п. Никогда не показывать CR без сравнения
4. **Бенчмарки после каждой воронки** с оценкой (✅ в норме / ⚠️ ниже нормы / 🔴 критично)
5. **Выкуп — лаговый показатель** (3-21 день). При DEPTH=day всегда помечать: "⚠️ лаговый показатель"
6. **ТОЛЬКО чистый Markdown** — pipe tables, `## ▶`, `### ▶`. Без HTML тегов (`<table>`, `<div>`, `<callout>`)
7. **ТОЛЬКО реальные числа** из входных данных. Никогда не генерировать и не интерполировать
8. **Формат чисел:** `1 234 567`, `24,1%`, `+3,2 п.п.` (запятая в дробях, пробел в тысячах, знак у Δ)
9. **Callouts:** `> ⚠️ текст` для аномалий CR, `> 💡 текст` для положительных инсайтов
10. **OZON органика недоступна** — всегда явно указывать: "Органические данные OZON недоступны через API"

## Дополнительные правила по DEPTH

### DEPTH=day
- Выкуп: НЕ анализировать как тренд, только показать факт с оговоркой о лаге
- Органика vs платное: акцент на сравнении с предыдущим днём, не с прошлой неделей
- Дневные воронки: одно сравнение день/день, без понедельной агрегации

### DEPTH=week
- Выкуп: анализировать с оговоркой "данные могут быть неполными"
- Органика vs платное: оценить недельный тренд, связать с активностью блогеров (лаг 3-7 дней)

### DEPTH=month
- Выкуп: достоверный показатель, интерпретировать без оговорок
- Органика vs платное: добавить понедельную динамику долей в таблицу 4б

## Тон и стиль

Сухой аналитический. Без эмоций, без восклицательных знаков. Факты → цифры → вывод → рекомендация.
Каждая рекомендация содержит направление действия и оценку эффекта в рублях если возможно.
Комментарии в таблицах: 3-7 слов максимум. Развёрнутый анализ — только в текстовых блоках.
