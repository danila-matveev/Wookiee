**Перед анализом** прочитай Knowledge Base: `.claude/skills/analytics-report/references/analytics-kb.md`.

### III.1 Внешний трафик (блогеры + ВК)

Источник данных:
- `{{ADVERTISING}}.wb_external_breakdown` — adv_bloggers + adv_vk (НЕ adv_creators!)
- `{{SHEETS_BLOGGERS}}` — клики, заказы из Sheets (если доступны)
- `{{SHEETS_VK}}` — переходы, заказы из Sheets (если доступны)

Расчёт:
- Бюджет внешний = adv_bloggers + adv_vk
- Переходы = vk_clicks + blogger_clicks (из Sheets или расчётные)
- Заказы атрибутированные = заказы из Sheets блогеров + заказы из Sheets ВК
- CPO = бюджет / заказы_атрибутированные
- ДРР продаж = бюджет / выручка_атрибутированная * 100
- ДРР заказов = бюджет / сумма_заказов * 100

**ASCII-диаграмма (ОБЯЗАТЕЛЬНО, внутри блока кода):**

```
┌──────────────────────────────────────────────────────┐
│             ВОРОНКА ВНЕШНЕГО ТРАФИКА                  │
│                                                      │
│  Бюджет: {spend_external} ₽                          │
│     ▼                                                │
│  {vk_clicks + blogger_clicks}  Переходы              │
│     ▼ ({cr}%)                                        │
│  {orders_attributed}  Заказы (атрибутированные)       │
│     CPO: {cpo} ₽                                     │
│  ДРР продаж: {drr_sales}%                           │
│  ДРР заказов: {drr_orders}%                          │
└──────────────────────────────────────────────────────┘
```

*Заполнить реальными числами. Формат: `1 234 567`.*

**Оценка по бенчмаркам (внешний трафик, женское бельё):**

| Метрика | Текущий | Предыдущий | Δ | Норма | Оценка |
|---|---|---|---|---|---|
| CPO ₽ | ₽ | ₽ | ₽ | 300–800 ₽ | ✅/⚠️/🔴 |
| ДРР продаж % | % | % | п.п. | 5–15% | ✅/⚠️/🔴 |
| ДРР заказов % | % | % | п.п. | 3–10% | ✅/⚠️/🔴 |
| CR переход→заказ % | % | % | п.п. | 1–5% | ✅/⚠️/🔴 |

> ⚠️ Если данных по внешнему трафику нет за период — указать "Нет размещений за период" и показать только бюджет из wb_external_breakdown.

**Вывод по внешнему трафику (2-3 строки):** эффективность внешних каналов, CPO vs внутренняя реклама, рекомендация.

### III.2 Внутренняя реклама МП (WB + OZON)

#### WB

Источник данных: `{{ADVERTISING}}.wb_ad_daily_series` — суммы по периоду, `{{TRAFFIC}}.wb_total` (рекламные метрики).

Поля: `views` (показы), `clicks` (клики), `to_cart` / `atbs` (корзина), `orders` (заказы).

Расчёт конверсий:
- CTR = `clicks / views * 100`
- CR клик→корзина = `to_cart / clicks * 100`
- CR корзина→заказ = `orders / to_cart * 100`
- Полная CR = `orders / clicks * 100`

**ASCII-диаграмма (ОБЯЗАТЕЛЬНО):**

```
┌──────────────────────────────────────────────────────┐
│          ВОРОНКА ВНУТРЕННЕЙ РЕКЛАМЫ WB               │
│                                                      │
│  {impressions}  Показы                               │
│     ▼ CTR {ctr}%                                     │
│  {clicks}  Клики                 [{prev}% → {Δ}пп]   │
│     ▼ ({cr_cart}%)                                   │
│  {cart}  Корзина                 [{prev}% → {Δ}пп]   │
│     ▼ ({cr_order}%)                                  │
│  {orders}  Заказы                [{prev}% → {Δ}пп]   │
└──────────────────────────────────────────────────────┘
Полная конверсия: {full_cr}%  [{prev}% → {Δ}пп]
CPO: {cpo} ₽ | CPC: {cpc} ₽
```

**Оценка по бенчмаркам (внутренняя реклама WB):**

| Шаг воронки | Текущий CR | Предыдущий CR | Δ п.п. | Норма | Оценка |
|---|---|---|---|---|---|
| CTR (показы → клики) | % | % | п.п. | 3–7% | ✅/⚠️/🔴 |
| Клик → корзина | % | % | п.п. | 15–25% | ✅/⚠️/🔴 |
| Корзина → заказ | % | % | п.п. | 25–40% | ✅/⚠️/🔴 |
| Полная CR (клик → заказ) | % | % | п.п. | 2–5% | ✅/⚠️/🔴 |

> ⚠️ Если CTR < 1,5% или полная CR < 1% — рекламные материалы требуют пересмотра. Указать конкретные кампании из `{{DIAGNOSTICS}}`.

**Вывод по рекламной воронке WB (2-3 строки):** ключевой провал CR, качество трафика, рекомендация.

### III.3 Органический трафик

**ВАЖНО:** Органика = Total − Внутренняя реклама − Внешняя реклама. Это РАСЧЁТНЫЙ показатель.

Источник данных:
- `{{TRAFFIC}}.wb_total` — card_opens (total organic opens из content_analysis), add_to_cart (organic)
- `{{ADVERTISING}}.wb_ad_daily_series` — рекламные заказы (для вычитания)
- `{{ADVERTISING}}.wb_external_breakdown` — внешние заказы (для вычитания)

Расчёт:
- Organic opens = card_opens (total organic opens from content_analysis)
- Organic cart = add_to_cart (organic)
- Organic orders = total_orders − ad_orders − external_attributed_orders
- CR переход→корзина = `organic_cart / organic_opens * 100`
- CR корзина→заказ = `organic_orders / organic_cart * 100`
- CR заказ→выкуп = `buyouts / organic_orders * 100`

**ASCII-диаграмма (ОБЯЗАТЕЛЬНО):**

```
┌──────────────────────────────────────────────────────┐
│          ОРГАНИЧЕСКАЯ ВОРОНКА WB                     │
│                                                      │
│  {organic_opens}  Переходы (органика)                │
│     ▼ ({cr_cart}%)                                   │
│  {organic_cart}  Корзина          [{prev}% → {Δ}пп]  │
│     ▼ ({cr_order}%)                                  │
│  {organic_orders}  Заказы         [{prev}% → {Δ}пп]  │
│     ▼ ({cr_buyout}%) ⚠️ лаг 3-21 дн.                │
│  {buyouts}  Выкупы               [{prev}% → {Δ}пп]  │
└──────────────────────────────────────────────────────┘
```

*Органика OZON недоступна через API. Показана только WB.*

**Оценка по бенчмаркам (органический трафик WB):**

| Шаг воронки | Текущий CR | Предыдущий CR | Δ п.п. | Норма | Оценка |
|---|---|---|---|---|---|
| Переход → корзина | % | % | п.п. | 5–15% | ✅/⚠️/🔴 |
| Корзина → заказ | % | % | п.п. | 20–40% | ✅/⚠️/🔴 |
| Заказ → выкуп | % | % | п.п. | 40–65% | ✅/⚠️/🔴 |

Шкала оценки:
- ✅ — в пределах нормы
- ⚠️ — ниже нормы
- 🔴 — критично: CR < 50% от нижней границы нормы

Специальные правила:
- Если выкуп упал > 5 п.п.: `> 🔴 Критическое падение конверсии выкупа — требует срочного анализа причин возвратов`
- При DEPTH=day: `> ⚠️ Выкуп — лаговый показатель (задержка 3-21 дней). Интерпретировать с осторожностью`
- При DEPTH=week: добавить оговорку "данные по выкупам могут быть неполными"
- При DEPTH=month: выкуп интерпретировать без оговорок (достоверный показатель)

**Вывод по органической воронке WB (2-3 строки):** какой шаг является узким местом, связь с `{{DIAGNOSTICS}}`, рекомендация.

## Критические правила (12)

1. **3 воронки по ИСТОЧНИКУ:** внешний (блогеры+ВК), внутренний МП (WB+OZON), органика (=total−internal−external). НЕ "органика vs реклама" как в v1
2. **ASCII-диаграммы ОБЯЗАТЕЛЬНЫ** — ключевая визуальная особенность. Пропуск недопустим
3. **ASCII внутри блоков кода** (``` ```) — обязательно для корректного рендеринга в Notion
4. **Каждый CR показывать с тремя значениями:** текущий + предыдущий + Δ в п.п.
5. **Бенчмарки после каждой воронки** с оценкой (✅/⚠️/🔴)
6. **Выкуп — лаговый показатель** (3-21 день). При DEPTH=day всегда помечать
7. **Sheets данные — ПАРСИТЬ:** фильтровать блогеров по "Дата публикации", ВК/SMM по датам. Никогда не писать "данные отсутствуют" если данные реально есть
8. **Внешняя реклама = блогеры + ВК.** adv_creators = 0, не используется. Значение 183К — это adv_vk (ВК), НЕ adv_creators
9. **Dual KPI:** ДРР продаж + ДРР заказов на каждый канал внешней рекламы
10. **ТОЛЬКО чистый Markdown** — pipe tables, `## ▶`, `### ▶`. Без HTML тегов
11. **ТОЛЬКО реальные числа** из входных данных. Никогда не генерировать
12. **Формат чисел:** `1 234 567`, `24,1%`, `+3,2 п.п.` (запятая в дробях, пробел в тысячах, знак у Δ)

## Дополнительные правила по DEPTH

### DEPTH=day
- Выкуп: НЕ анализировать как тренд, только показать факт с оговоркой о лаге
- Внешняя реклама: показать размещения за конкретный день (может быть пусто)
- Дневные воронки: одно сравнение день/день

### DEPTH=week
- Выкуп: анализировать с оговоркой "данные могут быть неполными"
- Внешняя реклама: агрегировать по неделе, показать каждое размещение
- Связь внешнего трафика и органики: оценить недельный лаг блогеров (3-7 дней)

### DEPTH=month
- Выкуп: достоверный показатель, интерпретировать без оговорок
- Внешняя реклама: полная помесячная агрегация, ROI по каждому каналу
- SMM: месячная динамика обязательна

## Тон и стиль

Сухой аналитический. Без эмоций, без восклицательных знаков. Факты → цифры → вывод → рекомендация.
Каждая рекомендация содержит направление действия и оценку эффекта в рублях если возможно.
Комментарии в таблицах: 3-7 слов максимум. Развёрнутый анализ — только в текстовых блоках.
Callouts: `> ⚠️ текст` для аномалий, `> 💡 текст` для инсайтов.
