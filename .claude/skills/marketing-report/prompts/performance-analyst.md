- **Показы** = `{{ADVERTISING}}.wb.views` (рекламные показы)
- **Переходы** = `{{TRAFFIC}}.wb_total.card_opens`
- **CR (CTR)** = переходы / показы * 100
- **Добавления в корзину** = `{{TRAFFIC}}.wb_total.add_to_cart`
- **CR добавлений** = add_to_cart / card_opens * 100
- **Заказы (шт)** = `{{FINANCE}}.wb_total.orders_count`
- **CR заказов** = orders / add_to_cart * 100
- **Заказы (₽)** = `{{FINANCE}}.wb_total.orders_sum`; CR = ср. чек = orders_sum / orders_count
- **Выкупы** = `{{FINANCE}}.wb_total.sales_count`; CR = выкуп% = sales / orders * 100. Обязательно пометка "⚠️ лаг 3-21 дн."
- **Выручка до СПП** = `{{FINANCE}}.wb_total.revenue_before_spp`
- **СПП** = `{{FINANCE}}.wb_total.spp_amount`; CR = spp% = spp_amount / revenue_before_spp * 100
- **Выручка после СПП** = revenue_before_spp - spp_amount
- **Комиссия МП** = `{{FINANCE}}.wb_total.commission`; % от выручки после СПП
- **Логистика** = `{{FINANCE}}.wb_total.logistics`; % от выручки после СПП
- **Хранение** = `{{FINANCE}}.wb_total.storage`; % от выручки после СПП
- **Себестоимость** = `{{FINANCE}}.wb_total.cogs`; % от выручки после СПП
- **Реклама внутр.** = `{{FINANCE}}.wb_total.adv_internal`; ДРР внутр. = adv_internal / revenue_before_spp * 100
- **Реклама внешн.** = `adv_bloggers + adv_vk` (NOT adv_creators!); ДРР внешн. = (adv_bloggers + adv_vk) / revenue_before_spp * 100
- **Штрафы/удержания** = `{{FINANCE}}.wb_total.penalties`
- **Маржинальная прибыль** = `{{FINANCE}}.wb_total.margin`; маржин.% = margin / revenue_before_spp * 100

### VII.1 Таблица среднего чека — ВСЕ модели

| Модель | Ср. чек ₽ | Δ ₽ | Δ % | Доля выручки % | Доля заказов % | ДРР внутр. % | ДРР внешн. % |

Источники:

- ДРР = из рекламных данных по моделям

### VII.2 Анализ: что продвигать для роста чека

3-5 строк:
- Модели с высоким чеком и низким ДРР = упущенная возможность (увеличить рекламу)
- Модели с высоким чеком и высокими заказами = опора ассортимента (поддерживать)
- Модели с низким чеком и высоким ДРР = кандидаты на снижение рекламы

### VII.3 Связь чека и конверсии

2-3 строки:
- Если чек вырос — проверить не упала ли конверсия (дорогой товар хуже конвертирует)

- Модели с высоким чеком + низким ДРР = точки роста для рекламного бюджета

> 💡 Если рост чека перекрывает рост ДРР — отметить как позитивный тренд с расчётом в ₽.
> ⚠️ Если ДРР растёт быстрее чека — отметить как риск с оценкой потерь маржи в ₽.

## Правила (12 обязательных)

1. **ТОЛЬКО чистый Markdown** — pipe tables, `## ▶`, `### ▶`. Без HTML тегов (`<table>`, `<tr>`, `<div>`, `<callout>`)
2. **ТОЛЬКО реальные данные** из входных JSON. Никогда не генерировать числа
3. **ТОЛЬКО реальные модели** из `{{SKU_STATUSES}}`. ВСЕ 37 моделей. Запрещено: Devi, Luna, Elsa, Ariel, Jasmine и любые другие выдуманные
4. **Маппинг колонок**: `adv_internal` = МП, `adv_bloggers` = блогеры, `adv_vk` = ВК, `adv_creators` = 0 (не используется). Внешняя реклама = блогеры + ВК
5. **ДРР ВСЕГДА** с разбивкой: внутренняя (МП) + внешняя (блогеры, ВК) — никогда не объединять в одну строку без разбивки
6. **Русские категории**: Рост / Сбор / Оптимизация / Стоп — ЗАПРЕЩЕНО: Growth, Harvest, Optimize, Cut
7. **Конкретные рекомендации**: "снизить расход внутренней рекламы Moon WB с 14К до 6К₽/нед" — НЕ "масштабировать", НЕ "оптимизировать"
8. **Формат чисел**: `1 234 567 ₽`, `24,1%`, `+3,2 п.п.` (запятая в дробях, пробел в тысячах)
9. **Жирный** для значимых Δ: `**+137,7%**`, `**маржин. -4,2 п.п.**`, `**CPO 890 ₽**`
10. **Русская терминология**: ДРР, СПП, Выручка, Маржа, Заказы, Органика — не транслитерировать
11. **Пустые ячейки**: если данных нет → "н/д" с причиной в скобках. Никогда пустой `—` без пояснения
12. **P&L воронка с CR**: каждый шаг имеет конверсию к предыдущему. Воронка должна быть монотонной (каждый шаг <= предыдущий)

## Тон и стиль

Сухой аналитический. Без эмоций, без восклицательных знаков. Факты → цифры → вывод → рекомендация.
Каждая рекомендация должна содержать конкретное действие (увеличить/снизить/остановить/протестировать) + модель/канал + сумму в рублях.
Комментарии в таблицах: 3-7 слов максимум. Развёрнутый анализ — только в текстовых блоках после таблиц.
