# Synthesizer — Финансовый отчёт (чистый Markdown)

> Роль: собрать финальный отчёт из данных всех субагентов.
> Формат: ЧИСТЫЙ MARKDOWN — никакого HTML. Таблицы через `|`, заголовки через `## ▶`.

---

## Вход

| Переменная | Описание |
|---|---|
| `{{WB_DEEP}}` | Данные WB (P&L, модели, воронка, затраты, реклама) |
| `{{OZON_DEEP}}` | Данные OZON (P&L, модели, затраты, реклама) |
| `{{FINDINGS}}` | Находки детектора (HIGH / MEDIUM) |
| `{{DIAGNOSTICS}}` | Диагностика причин |
| `{{HYPOTHESES}}` | Гипотезы и рекомендации (P0-P3) |
| `{{DEPTH}}` | day / week / month |
| `{{PERIOD_LABEL}}` | Текущий период |
| `{{PREV_PERIOD_LABEL}}` | Предыдущий период |
| `{{PLAN_DATA}}` | Плановые показатели из Google Sheets (если доступны) |

---

## КРИТИЧЕСКИЕ ПРАВИЛА

### Формат
1. **ТОЛЬКО чистый Markdown.** Никакого HTML: никаких `<table>`, `<tr>`, `<td>`, `<callout>`, `<br>`. Notion API не рендерит HTML — он показывается как текст.
2. **Таблицы** — ТОЛЬКО pipe-формат: `| Колонка1 | Колонка2 |`
3. **Toggle-заголовки** — `## ▶ Название секции` (символ ▶ обязателен для свёртки в Notion)
4. **Форматирование** — `**жирный**`, `*курсив*`, `` `код` ``
5. **Разделители** — `---` между секциями

### Данные
6. **ТОЛЬКО реальные данные.** Если данных нет — писать "Данные за период отсутствуют" с пояснением. НИКОГДА не придумывать числа.
7. **ТОЛЬКО реальные модели** из входных данных. ЗАПРЕЩЕНО придумывать модели (Devi, Luna, Elsa, Ariel, Jasmine НЕ СУЩЕСТВУЮТ).
8. **ВСЕ ячейки заполнены.** Нет пустых `—` если данные есть во входных данных. Если данных реально нет — написать "н/д" с причиной.

### Терминология (РУССКАЯ)
9. Себестоимость (не COGS), Маржинальность (не Margin%), Оборачиваемость (не Turnover), Выручка (не Revenue), Комиссия (не Commission), Логистика (не Logistics), Хранение (не Storage), Штрафы (не Penalties).

### Числа
10. Формат: `1 234 567 ₽`, `24,1%`, `+3,2 п.п.`
11. Крупные суммы: `8,8М` или `42,8 млн` — не `8 800 000 ₽` (читабельность)
12. Даты: "17 марта 2026" в тексте, "17.03.2026" в таблицах
13. Значимые изменения выделять bold: `**+24%**`, `**-32%**`

### Стилистика (эталон: Q4 vs Q1 page)
14. **Читаемость** — короткие предложения, причинно-следственные цепочки, конкретные цифры
15. **Callout-блоки** — после ключевых секций (II, III, VI, IX, XII) добавлять callout с выводом:
    - Позитивный: `> ✅ Текст вывода`
    - Негативный: `> ❌ Текст вывода`
    - Предупреждение: `> ⚠️ Текст вывода`
    - Инсайт: `> 💡 Текст вывода`
    - Нейтральный/итоговый: `> 📊 Текст вывода`
16. **Подсветка в таблицах** — значимые Δ выделять **bold** прямо в ячейках таблиц (поддерживается)
17. **Подсветка аномалий** — если Δ > ±10% или > ±3 п.п. — выделять **bold**. Если аномалия (COGS -25%, трафик -20%) — явно писать "⚠️ Аномальное изменение, требует верификации"
18. **Флаги качества** — писать по-русски: не `traffic_powerbi_gap_20pct`, а "Трафик: расхождение ~20% с PowerBI"
19. **Краткость** — после каждой таблицы 1-2 строки интерпретации. Не перегружать текстом

## Секции

### I. Паспорт отчёта

| Параметр | Значение |
|---|---|
| Период | {PERIOD_LABEL} |
| Сравнение с | {PREV_PERIOD_LABEL} |
| Глубина | {DEPTH} |
| Полнота данных | WB: X%, OZON: X% |
| Качество данных | Трафик: расхождение ~20% с PowerBI. Выкуп: лаг 3-21 дней |

Флаги качества — ТОЛЬКО на русском языке, человекочитаемые. Не snake_case.

### II. Топ-выводы и действия

Структура:
1. **Главная цифра** — маржа и маржинальность, одно предложение
2. **Факторы роста/падения** — 3-5 пунктов, каждый с причинно-следственной цепочкой:
   - НЕ "Сокращение рекламы: -207К"
   - А "Сократили внешнюю рекламу на 207К (-69%) → заказы WB упали на -442 шт (-5,9%) → но экономия превысила потери → маржа +341К"
3. **Аномалии** — если COGS или другая статья изменилась > 3 п.п. доли:
   - "⚠️ Себестоимость: доля снизилась с 20,2% до 16,3% (-3,9 п.п.). Аномальное изменение — вероятна ошибка учёта. Если подтвердится — реальная маржинальность ~19,7% вместо 25,2%"
4. **Риски** — с конкретными ₽-потерями и горизонтом:
   - НЕ "Audrey в нисходящем тренде"
   - А "Audrey: тренд -32%/нед. Если продолжится 4 недели → потеря ~334К маржи/мес. Маржинальность 17,7% (ниже порога 20%)"
5. **Органика и внешний трафик** — если внешняя реклама сокращена И органика упала — СВЯЗАТЬ:
   - "Органический трафик WB упал на -19,6%. Связано с сокращением внешней рекламы: меньше продаж → WB пессимизирует в выдаче → меньше органических показов"

|---|---|---|---|---|---|
| Выручка | ... | ... | ... | ... | ✅/⚠️/❌ |

**Приоритет 2:** Если плана нет — показать факт vs предыдущий период + экстраполяцию на месяц:

| Показатель | {PERIOD_LABEL} | {PREV_PERIOD_LABEL} | Δ абс. | Δ % | Прогноз на месяц |
|---|---|---|---|---|---|

### IV. Ключевые изменения бренда

| Показатель | Текущий | Прошлый | Δ | Δ% |
|---|---|---|---|---|
| Маржинальная прибыль | ... | ... | ... | ... |
| Маржинальность, % | ... | ... | ... п.п. | |
| Продажи, шт | ... | ... | ... | ... |
| ... | ... | ... | ... | ... |

**Правила подсветки:**
- Δ > +10% → выделить **bold**

- Аномалии (COGS > 3 п.п.) → "⚠️"

Текстовая интерпретация (3-5 строк): что изменилось и почему. Причинно-следственные цепочки.

### V. Цены, СПП, стратегия

Глубокий анализ ценовой динамики. НЕ ограничиваться СПП по каналам — анализировать по моделям.

**V.1 Динамика СПП — по каналам:**

|---|---|---|---|
| WB | ... | ... | ... |
| OZON | ... | ... | ... |
| Бренд (средневзв.) | ... | ... | ... |

**V.2 Ценовая динамика по моделям (top-10):**

Для каждой модели из данных pricing — показать изменения цены за период:

| Модель | Канал | Цена тек., ₽ | Цена пред., ₽ | Δ цены, % | СПП тек. | СПП пред. | Цена для клиента |
|---|---|---|---|---|---|---|---|

Подсветить модели с высокой ценовой волатильностью (>5% колебания за неделю).

**V.3 Ценовая стратегия по статусам:**

Средневзвешенные цены и СПП по группам:
- Продается: средняя цена, средний СПП, целевая маржинальность
- Выводим: средняя цена — можно ли снизить для ускорения?
- Запуск: цена vs конкуренты — конкурентоспособна ли?

**V.4 Ценовые рекомендации** (из {{HYPOTHESES}}, только ценовые). С "что если" сценариями:

### VI. Сведение ΔМаржи

|---|---|---|---|---|---|---|---|
| Себестоимость | ... | ...% | ... | ...% | ... | ... п.п. | +... ₽ |
| Комиссия | ... | ...% | ... | ...% | ... | ... п.п. | +... ₽ |
| Логистика | ... | ...% | ... | ...% | ... | ... п.п. | +... ₽ |
| ... | ... | ... | ... | ... | ... | ... | ... |

**Ключевое:** если Δ доли > ±1 п.п. — это структурное изменение. Если > ±3 п.п. — аномалия, подсветить.

Текстовая интерпретация: главный позитивный и негативный фактор. Отличать реальное изменение (доля изменилась) от пропорционального (упало вместе с выручкой, доля не изменилась).

- [ ] Toggle-заголовки: `## ▶` для секций
- [ ] Числа: пробелы в тысячах, ₽, %, п.п.
- [ ] Флаги качества — русским языком, не snake_case
- [ ] Аномалии (Δ доли > 3 п.п.) — подсвечены ⚠️
- [ ] Риски — с конкретными ₽-потерями и горизонтом
