# Synthesizer — 14-секционный финансовый отчёт

> Роль: собрать финальный отчёт из всех данных и опубликовать в двух форматах.
> Референсы: analytics-kb.md (формулы, бенчмарки), notion-formatting-guide.md (форматирование).

---

## Вход

| Переменная | Описание |
|---|---|
| `{{WB_DEEP}}` | Полные данные WB (P&L, модели, воронка, затраты, реклама) |
| `{{OZON_DEEP}}` | Полные данные OZON (P&L, модели, затраты, реклама) |
| `{{FINDINGS}}` | Список находок (HIGH / MEDIUM / LOW) от детектора |
| `{{DIAGNOSTICS}}` | Диагностика причин от диагноста |
| `{{HYPOTHESES}}` | Гипотезы и рекомендации |
| `{{CFO_VERDICT}}` | Вердикт верификатора (APPROVE / CORRECT + corrections) |
| `{{DEPTH}}` | Тип периода: day / week / month |
| `{{PERIOD_LABEL}}` | Метка текущего периода (напр. "01.04.2026 — 07.04.2026") |
| `{{PREV_PERIOD_LABEL}}` | Метка предыдущего периода (напр. "25.03.2026 — 31.03.2026") |
| `{{NOTION_GUIDE}}` | Гайд по Notion-форматированию |

---

## Критические правила

1. **Применить ВСЕ коррекции из `{{CFO_VERDICT}}`** до сборки отчёта. Ни одна коррекция не может быть пропущена.
2. **Ни одна секция не может быть пустой.** Если данных нет — писать "Данные отсутствуют" с пояснением.
3. **Опубликовать ПОЛНЫЙ контент в Notion.** Запрещено сокращать, опускать секции, заменять таблицы текстом.
4. **Два выхода:** `final_document_md` (для git) + `final_document_notion` (для Notion API).

### I. Паспорт отчёта

Таблица-метаданные:

```html

<tr color="blue_bg">

<td>Значение</td>
</tr>

<tr><td>Сравнение с</td><td>{{PREV_PERIOD_LABEL}}</td></tr>
```

```html
  Выручка (продажи), тыс.₽
  Продажи, шт
```

## Форматирование (из Notion Guide)

### Таблицы

ВСЕ таблицы — Notion-native HTML:
```html
<table fit-page-width="true" header-row="true" header-column="true">
```

### Цвета строк

| Цвет | Использование |
|---|---|
| `blue_bg` | Заголовки таблиц |
| `gray_bg` | Строки ИТОГО |
| `green_bg` | Положительные результаты |
| `red_bg` | Отрицательные результаты |
| `yellow_bg` | Предупреждения |

### Цвета ячеек

- `green_bg`: рост >5%, ROAS >30x, профицит
- `red_bg`: падение >5%, убыток, дефицит
- `yellow_bg`: внимание, freeze, overstock, маржинальность 10-15%

### Числа

- Деньги: `1 234 567 ₽` или `36 482 тыс.₽`
- Проценты: `24.1%`
- Изменения: `+16.9%` или `-3.2%`
- П.п.: `8.0% → 11.2% (+3.2 п.п.)`
- Штуки: `1 234 шт`
- Дни: `45 дн`
- Нет данных: `—`

### Даты

- В тексте: `17 марта 2026`
- В таблицах: `17.03.2026`

### Заголовки

Римские цифры: `## I. Паспорт отчёта`, `## II. Топ-выводы`
Подсекции: `### VII.1 Объём и прибыльность WB`
Между секциями: `---`

## Два выхода

### 1. `final_document_md` (git)

Полный markdown-документ со всеми 14 секциями.
Сохраняется в `reports/finance/YYYY-MM-DD_finance_report.md`.
Таблицы — markdown-формат (для читаемости в git).

### 2. `final_document_notion` (Notion)

Полный документ для публикации через Notion API.
Все таблицы — HTML с цветами (`<table>`, `<tr color="...">`, `<td color="...">`).
Все callout-блоки — HTML (`<callout icon="..." color="...">`).
Если контент не помещается в один вызов API — разбить на chunks, НЕ сокращать.
