# Задачи

## 1. Валидация иерархии
- Правильно ли агрегированы color_code? Учтены ли пары ИП/ООО (Vuki+Vuki2, Moon+Moon2)?
- Нет ли артикулов, выпавших из иерархии?

## 2. Трикотажная коллекция — анализ по color_code

### Слой «Комплекты белья» (Vuki, Vuki2, Moon, Moon2, Ruby, Joy)
По каждому color_code:
- Агрегатная оценка: суммарная маржа, средний ROI, среднедневные продажи
- Drill-down по моделям с бенчмарком (медиана коллекции)
- Слабая модель внутри сильного color_code → НЕ "выводить", а "что с ней не так?" (контент? маркетинг? цена?)
- Производственное ограничение: если производим цвет — производим для ВСЕХ моделей в слое

### Слой «Сеты трусов» (Set Vuki, Set Moon, Set Ruby)
- Отдельный color_code слой, аналогичная логика
- Решения по слоям НЕЗАВИСИМЫ

## 3. Бесшовная и другие модели
- Wendy, Audrey, Eva, Charlotte, Bella, Lana, Mia — без color_code привязки
- Каждый артикул (модель + цвет) анализируется НЕЗАВИСИМО
- Группировка по моделям для удобства

## 4. Анализ по статусам
- "Продается" → основной анализ через матрицу ABC × ROI
- "Выводим" → поиск кандидатов на ВОЗВРАТ в "Продается" (восстановился спрос? хорошая маржа?)
- "Новинка/Запуск" → информационный блок: текущие метрики, тренд

## 5. MOQ-анализ
По каждому артикулу/color_code:
- При текущих продажах за сколько месяцев продадим 500 шт?
- Пороги: <1 мес (отлично), 1-3 мес (нормально), 3-6 мес (приемлемо), 6-12 мес (плохо), >12 мес (лимит)
- Выгодно ли заказывать при текущей оборачиваемости?

## 6. Возможности расширения матрицы
- **По цветам:** модель с 2-3 цветами и сильными метриками → рекомендовать добавить новые цвета
- **По размерам:** анализ распределения продаж. Если крайний размер (L) продаётся >50% от соседнего (M) → рекомендовать добавить XL. С оговоркой о влиянии на % выкупа.

# Правила
- Причинно-следственные цепочки обязательны для каждой находки

- Слабая модель внутри сильного color_code ≠ вывод. Разберись ПОЧЕМУ она слабая.
- Сезонность: если 🔄 Сезонный и низкий сезон → не рекомендовать вывод
