### ▶ III.1 Полная артикульная таблица

**Это ядро отчёта.** Полная таблица ВСЕХ артикулов в статусе "Продается", сгруппированная по моделям, внутри моделей — от лучшего к худшему по марже m3.

Сортировка: сначала по модели (алфавит), внутри модели — по margin m3 DESC.

Формат таблицы:

```
| Модель | Артикул | ABC | Ячейка матрицы | Выручка m3 | Маржа m3 | Маржин.% | Реклама m3 | ДРР% | Оборач. дн | Действие |
|---|---|---|---|---|---|---|---|---|---|---|
| **Wendy** | wendy/brown | A | ТЯЖЕЛОВЕС | 612К ₽ | 134К ₽ | 21,9% | 45К ₽ | 7,3% | 720 дн | Оптимизировать стоки |
| | wendy/white | A | ТЯЖЕЛОВЕС | 531К ₽ | 131К ₽ | 24,7% | 30К ₽ | 5,7% | 850 дн | Оптимизировать стоки |
| | wendy/black | A | ТЯЖЕЛОВЕС | 610К ₽ | 124К ₽ | 20,4% | 59К ₽ | 9,7% | 1100 дн | ⚠️ Снизить ДРР |
| **Audrey** | audrey/fig | A | ЗВЕЗДА | 180К ₽ | 52К ₽ | 28,9% | 8К ₽ | 4,4% | 12 дн | 🔴 Пополнить |
```

Для каждого артикула:
- **ABC** — категория A/B/C (по марже m3)
- **Ячейка матрицы** — ЗВЕЗДА, РАБОЧАЯ ЛОШАДКА, ТЯЖЕЛОВЕС и т.д.
- **Выручка m3** — за последний месяц (март)
- **Маржа m3** — абсолютная, в рублях
- **Маржин.%** — маржинальность за m3
- **Реклама m3** — суммарные расходы (internal + external)
- **ДРР%** — за m3
- **Оборач. дн** — оборачиваемость в днях
- **Действие** — краткое из вердикта Арбитра (1-3 слова + эмодзи приоритета)

Модель выделяется **жирным** в первой строке группы, последующие строки модели — пустое поле модели.

Если артикулов >100 — всё равно показать ВСЕ. Это справочная таблица, не summary.

После таблицы "Продается" — аналогичная таблица для статуса "Выводим" (отдельная, с пометкой).

# Правила форматирования

## Терминология — только русский
| Запрещено | Правильно |
|---|---|
| COGS | Себестоимость |
| Margin% | Маржинальность |
| Turnover | Оборачиваемость |
| Revenue | Выручка |
| ROI | Годовая доходность (ROI) |
| DRR | ДРР |

## Числа
| Тип | Формат | Пример |
|---|---|---|
| Рубли | Пробел-разделитель + ₽ | 1 234 567 ₽ |
| Крупные суммы | Сокращённо | 8,8М ₽ |
| Проценты | Одна десятая | 24,1% |
| Изменение % | Со знаком | +16,9%, -3,2% |
| п.п. | п.п. | +3,2 п.п. |
| Штуки | Пробел | 1 234 шт |
| Дни | + дн | 45 дн |
| Нет данных | — | — |

Значимые изменения (>+/-10%) — **жирным**.

# Правила
- ТОЛЬКО данные из вердиктов Арбитра. Никакой собственной аналитики.
- Все ячейки заполнены если данные есть
- Нет данных → "—"
- Никогда не выдумывать модели или артикулы
- Если причина проблемы неизвестна — "требуется проверка", НЕ предполагать конкретную причину
- Недопустимо: "фото не соответствуют цвету". Допустимо: "требуется аудит карточки"
- "Возврат из вывода" — всегда с оговоркой "требует подтверждения причины вывода"
- Рекомендации по рекламе — "протестировать", не "остановить"
- Находки с >5 элементами — ВСЕГДА в табличной форме, НЕ bullet-списками
- Цветовые коды, кандидаты на вывод, растущие артикулы, дефицит запасов, оверсток — ВСЁ таблицами
- Формат таблицы: | Артикул/CC | Маржа | Выручка | ABC | Ключевой показатель | Действие |
- Bullet-списки допустимы только для 1-4 элементов
- Секции с расчётными данными (находки товароведа, финансиста, маркетолога) — в конец отчёта, после действий
