# ABC-Audit Knowledge Base

Справочник бизнес-правил, формул, порогов и терминологии для ABC-аудита ассортимента Wookiee.

---

## 1. ABC-классификация (Парето)

Артикулы ранжируются по убыванию суммарной маржи (абсолютной, в рублях). Накопительным итогом определяются три группы:

| Группа | Доля суммарной маржи | Описание |
|--------|----------------------|----------|
| **A** | 80% | Ключевые артикулы — генерируют основную прибыль |
| **B** | следующие 15% (80-95%) | Средний сегмент — поддерживающий ассортимент |
| **C** | оставшиеся 5% (95-100%) | Хвост — минимальный вклад в прибыль |

Классификация проводится по **абсолютной марже** (не по выручке, не по количеству продаж).

## 2. ROI: формула и зоны

### Формула

```
ROI = маржинальность% × (365 / оборачиваемость_дней)
```

Где:
- `маржинальность%` — маржинальность артикула в процентах (например, 25 означает 25%)
- `оборачиваемость_дней` — количество дней, за которое текущий сток будет распродан при текущей скорости продаж

### Зоны ROI

| Зона | Диапазон | Интерпретация |
|------|----------|---------------|
| Высокий ROI | > 150% | Капитал оборачивается быстро и с высокой отдачей |
| Средний ROI | 50-150% | Приемлемая отдача, есть потенциал оптимизации |
| Низкий ROI | < 50% | Капитал заморожен или маржа слишком низкая |

## 3. Матрица решений 3x3 (ABC x ROI)

| | Высокий ROI (>150%) | Средний ROI (50-150%) | Низкий ROI (<50%) |
|---|---|---|---|
| **A** (80% маржи) | **ЗВЕЗДА** — наращивать производство | **РАБОЧАЯ ЛОШАДКА** — оптимизировать оборачиваемость | **ТЯЖЕЛОВЕС** — срочно работать с ценой/стоками |
| **B** (15% маржи) | **ПОТЕНЦИАЛ** — усилить маркетинг | **СЕРЕДНЯК** — мониторить | **КАНДИДАТ НА ВЫВОД** — снижать остатки |
| **C** (5% маржи) | **НИШЕВЫЙ** — оставить, не инвестировать | **БАЛЛАСТ** — готовить к выводу | **МЁРТВЫЙ ГРУЗ** — выводить немедленно |

### Названия ячеек (для ссылок в отчётах)

- A + Высокий ROI = ЗВЕЗДА
- A + Средний ROI = РАБОЧАЯ ЛОШАДКА
- A + Низкий ROI = ТЯЖЕЛОВЕС
- B + Высокий ROI = ПОТЕНЦИАЛ
- B + Средний ROI = СЕРЕДНЯК
- B + Низкий ROI = КАНДИДАТ НА ВЫВОД
- C + Высокий ROI = НИШЕВЫЙ
- C + Средний ROI = БАЛЛАСТ
- C + Низкий ROI = МЁРТВЫЙ ГРУЗ

## 4. Расчёт оборачиваемости

### Формула

```
оборачиваемость_дней = суммарный_сток_всех_складов / среднедневные_продажи
```

### Источники стока (ВСЕ склады суммируются)

| Склад | Источник | Описание |
|-------|----------|----------|
| WB (FBO) | API Wildberries | Остатки на складах Wildberries |
| OZON | API OZON | Остатки на складах OZON |
| МойСклад (основной) | API МойСклад | Остатки на основном складе |
| МойСклад (в пути) | API МойСклад | Товары в пути к маркетплейсам |

```
суммарный_сток = сток_WB + сток_OZON + сток_МойСклад_основной + сток_МойСклад_в_пути
```

### Среднедневные продажи

Рассчитываются за выбранный горизонт анализа (30 / 90 / 180 дней) как сумма продаж всех каналов, делённая на количество дней.

## 5. MOQ-пороги (MOQ = 500 шт)

Минимальный заказ на производстве — 500 штук. Оценка: на сколько месяцев хватит одного MOQ при текущей скорости продаж.

### Формула

```
месяцев_на_MOQ = 500 / (среднедневные_продажи × 30)
```

### Пороги

| Оценка | Диапазон | Интерпретация |
|--------|----------|---------------|
| Отлично | < 1 мес | Один MOQ расходуется быстро, можно заказывать часто |
| Нормально | 1-3 мес | Стандартная ситуация, MOQ адекватен |
| Приемлемо | 3-6 мес | Запас на полгода — допустимо, но капитал заморожен |
| Плохо | 6-12 мес | Слишком долгая заморозка капитала |
| Лимит | > 12 мес | Заказывать MOQ невыгодно — капитал заморожен > года |

## 6. Color_code: правила группировки

### Трикотажная коллекция (tip_kollekcii = 'tricot')

Трикотаж анализируется по **ДВУМ независимым слоям**. Решения внутри каждого слоя принимаются независимо.

#### Слой 1: Комплекты белья

Модели: **Vuki, Vuki2, Moon, Moon2, Ruby, Joy**

- Артикулы группируются по `color_code`
- Решение принимается на уровне color_code (цвет внутри модели)
- Пример: Vuki/чёрный и Vuki2/чёрный — разные артикулы, но один color_code анализируется в контексте своей модели

#### Слой 2: Сеты трусов

Модели: **Set Vuki, Set Moon, Set Ruby**

- Независимый слой с собственной color_code привязкой
- Решения по сетам НЕ влияют на решения по комплектам и наоборот

### Бесшовная коллекция

Модели: **Wendy, Audrey, Eva, Charlotte, Bella, Lana, Mia**

- **Без color_code** — каждый артикул анализируется независимо
- Нет группировки по цветам

## 8. ИП vs ООО: стратегический контекст

### Юрлица и их стратегии

| Юрлицо | Модели | Стратегия | Целевые показатели |
|--------|--------|-----------|-------------------|
| **ИП** | Vuki, Moon | Высокий объём + приемлемая маржа | Массовый сегмент: доля рынка, оборот |
| **ООО** | Vuki2, Moon2 | Меньший объём + высокая маржа + лучшая упаковка | Премиум: маржинальность, LTV |

### Правило оценки

Каждое юрлицо оценивается по достижению **СВОЕЙ** цели:
- ИП: достигает ли целевого объёма и оборачиваемости?
- ООО: достигает ли целевой маржинальности?

**НЕ допускается** прямое сравнение "кто эффективнее" между ИП и ООО — у них разные бизнес-модели.

## 9. Защита сезонных товаров

### Правило

Если артикул находится в **низком сезоне** и помечен как 🔄 Сезонный:
- **НЕ рекомендовать вывод**
- Вместо этого: "пересмотреть через N месяцев (начало высокого сезона)"

### Применение

- При расчёте ROI и оборачиваемости — отметить, что текущие продажи не репрезентативны
- В рекомендации указать конкретную дату пересмотра
- Confidence автоматически снижается (см. раздел 12)

## 10. Стратегия продажи в убыток

### Когда допустимо

Продажа в убыток допустима, если цель — **быстро высвободить капитал для A-товаров** (ЗВЁЗД).

### Расчёт обоснования

```
убыток_от_распродажи = (себестоимость - цена_распродажи) × количество

стоимость_замороженного_капитала = себестоимость × количество × (оборачиваемость_дней / 365) × ставка_альтернативной_доходности
  + хранение_в_день × количество × оборачиваемость_дней
  + упущенная_выгода_от_A_товаров
```

Если `стоимость_замороженного_капитала > убыток_от_распродажи` — распродажа оправдана.

## 11. Форматирование и терминология

### Терминология (русский язык)

| Термин | Значение |
|--------|----------|
| Маржинальность | Маржа в % от выручки |
| Маржа | Абсолютная прибыль в рублях |
| Оборачиваемость | Дни до полной распродажи текущего стока |
| ДРР | Доля рекламных расходов (% от выручки) |
| СПП | Скидка постоянного покупателя |
| Себестоимость | Закупочная цена + доставка до склада |
| Выкуп | Процент выкупленных заказов (лаговый, 3-21 дн.) |
| OOS | Out of Stock — отсутствие на складе |
| MOQ | Minimum Order Quantity — минимальный заказ |

### Форматы чисел

| Тип | Формат | Пример |
|-----|--------|--------|
| Деньги | Пробел-разделитель + ₽ | 1 234 567 ₽ |
| Проценты | Одна десятая + % | 23.4% |
| Процентные пункты | п.п. | +2.1 п.п. |
| Штуки | Пробел-разделитель + шт | 1 234 шт |
| Дни | дн | 45 дн |
| Изменение (рост) | ▲ | ▲ 12.3% |
| Изменение (падение) | ▼ | ▼ 5.7% |
| Без изменений | ◼ | ◼ 0.0% |

### Callout-блоки

- 🔴 **Критично** — требует немедленного действия
- 🟡 **Внимание** — требует решения в ближайшее время
- 🟢 **Норма** — показатели в пределах целевых значений
- ℹ️ **Информация** — контекст без требования действий

## 12. Уровни уверенности (Confidence)

| Уровень | Диапазон | Критерии |
|---------|----------|----------|
| **HIGH** | > 80% | Данных достаточно (> 30 продаж), периоды согласованы, эксперты согласны |
| **MEDIUM** | 60-80% | Расхождения между периодами или экспертами |
| **LOW** | < 60% | Мало данных, сильная сезонность, эксперты противоречат |

### Факторы, снижающие confidence

- Менее 30 продаж за период
- Расхождение метрик 30д vs 90д > 20%
- Артикул в переходном сезоне
- Недавнее изменение цены (< 14 дней)
- Отсутствие данных по одному из каналов

---

## 13. Адаптивная глубина анализа

### Горизонты

| Горизонт | Применение |
|----------|------------|
| **30 дней** | Базовый — основной период для всех артикулов |
| **90 дней** | Контрольный — сверка с 30д. Если расхождение > 20% → снижаем confidence |
| **180 дней** | Только для спорных случаев (MEDIUM/LOW confidence после 30д и 90д) |

### Правило расширения

Если у артикула **менее 30 продаж** за текущий горизонт — автоматически расширить горизонт до следующего уровня (30 → 90 → 180).

### Логика сверки

1. Рассчитать метрики за 30 дней
2. Рассчитать метрики за 90 дней
3. Если расхождение ключевых метрик (маржинальность, оборачиваемость, ROI) > 20% — пометить как MEDIUM confidence
4. Для MEDIUM — дополнительно проверить 180 дней
5. Если все горизонты согласованы — вернуть HIGH

## 15. Вердикты

Каждый артикул (или color_code группа) получает один из пяти вердиктов:

| Вердикт | Описание | Типичные ячейки матрицы |
|---------|----------|------------------------|
| **НАРАЩИВАТЬ** | Увеличить производство и запасы | ЗВЕЗДА |
| **ПОДДЕРЖИВАТЬ** | Сохранить текущий уровень, не снижать | РАБОЧАЯ ЛОШАДКА, ПОТЕНЦИАЛ |
| **ОПТИМИЗИРОВАТЬ** | Улучшить метрики (цена, реклама, оборачиваемость) | ТЯЖЕЛОВЕС, СЕРЕДНЯК |
| **ГОТОВИТЬ К ВЫВОДУ** | Снизить остатки, не пополнять, найти замену | КАНДИДАТ НА ВЫВОД, БАЛЛАСТ |
| **ВЫВОДИТЬ НЕМЕДЛЕННО** | Распродать остатки любым способом | МЁРТВЫЙ ГРУЗ |

Для НИШЕВОГО — вердикт ПОДДЕРЖИВАТЬ (оставить, не инвестировать).

## 16. Формат причинно-следственной цепочки

### Неправильно

> Joy/чёрный — кандидат на вывод

### Правильно

> Joy/чёрный: маржинальность 18%, ROI 72% (vs медиана коллекции 176%). ДРР = 2% (другие модели 6-9%) → недоинвестирован в рекламу → низкие продажи (2 шт/д vs медиана 8) → медленная оборачиваемость (92 дн)

### Шаблон

```
{артикул}: {ключевая_метрика} {значение} (vs {бенчмарк} {значение_бенчмарка}).
{причина_1} → {следствие_1} → {следствие_2} → {итоговый_эффект}
```

### Правила

1. **Всегда** указывать числа с единицами измерения
2. **Всегда** давать бенчмарк (медиана коллекции, среднее по группе ABC)
3. **Всегда** строить цепочку от причины к следствию (не просто констатировать факт)
4. Цепочка должна быть **проверяемой** — каждое звено опирается на данные
5. Если причина неизвестна — явно указать: "причина не определена, требуется проверка"
