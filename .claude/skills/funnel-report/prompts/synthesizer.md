## КРИТИЧЕСКИЕ ПРАВИЛА

### Формат — Notion Enhanced Markdown

1. **Таблицы** — через `<table>` с атрибутами:
   - `fit-page-width="true"` — растягивать на ширину страницы
   - `header-row="true"` — первая строка = шапка
   - `header-column="true"` — первая колонка = заголовок строки (если применимо)

2. **Цвета строк** — `<tr color="blue_bg">` для шапок таблиц:
   - `blue_bg` — шапки таблиц (заголовки колонок)
   - `green_bg` — позитивные строки (рост CRO, рост заказов, рост маржи)
   - `red_bg` — негативные строки (падение CRO > 0.5pp, падение заказов > 15%)
   - `gray_bg` — итоговые строки

3. **Цвета ячеек** — `<td color="green_bg">` / `<td color="red_bg">` для отдельных значений:
   - Значимый рост (> +10%) → `color="green_bg"`
   - Значимое падение (> -10%) → `color="red_bg"`

4. **Callout-блоки** — `<callout icon="💡" color="green_bg">`:
   - `icon="💡" color="green_bg"` — позитивный инсайт
   - `icon="⚠️" color="yellow_bg"` — предупреждение (выкупы, качество данных)
   - `icon="🔥" color="green_bg"` — сильный позитив
   - `icon="📊" color="blue_bg"` — нейтральный итог
   - `icon="🚨" color="red_bg"` — критическая проблема

5. **Toggle-заголовки** — `## Модель: Name — headline {toggle="true"}` для модельных секций
6. **Tab-indentation** (`\t`) для контента внутри toggle
7. **Разделители** — `---` между секциями

### Данные

8. **ТОЛЬКО реальные данные.** Если данных нет — писать "н/д". НИКОГДА не придумывать.
9. **ТОЛЬКО реальные модели** из sku_statuses. ЗАПРЕЩЕНО придумывать.
10. **Выкуп\*:** ВСЕГДА со сноской `*Данные по выкупам неполные (лаг 3-21 день)`.

### Числа

11. Формат: `1 234 567 ₽`, `24,1%`, `+3,2 п.п.`
12. Крупные суммы: `8,8М` — не `8 800 000 ₽`
13. Значимые изменения: **bold** (`**-21.5%**`, `**+0.50pp**`)

### Терминология

14. **Русская ТОЛЬКО:** Переходы (не card_opens), Корзина (не cart), Выкупы (не buyouts), ДРР (не DRR), Маржа (не Margin).
