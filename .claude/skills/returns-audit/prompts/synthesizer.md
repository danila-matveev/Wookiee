```json
{{MODEL_CARDS}}
```

### Верификатор:
```json
{{VERIFIER_OUTPUT}}
```

### Период: {{PERIOD_LABEL}}

## Задача

Собери ОДИН связный Notion-документ. Не просто объединяй — переосмысли, расставь приоритеты, свяжи факты.

## Структура документа

### 1. Заголовок
"Аудит возвратов — {{PERIOD_LABEL}}"

### 2. Сводка (callout, синий)
- Всего заявок: N (N% к заказам)
- Тренд: растёт / стабильно / падает
- Топ-3 проблемных модели (одной строкой)

### 3. Динамика по дням
- Таблица: дата | заявки | кумулятивно
- Под таблицей: комментарий по пиковым дням

### 4. Причины возвратов
- Таблица: причина | кол-во | % | actionable?
- Топ-3 причины с пояснением
- Callout (жёлтый): рекомендации по actionable причинам

### 5. Карточки моделей
Для каждой модели — toggle heading:
- **Модель X** (N заявок, N% к продажам) [красный если rate > brand avg × 1.5, жёлтый если > avg, зелёный если < avg]
  - Причины: bullet list
  - Комментарии: 2-3 цитаты курсивом
  - Рекомендация: bold text

### 6. Прочие модели (1-2 заявки)
- Сводная таблица: модель | заявок | причина

### 7. Выводы и рекомендации
- Numbered list, sorted by priority (HIGH → LOW)
- Каждая рекомендация: что делать + ожидаемый эффект

## Notion-форматирование
- Toggle headings (##) для карточек моделей
- Callout blocks: красный для критичных, жёлтый для внимания, синий для info, зелёный для позитива
- Native tables (не markdown tables)
- Bold для ключевых метрик
- Italic для цитат покупателей

## Формат вывода

Верни JSON:

```json
{
  "final_document_notion": "... полный текст с Notion-разметкой ...",
  "publish_title": "Аудит возвратов — {{PERIOD_LABEL}}",
  "publish_data_source_id": "returns_audit"
}
```

## Правила
- Если верификатор дал FLAG — добавь callout с пометками в начало документа
- Если REJECT — не создавай документ, верни ошибку
- Все цифры проверяй по сырым данным, не доверяй только аналитикам
- Документ должен быть actionable — читатель должен знать что делать после прочтения
