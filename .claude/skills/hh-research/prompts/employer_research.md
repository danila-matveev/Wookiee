# Employer Research — детальные инструкции

Цель: собрать профиль компании на HH — описание, активные вакансии, частоту найма, разрезы по ролям/уровням, технологический стек.

## Шаг 1. Идентификация работодателя

Попроси у пользователя название компании. При наличии прямого `employer_id` или ссылки на профиль hh.ru — извлеки `employer_id` из URL (формат `https://hh.ru/employer/<id>`).

Если только название:
1. `mcp__headhunter__hh_search_vacancies(text="<имя компании>", per_page=10, page=0)` — выводы возвращают `employer.id` и `employer.name`.
2. Возьми `employer.id` первой вакансии, у которой `employer.name` точно совпадает (нечёткое совпадение — переспроси пользователя «вы имели в виду X или Y?»).

## Шаг 2. Профиль работодателя

`mcp__headhunter__hh_get_employer(employer_id=<id>)`

Извлеки:
- `name`, `description` (HTML, очисть от тегов),
- `site_url`, `industries[].name`,
- размер компании, если есть,
- `vacancies_url` для проверки.

## Шаг 3. Активные вакансии

```
mcp__headhunter__hh_search_vacancies(
  employer_id=<id>,
  per_page=100,
  page=0
)
```

Если `found > 100` — продолжи `page=1, 2, ...` до полного покрытия (макс 5 страниц).

## Шаг 4. Агрегация

Считай:

### A. Открытые позиции
- Всего активных вакансий.
- Топ-5 ролей (группировка по `name` через нормализацию: lower + удаление уровня junior/middle/senior).
- Распределение по уровням (`experience` enum).

### B. Темп найма
- Гистограмма `published_at` за последние 30/60/90 дней.
- Сколько вакансий опубликовано за последние 7 дней — индикатор активного найма прямо сейчас.
- Какие роли публикуются быстрее всего (если несколько вакансий с одинаковым названием — это «горящая» роль).

### C. Зарплатная политика
- Какая доля вакансий с указанной зарплатой.
- Медиана `salary.from` / `salary.to` по компании vs медиана рынка (для сопоставимой роли запроси Stage 1: market_scan по той же роли в том же регионе для сравнения).
- Где компания платит выше / ниже рынка.

### D. Технологический стек
- Из `key_skills` всех вакансий — топ-15 тегов.
- Группировка по доменам: backend / frontend / data / devops / ML / mobile / management.

### E. Формат и география
- Распределение `schedule` (remote / fullDay / flexible).
- Топ-3 регионов публикации.

## Шаг 5. Выдача

```markdown
# Профиль работодателя: <Название>

**Индустрия:** <…>
**Сайт:** <…>
**Размер:** <если есть>

## Активный найм
- Открытых позиций: <N>
- Опубликовано за 7 дней: <K> ⚠️ — индикатор горящего найма
- Темп публикации: <среднее в неделю за 90 дней>

## Топ ролей
| Роль | Вакансий | Уровень |
|------|----------|---------|
| ... | ... | ... |

## Технологический стек (топ-15)
- ...

## Зарплатная политика
- Медиана от/до: <…>
- vs рынок: <выше/ниже на X%>
- Доля вакансий с указанной зарплатой: <…>

## Формат работы
- Remote: X% / FullDay: Y% / Flexible: Z%

## Выводы
- Чем компания привлекательна для соискателя: <…>
- Что насторожит: <…>
- Сегменты, где они активно растут: <…>
```

Сохрани в `data/hh_research/employers/<employer-slug>/<YYYY-MM-DD>.md`.

В чат — краткий summary: «Компания X нанимает активно (Y вакансий, Z за неделю), фокус на ролях A/B/C, зарплаты на N% выше/ниже рынка».
