# Notion Formatting Guide — Analytics Report

## Table Format

Все таблицы — Notion-native HTML. Markdown-таблицы запрещены.

```html
<table fit-page-width="true" header-row="true" header-column="true">
<tr color="blue_bg">
<td>Показатель</td>
<td>Текущий период, тыс.₽</td>
<td>Предыдущий период, тыс.₽</td>
<td>Δ</td>
<td>Δ%</td>
</tr>
<tr>
<td>Выручка</td>
<td>36 482</td>
<td>31 205</td>
<td color="green_bg">+5 277</td>
<td color="green_bg">+16.9%</td>
</tr>
<tr color="gray_bg">
<td>**ИТОГО**</td>
<td>**43 077**</td>
<td>**37 800**</td>
<td>**+5 277**</td>
<td>**+14.0%**</td>
</tr>
</table>
```

## Row Colors

| Color | Usage |
|---|---|
| `blue_bg` | Header rows (заголовки таблиц) |
| `gray_bg` | Total/summary rows (строки ИТОГО) |
| `green_bg` | Positive highlight rows (положительные результаты) |
| `red_bg` | Negative/warning rows (отрицательные результаты) |
| `yellow_bg` | Warning rows (требуют внимания) |
| `purple_bg` | Strategic decision rows (стратегические решения) |

## Cell Colors

Применяются к отдельным ячейкам через `<td color="...">`:

| Color | Usage |
|---|---|
| `green_bg` | Положительные значения: рост >5%, ROAS >30x, статус OK, профицит |
| `red_bg` | Отрицательные значения: падение >5%, УБЫТОК, ДЕФИЦИТ, МЕРТВЫЙ СТОК |
| `yellow_bg` | Предупреждения: ВНИМАНИЕ, FREEZE, OVERSTOCK, маржинальность 10-15% |

## Callout Blocks

```html
<callout icon="📊" color="blue_bg">
  Сводка по секции — ключевые цифры.
</callout>

<callout icon="💡" color="green_bg">
  Позитивный инсайт или рекомендация.
</callout>

<callout icon="⚠️" color="yellow_bg">
  Дисклеймер по качеству данных или предупреждение.
</callout>

<callout icon="🚨" color="red_bg">
  СРОЧНОЕ действие. Потери X ₽/мес при бездействии.
</callout>

<callout icon="🔥" color="purple_bg">
  Стратегическое решение или новая инициатива.
</callout>
```

Когда использовать callout:
- После Section 0 (Executive Summary) — ключевые метрики
- После Section 1 (Топ-находки) — срочные действия
- После Section 4 (Цены) — топ рекомендация по ценам
- После Section 7 (Маркетинг) — решение по бюджету
- В Section 9 (Драйверы) — зеленый для драйверов, красный для антидрайверов
- В Section 11 (Рекомендации) — по приоритетам

## Toggle Headings

**ВСЕ заголовки на ЛЮБОМ уровне должны быть toggle headings:**

| Markdown | Notion | Уровень |
|---|---|---|
| `##` (H2) | Toggle Heading 1 | Основные секции (0-12) |
| `###` (H3) | Toggle Heading 2 | Подсекции (WB, OZON, внутри секций) |
| `####` (H4) | Toggle Heading 3 | Детали (по модели, по кампании) |

Каждая секция сворачиваема. Читатель видит outline документа и раскрывает нужные секции.

## Number Formatting

| Тип | Формат | Пример |
|---|---|---|
| Деньги (рубли) | Пробел-разделитель тысяч + ₽ | `1 234 567 ₽` |
| Деньги (тысячи) | + тыс.₽ | `36 482 тыс.₽` |
| Проценты | Одна десятая | `24.1%` |
| Изменение % | Знак + стрелка | `+16.9%` или `-3.2%` |
| Изменение п.п. | С указанием п.п. | `8.0% → 11.2% (+3.2 п.п.)` |
| Штуки | Пробел-разделитель | `1 234 шт` |
| Дни | + дн | `45 дн` |
| Отсутствие данных | Тире | `—` (не "-", не "N/A") |

ЗАПРЕЩЕНО:
- `1,234,567` (английский формат)
- `1234567` (без разделителей)
- `24,1%` (запятая в процентах — только точка)
- `~24%` (приблизительно — указывать точно или маркировать confidence)

## Date Format

| Контекст | Формат | Пример |
|---|---|---|
| В тексте | День + месяц прописью + год | `17 марта 2026` |
| В таблицах | ДД.ММ.ГГГГ | `17.03.2026` |
| В заголовках секций | Месяц + год | `Март 2026` |
| Период | Через тире | `01.03.2026 — 31.03.2026` |

ЗАПРЕЩЕНО:
- `2026-03-17` (ISO формат)
- `17/03/2026` (slash формат)
- `Mar 17, 2026` (английский формат)

## Russian Headers

Все заголовки — на русском языке. Модели — с заглавной буквы.

Правильно:
- `## Ключевые изменения бренда`
- `### Wildberries`
- `#### Модель Wendy`

Неправильно:
- `## Key Changes`
- `### WB Revenue`
- `#### wendy model`

## Table Column Headers

Заголовки столбцов — полные, на русском, с единицами:

Правильно:
- `Выручка WB, тыс.₽`
- `Маржинальность, %`
- `Остаток, шт`
- `Оборачиваемость, дн`

Неправильно:
- `WB Rev, K` (аббревиатуры, английский)
- `Margin%` (без пробела, английский)
- `Stock` (без единицы, английский)

## Delta Formatting in Tables

Для колонок с изменениями:

```html
<td color="green_bg">+5 277 (+16.9%)</td>
<td color="red_bg">-1 423 (-8.3%)</td>
<td>+124 (+0.8%)</td>  <!-- нейтральный: |Δ| < 5% -->
```

Пороги цветовой индикации:
- Зеленый: изменение >+5% (или >+2 п.п. для маржинальности)
- Красный: изменение <-5% (или <-2 п.п. для маржинальности)
- Без цвета: -5% < Δ < +5%

## Model Name Capitalization

Модели всегда с заглавной буквы:
- Wendy, Devi, Luna, Elsa, Ariel, Jasmine, Belle, Aurora, Moana, Tiana, Mulan, Rapunzel, Merida, Pocahontas, Cinderella

В GROUP BY используется LOWER(), но в отображении — Title Case.
