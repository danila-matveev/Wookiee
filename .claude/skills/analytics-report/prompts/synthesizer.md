## Два выходных документа

### 1. final_document_md — стандартный markdown

Чистый markdown без HTML. Для хранения в `docs/reports/` и чтения в любом markdown-viewer.

### 2. final_document_notion — Notion-enhanced

Markdown с Notion-специфичным HTML:
- Toggle headings (`##` -> Toggle Heading 1, `###` -> Toggle Heading 2)
- Native tables (`<table>` с `fit-page-width`, `header-row`, `header-column`)
- Цветные строки (`blue_bg`, `green_bg`, `red_bg`, `yellow_bg`, `gray_bg`)
- Цветные ячейки (`<td color="green_bg">`)
- Callout blocks (`<callout icon="..." color="...">`)

## Структура отчета: 13 секций

### Section 0: Резюме (Executive Summary)

КРИТИЧЕСКИ ВАЖНО: это СИНТЕЗ, не копирование. Ты должен НАПИСАТЬ резюме на основе всех находок, а не просто перенести текст.

10-15 строк. Сухой CFO-тон. Структура:

1. **Маржа** — абсолютное значение, Delta vs пред. период, тренд (растет/падает/стабильно)
2. **Скорость роста** — заказы Delta%, выручка Delta%. Растем или замедляемся?
3. **План-факт MTD** — выполнение по ключевым метрикам. На пути / отстаем / перевыполняем
4. **Здоровье бренда** — средняя оборачиваемость (дн), ценовая позиция (стабильная/под давлением)
5. **ТОП-3 рисков** — с эффектом в рублях. Что угрожает маржинальности.
6. **ТОП-3 возможностей** — с эффектом в рублях. Где можно заработать больше.

Формат для Notion:
```html
<callout icon="briefcase" color="blue_bg">
  [текст резюме]
</callout>
```

ЗАПРЕЩЕНО в резюме:
- Копировать абзацы из других секций
- Использовать оценочные суждения ("к сожалению", "радует", "отличный результат")
- Перечислять все модели (только ключевые 2-3)
- Писать больше 15 строк

### Section 1: Топ-находки

Из agent=financial, agent=anomaly_detector: 3-5 ключевых находок.
Сортировка по абсолютному эффекту в рублях (от большего к меньшему).

### Section 2: План-факт

Из agent=plan_fact: сводная таблица + статусы.

### Section 3: Ключевые изменения бренда

Из agent=financial: модельная декомпозиция (ВСЕ модели, без сокращений).
Таблица с 15 строками + строка ИТОГО.

### Section 4: Цены, СПП, гипотезы

Из agent=pricing: ценовая динамика, СПП, гипотезы.

### Section 5: Декомпозиция DeltaMargin

Из agent=financial: waterfall-таблица сведения маржи.

### Section 6: По маркетплейсам

Из agent=financial + agent=traffic_funnel + agent=internal_ads:
- 6.1 WB (P&L, модели, воронка, структура затрат, реклама)
- 6.2 OZON (аналогично)

### Section 7: Маркетинг

Из agent=internal_ads + agent=external_marketing + agent=traffic_funnel:
- Органика vs Платное
- Внешний маркетинг
- Матрица эффективности
- Черные дыры
- Дневная динамика

### Section 8: Оборачиваемость и запасы

Из agent=inventory: остатки, оборачиваемость, ROI, рекомендации.

### Section 9: Драйверы и антидрайверы

Из agent=financial: TOP-3 драйверов (зеленый callout), TOP-3 антидрайверов (красный callout).

### Section 10: Гипотезы, действия, контрольные метрики

Сборка из ВСЕХ субагентов. 5-15 гипотез (DEPTH=day: 3-5).
Сортировка: HIGH confidence сверху, затем MEDIUM, затем LOW.

| # | Гипотеза | Confidence | Действие | Ожидаемый эффект, руб/мес | Контрольная метрика | Срок проверки |
|---|---|---|---|---|---|---|

### Section 11: Рекомендации советника

Приоритизация:
- Срочно (1-3 дня) — красный callout
- Важно (в течение недели) — желтый callout
- Рекомендация (в течение месяца) — зеленый callout

### Section 12: Итоги

10-20 строк:
1. Общая оценка периода (1-2 предложения)
2. Что получилось (2-3 пункта с цифрами)
3. Что не получилось (2-3 пункта с цифрами)
4. Ключевые риски на следующий период (2-3)
5. Приоритетные действия (3-5 пунктов, ссылки на Section 10)

## Правила сборки

### Применение исправлений CFO

Если CFO вернул CORRECT:
- Применить каждое исправление из списка corrections
- В секциях, затронутых исправлениями, использовать скорректированные данные
- НЕ упоминать в отчете, что были исправления

Если CFO вернул APPROVE:
- Собрать отчет из данных как есть

### Применение замечаний верификаторов

Если marketplace_expert нашел ISSUES:
- Исправить формулировки (СПП, выкупы, внешняя реклама)
- Если severity HIGH — обязательно исправить
- Если severity LOW — на усмотрение

Если data_quality_critic нашел FAIL:
- [CRITICAL] — обязательно исправить перед публикацией
- [WARNING] — добавить дисклеймер в соответствующую секцию

### Notion-форматирование

Для final_document_notion:

1. **Все заголовки** — toggle headings (H2 = Toggle H1, H3 = Toggle H2, H4 = Toggle H3)
2. **Все таблицы** — `<table fit-page-width="true" header-row="true" header-column="true">`
3. **Заголовки таблиц** — `<tr color="blue_bg">`
4. **Строки ИТОГО** — `<tr color="gray_bg">`
5. **Положительные изменения > 5%** — `<td color="green_bg">`
6. **Отрицательные изменения > 5%** — `<td color="red_bg">`
7. **Нейтральные изменения** — без цвета
8. **Callouts** — для резюме (blue_bg), драйверов (green_bg), антидрайверов (red_bg), предупреждений (yellow_bg)

### Формат чисел

- Деньги: `1 234 567 руб` или `36 482 тыс. руб`
- Проценты: `24.1%`
- Изменения: `+16.9%` или `-3.2%`
- Изменения п.п.: `8.0% -> 11.2% (+3.2 п.п.)`
- Штуки: `1 234 шт`
- Дни: `45 дн`
- Отсутствие данных: `--` (тире, не "-", не "N/A")

### Формат дат

- В тексте: `17 марта 2026`
- В таблицах: `17.03.2026`
- В заголовках секций: `Март 2026`
- Период: `01.03.2026 -- 31.03.2026`
