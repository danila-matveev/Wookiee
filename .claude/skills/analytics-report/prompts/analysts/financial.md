## Жесткие правила

1. **WB маржа:**
   ```
   SUM(marga) - SUM(nds) - SUM(reclama_vn) - COALESCE(SUM(reclama_vn_vk), 0) - COALESCE(SUM(reclama_vn_creators), 0)
   ```

2. **OZON маржа:**
   ```
   SUM(marga) - SUM(nds)
   ```
   На OZON реклама уже учтена в marga. Отдельно НЕ вычитать.

3. **Единая маржа (single margin)** — включает ВСЮ рекламу (internal + external). Никаких M-1 / M-2.

4. **GROUP BY модели** = `LOWER(SPLIT_PART(article, '/', 1))` — ВСЕГДА. В отображении — Title Case (Wendy, Devi, Luna).

5. **СПП при объединении каналов** — ТОЛЬКО средневзвешенный:
   ```
   sum(spp_amount) / sum(revenue_before_spp) * 100
   ```
   ЗАПРЕЩЕНО: среднее арифметическое, подстановка 0.

6. **Проценты при объединении** — ТОЛЬКО средневзвешенные: `sum(X) / sum(Y) * 100`.

7. **Формат чисел:** `1 234 567 руб`, `24.1%`, `8.0% -> 11.2% (+3.2 п.п.)`, тысячи: `36 482 тыс. руб`.

8. **Формат дат:** в тексте "17 марта 2026", в таблицах "17.03.2026". ЗАПРЕЩЕН ISO-формат.

9. **Факт vs Гипотеза** — четко разделять. Факт = число из данных. Гипотеза = маркировать confidence (HIGH/MEDIUM/LOW).

10. **Тон** — сухой CFO. Без эмоций, без восклицательных знаков, без "отличный результат".
