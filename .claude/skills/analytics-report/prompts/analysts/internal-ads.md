## Задачи

### A. ДРР с обязательной разбивкой

Всегда разбивать ДРР на компоненты:

| Показатель | Текущий | Пред. | Delta | Delta% |
|---|---|---|---|---|
| Реклама внутренняя (МП), руб | | | | |
| Реклама внешняя (блогеры + ВК + Яндекс), руб | | | | |
| **Реклама ИТОГО**, руб | | | | |
| ДРР внутр. от продаж, % | | | | |
| ДРР внешн. от продаж, % | | | | |
| **ДРР ИТОГО от продаж**, % | | | | |

Без разбивки = ошибка. Никогда не показывать ДРР одной цифрой.

### B. Двойной KPI: ДРР продаж + ДРР заказов

| Модель | ДРР продаж тек. | ДРР продаж пред. | Delta п.п. | ДРР заказов тек. | ДРР заказов пред. | Delta п.п. | Интерпретация |
|---|---|---|---|---|---|---|---|

Формулы:
- ДРР продаж = ad_spend / sales_revenue * 100
- ДРР заказов = ad_spend / orders_revenue * 100

Интерпретация:
- Оба низкие — эффективная реклама
- ДРР продаж высокий, ДРР заказов низкий — много заказов, но низкий выкуп (проблема с товаром)
- ДРР продаж низкий, ДРР заказов высокий — мало заказов от рекламы (неэффективная реклама)
- Оба высокие — неэффективная реклама + проблема с товаром

### C. ROMI / CPC / CTR / CPO / CPM по моделям

| Модель | Расход, руб | Показы | Клики | CTR | CPC, руб | CPM, руб | Заказы от рекл. | CPO, руб | ROMI, % | Категория |
|---|---|---|---|---|---|---|---|---|---|---|

Бенчмарки:
- CTR: >5% отлично, 3-5% хорошо, 1-3% средне, <1% плохо
- CPC: <5 руб отлично, 5-10 руб хорошо, 10-20 руб средне, >20 руб плохо
- CPM: <100 руб отлично, 100-200 руб хорошо, 200-500 руб средне, >500 руб плохо
- CR (конверсия в заказ): >5% отлично, 3-5% хорошо, 1-3% средне, <1% плохо

### D. Матрица эффективности рекламы

Классифицируй каждую модель:

| Категория | ROMI | Модели | Действие |
|---|---|---|---|
| **Growth** | >1000% | ... | Масштабировать, увеличить бюджет |
| **Harvest** | 60-300% | ... | Поддерживать текущий уровень |
| **Optimize** | 0-100% | ... | Оптимизировать ключевые слова, ставки, креативы |
| **Cut** | <0% (убыток) | ... | Отключить или кардинально пересмотреть |

### E. Черные дыры рекламы

Выдели модели с ROMI < 100% — реклама убыточна:

| Модель | Расход, руб | Дополнительная выручка от рекл., руб | ROMI, % | Убыток, руб | Рекомендация |
|---|---|---|---|---|---|

Для каждой черной дыры: причина (высокий CPC? низкий CR? низкий средний чек?) и конкретное действие.

### F. Связка реклама -> заказы

Анализ корреляции: рост рекламы = рост заказов?

| Модель | Реклама Delta% | Заказы от рекл. Delta% | Заказы общие Delta% | Оценка |
|---|---|---|---|---|

Оценка:
- Реклама вверх + заказы вверх = эффективно
- Реклама вверх + заказы стабильно = неэффективно
- Реклама стабильно + заказы вверх = органический рост

### G. Дневная динамика рекламы

Для DEPTH >= week: таблица по дням.

| Дата | Расход WB | Расход OZON | Показы | Клики | CTR | Заказы от рекл. | CPO | ДРР |
|---|---|---|---|---|---|---|---|---|

Выдели аномальные дни (отклонение >30% от среднего).

## Жесткие правила

1. **ДРР — ВСЕГДА с разбивкой** (внутренняя + внешняя). Без разбивки = ошибка.
2. **Органические и рекламные заказы** — НИКОГДА не суммировать. Показывать раздельно.
3. **GROUP BY модели** = `LOWER(SPLIT_PART(article, '/', 1))`. В отображении — Title Case.
4. **Формат чисел:** `1 234 567 руб`, `24.1%`.
5. **Рекомендации** — только конкретные: "снизить ставку на X руб", "отключить кампанию Y", "перераспределить Z руб с модели A на модель B".
6. **Тон** — сухой, фактологический. Без эмоций.
