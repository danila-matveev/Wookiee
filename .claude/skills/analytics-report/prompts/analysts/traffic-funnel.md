### C. Органический vs Платный трафик

| Заказы | | | | | | |
| CR переход->заказ | | | | | | |

### D. Модели: ad-dependent vs organic

Классифицируй модели по зависимости от рекламы:

| Модель | Заказы общие | Заказы от рекл. | Доля рекл. заказов, % | Тип | Риск |
|---|---|---|---|---|---|

Тип:
- **Organic-driven** (доля рекл. < 30%) — устойчивая модель
- **Mixed** (30-60%) — сбалансированная
- **Ad-dependent** (> 60%) — при снижении рекламы заказы упадут

### E. Воронка по моделям

| Модель | Показы | CR карт. | CR корз. | CR заказ | Выкуп%* | Delta CR корз. п.п. | Delta CR заказ п.п. | Проблема |
|---|---|---|---|---|---|---|---|---|

Выделить модели с аномальными конверсиями.

### F. Диагностика аномалий воронки

При обнаружении аномалии — диагностировать по таблице:

| Симптом | Что проверить | Вероятная причина |
|---|---|---|
| CR карточка->корзина < 5% | Контент/фото, цена vs конкуренты | Неконкурентная карточка или цена |
| CR корзина->заказ < 15% | Ценовая конкурентоспособность | Покупатель сравнивает и уходит к конкурентам |
| Переходы падение > 20% | Позиции, рекламный бюджет, сезонность | Потеря позиций или снижение рекламы |
| ДРР рост > 3 п.п. | Стоимость клика, конверсия рекламы | Перегрев аукциона или падение качества |
| Органика падение > 10 п.п. | Позиции, новые конкуренты | Алгоритмические изменения WB |
| Выкупы падение (заказы стабильны) | Качество товара, размерный ряд | Проблема с товаром, не с маркетингом |
| CAC > прибыль на продажу | Вся рекламная воронка | Реклама убивает маржу |
| ROMI < 100% | Ставки, ключевые слова, CR | Убыточная реклама |

Для каждой найденной аномалии: описание + числа + вероятная причина + рекомендуемое действие.

## Предупреждения по качеству данных

1. **Выкупы** — ЛАГОВЫЙ показатель (3-21 дн). ЗАПРЕЩЕНО использовать как причину изменения маржинальности в дневном отчете. В недельном — с оговоркой. В месячном — полностью надежен.

2. **Трафик** — расхождение ~20% с PowerBI. Данные `content_analysis` дают цифры на ~20% ниже. Использовать как ТРЕНД, не абсолют. Указать в дисклеймере.

3. **OZON органика** — недоступна через API. Используется `get_ozon_organic_estimated()` — маркировать как "оценка".

---

## Жесткие правила

1. **Органик + Paid** = НИКОГДА не суммировать. Несравнимые, перекрывающиеся метрики.
2. **Выкуп %** — не причина в дневном анализе. Пометка "лаг 3-21 дн."
3. **GROUP BY модели** = `LOWER(SPLIT_PART(article, '/', 1))`. В отображении — Title Case.
4. **Формат чисел:** `1 234 567`, `24.1%`, `+3.2 п.п.`.
5. **Тон** — сухой, фактологический.
