## Задачи

### A. Сводная таблица план-факт

| Метрика | План (месяц) | Факт MTD | План MTD* | Выполнение MTD, % | Прогноз** | Прогноз vs План, % | Статус |
|---|---|---|---|---|---|---|---|

*План MTD = План(месяц) * (дней прошло / дней в месяце) — пропорциональный.
**Прогноз = Факт MTD / дней прошло * дней в месяце.

Метрики (все с разбивкой WB + OZON + ИТОГО):
1. Заказы, шт
2. Заказы, руб
3. Продажи, шт
4. Продажи, руб
5. Выручка, руб
6. Маржа, руб
7. Маржинальность, %
8. Реклама внутренняя, руб
9. Реклама внешняя, руб
10. ДРР внутр., %
11. ДРР внешн., %

### B. Статусы выполнения

Для каждой метрики:

| Статус | Выполнение MTD | Индикатор |
|---|---|---|
| Перевыполнение | > 105% | Зеленый |
| В плане | 95-105% | Желтый (нейтральный) |
| Отставание | < 95% | Красный |

### C. План-факт по моделям/артикулам

| Модель | Канал | Заказы план, шт | Заказы факт, шт | Выполн., % | Выручка план | Выручка факт | Выполн., % | Маржа план | Маржа факт | Выполн., % | Статус |
|---|---|---|---|---|---|---|---|---|---|---|---|

Сортировка: модели с наибольшим отставанием от плана сверху.

### D. Прогноз и рекомендации

Для каждой метрики с отставанием (< 95%):

| Метрика | Отставание, руб | Что нужно для выполнения | Рекомендация | Реалистичность |
|---|---|---|---|---|

Рекомендации — конкретные:
- "Для выполнения плана по заказам WB: необходимо +X заказов/дн (текущий темп Y/дн)"
- "Увеличить рекламный бюджет WB на X руб/дн для привлечения дополнительных Z заказов"
- "Снизить цену модели A на X% для ускорения продаж (текущий остаток позволяет)"

Реалистичность:
- **Достижимо** — отставание < 10%, есть рычаги (бюджет, цена, остатки)
- **Под вопросом** — отставание 10-25%, требуются значительные усилия
- **Нереалистично** — отставание > 25%, план не будет выполнен

### E. Анализ отклонений

Для каждой модели с значимым отклонением (> 10%):

| Модель | Отклонение, % | Вероятная причина | Рекомендуемое действие |
|---|---|---|---|

Причины проверять:
- OOS (out of stock) — модель отсутствовала на складе
- Ценовое изменение — цена выросла/снизилась
- Рекламное изменение — бюджет увеличен/сокращен
- Сезонность — спрос выше/ниже ожидаемого
- Конкуренция — новые конкуренты/акции конкурентов

## Жесткие правила

1. **План-факт ОБЯЗАТЕЛЕН** в каждом отчете, независимо от DEPTH.
2. **План MTD** — пропорциональный (линейная интерполяция). Не корректировать на сезонность без данных.
3. **Прогноз** — экстраполяция текущего темпа. Для DEPTH=month можно использовать тренд.
4. **Все метрики** — с разбивкой WB + OZON + ИТОГО.
5. **GROUP BY модели** = `LOWER(SPLIT_PART(article, '/', 1))`. В отображении — Title Case.
6. **Формат чисел:** `1 234 567 руб`, `24.1%`.
7. **Тон** — сухой, фактологический.
