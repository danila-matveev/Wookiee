# Analytics Rules Reference — Analytics Report

## Hard Rules (violation = error)

### 1. GROUP BY model = LOWER()

```sql
GROUP BY LOWER(SPLIT_PART(article, '/', 1))
```

В БД артикулы с разным регистром ("wendy" и "Wendy"). Без LOWER() они попадут в разные группы и исказят суммы на десятки процентов. Все data_layer функции уже применяют LOWER().

### 2. Процентные метрики — ТОЛЬКО средневзвешенные

При объединении каналов (WB+OZON) или агрегации по моделям:

```
ПРАВИЛЬНО: sum(spp_amount) / sum(revenue) * 100
НЕПРАВИЛЬНО: (spp_wb% + spp_ozon%) / 2
НЕПРАВИЛЬНО: spp_wb% (присвоить 0 для OZON)
```

### 3. ДРР — ВСЕГДА с разбивкой

ДРР (доля рекламных расходов) обязательно разбивать:
- **Внутренняя реклама** (МП): WB promotion, OZON performance
- **Внешняя реклама**: блогеры, VK, Yandex

Без разбивки невозможно определить причину роста ДРР.

### 4. Единая маржа

Single margin = включает ВСЮ рекламу. Нет M-1/M-2 в отчетах.

### 5. Формула маржи WB

```
WB margin = SUM(marga) - SUM(nds) - SUM(reclama_vn) - COALESCE(SUM(reclama_vn_vk), 0) - COALESCE(SUM(reclama_vn_creators), 0)
```

Поля `reclama_vn_vk` и `reclama_vn_creators` могут быть NULL — обязательно COALESCE.

### 6. Формула маржи OZON

```
OZON margin = SUM(marga) - SUM(nds)
```

На OZON реклама учтена в marga. Отдельно не вычитать.

### 7. Органика + Реклама = НИКОГДА не суммировать

Органические и рекламные заказы перекрываются. Суммирование дает завышенный итог. Показывать раздельно.

### 8. Числа только из данных

Никогда не генерировать числа. Каждое значение = результат SQL-запроса через data_layer. Если данных нет — писать "н/д", не оценивать.

### 9. Факт vs Гипотеза

Четко разделять:
- **Факт**: "маржинальность снизилась на 3.2 п.п."
- **Гипотеза**: "вероятная причина — рост логистики из-за сезонного пика"

### 10. Рекомендации — только через цену/маркетинг

Рекомендации должны быть actionable:
- Изменить цену на X%
- Перераспределить рекламный бюджет
- Поставить на скидку / снять со скидки

НЕ рекомендовать: "улучшить качество", "оптимизировать процессы".

### 11. Формат чисел

- Целые: `1 234 567 ₽` (пробел-разделитель тысяч)
- Проценты: `24.1%`
- Изменения п.п.: `8.0% → 11.2% (+3.2 п.п.)`
- Тысячи: `36 482 тыс.₽`
- Штуки: `1 234 шт`
- Дни: `45 дн`

### 12. Формат дат

- В тексте: `17 марта 2026`
- В таблицах: `17.03.2026`
- ЗАПРЕЩЕНО: `2026-03-17` (ISO формат), `17/03/2026`

## Лаговые и неуправляемые метрики

### Выкуп % — лаговый (3-21 дней)

Нельзя использовать как причину изменения маржинальности в дневном/недельном отчете. Показывать только как информационный показатель с пометкой "лаг 3-21 дн."

### СПП (скидка постоянного покупателя) — неуправляемый

WB устанавливает СПП самостоятельно. Бренд не может влиять. Показывать динамику, но не рекомендовать "снизить СПП".

### Комиссия маркетплейса — неуправляемый

Фиксированный % по категории. Показывать для информации, не рекомендовать изменений.

## Double KPI: ДРР по продажам + ДРР по заказам

| ДРР продаж | ДРР заказов | Интерпретация |
|---|---|---|
| Низкий | Низкий | Эффективная реклама |
| Высокий | Низкий | Много заказов, но низкий выкуп — проблема с товаром |
| Низкий | Высокий | Мало заказов от рекламы, но хороший выкуп — неэффективная реклама |
| Высокий | Высокий | Неэффективная реклама + проблема с товаром |

## Матрица эффективности рекламы

| Категория | ROMI | Действие |
|---|---|---|
| Growth | >1000% | Масштабировать, увеличить бюджет |
| Harvest | 60-300% | Поддерживать текущий уровень |
| Optimize | 0-100% | Оптимизировать: ключевые слова, ставки, креативы |
| Cut | <0% (убыток) | Отключить или кардинально пересмотреть |

## Пороги оборачиваемости запасов

| Дней остатка | Статус | Действие |
|---|---|---|
| <14 дн | ДЕФИЦИТ | Срочная поставка, повысить цену если маржа позволяет |
| 14-60 дн | НОРМА | Штатный режим |
| 60-90 дн | ВНИМАНИЕ | Снизить закупку, мониторить |
| 90-250 дн | ЗАТОВАРИВАНИЕ | Скидка, акция, перемещение на другие склады |
| >250 дн | МЕРТВЫЙ СТОК | Глубокая распродажа, вывод с площадки |

---

## Ценовая логика

| Условие | Действие |
|---|---|
| ЗАТОВАРИВАНИЕ + маржа >15% | Снизить цену для ускорения оборота |
| ДЕФИЦИТ | Повысить цену (если эластичность позволяет) |
| Маржа <15% | НЕ снижать цену, искать другие рычаги |
| МЕРТВЫЙ СТОК | Глубокая уценка, вплоть до себестоимости |
| Высокий СПП (>30%) | Рассмотреть повышение базовой цены |

---

## Дисциплина уверенности

| Уровень | Критерий | Допустимые действия |
|---|---|---|
| HIGH | Данные за 7+ дней, тренд устойчивый, причина ясна | Рекомендации с конкретными цифрами |
| MEDIUM | Данные за 3-7 дней или причина не однозначна | Гипотезы с оговорками, "необходимо наблюдение" |
| LOW | <3 дней данных, аномалия, внешний фактор | Только наблюдение, "рано делать выводы" |

Запрещено: давать конкретные рекомендации при LOW confidence.

## Годовой ROI

```
ROI = margin% * (365 / turnover_days)
```

| Категория | ROI | Описание |
|---|---|---|
| ROI Leader | >500% | Звезды ассортимента |
| Healthy | 200-500% | Здоровые модели |
| Underperformer | 50-200% | Требуют оптимизации (цена, реклама, остатки) |
| Deadstock Risk | <50% | Кандидаты на вывод |

## Систематичность внешних каналов

| Канал | Тип | Анализ в дневном отчете |
|---|---|---|
| Блогеры | Случайный (random) | НЕ анализировать по дням, только за период |
| VK / Yandex | Систематический | Можно анализировать по дням |
| SMM | Систематический | Можно анализировать по дням |

Блогеры работают импульсно — один пост может дать всплеск на 1-3 дня. Дневной анализ ДРР блогеров бессмыслен.

## Запрещенные фразы

Никогда не использовать в отчетах:

- "Необходимо проанализировать" (ты и есть аналитик — анализируй)
- "Рекомендуется обратить внимание" (конкретизируй — на что и почему)
- "Возможно, это связано с..." без данных (либо подтверди данными, либо маркируй как LOW confidence)
- "Оптимизировать процессы" (слишком абстрактно — что именно оптимизировать)
- "В целом, ситуация стабильная" (если стабильная — покажи метрики стабильности)
- "Существенный рост/падение" без цифры (насколько?)
- "Рекомендуется увеличить/уменьшить" без конкретного % или суммы

## Аналитические принципы

1. **Системное мышление**: каждое изменение влияет на всю цепочку (цена → заказы → выкуп → оборачиваемость → маржа)
2. **Evidence-based**: каждое утверждение подкреплено числом из data_layer
3. **Эффект в рублях**: каждая рекомендация с оценкой эффекта в ₽/мес
4. **Последствия 2-го порядка**: снижение цены → рост заказов, но и рост логистики, снижение маржинальности
5. **Тон**: сухой CFO. Без эмоций, без восклицательных знаков, без "отличный результат!"

## Известные проблемы качества данных

### retention == deduction (дубли)

В WB abc_date поля `retention` и `deduction` часто дублируют друг друга. Не суммировать — использовать только одно.

### Fan-out JOIN

При JOIN abc_date с orders/sales по баркоду без агрегации — строки размножаются. Всегда агрегировать до JOIN или использовать подзапросы.

### Трафик: расхождение ~20% с PowerBI

`content_analysis` дает цифры трафика на ~20% ниже, чем PowerBI. Причина: неизвестные фильтры на стороне PowerBI. Указывать в дисклеймере.

### OZON органика недоступна

OZON не предоставляет данные об органическом трафике через API. Используется оценочная функция `get_ozon_organic_estimated()` — маркировать как "оценка".

### Неполные месяцы

При анализе текущего месяца — данные неполные. Обязательно указывать: "данные за N дней из M".

## Диагностика аномалий воронки

| Симптом | Что проверить |
|---|---|
| Показы упали, остальное стабильно | SEO-позиции, изменение ключевых слов, WB-алгоритм |
| CTR упал при стабильных показах | Фото карточки, позиция в выдаче, новые конкуренты |
| CR корзина упала | Цена vs конкуренты, отзывы, рейтинг, наличие |
| CR заказ упал при стабильной корзине | Доставка (сроки, доступность), проблемы с оплатой |
| Выкуп упал | Качество товара, несоответствие описанию, размерная сетка |
| Заказы растут, выручка нет | Средний чек падает (скидки, СПП, ассортиментный сдвиг) |
| Выручка растет, маржа падает | Рост логистики/хранения, рост рекламы, снижение цены |
| ДРР растет, заказы стабильны | Неэффективная реклама — Cut/Optimize по матрице |
| Остатки растут при стабильных продажах | Избыточная поставка, сезонный спад ожидается |
| Маржинальность модели <15% | Проверить: цену, себестоимость, логистику, хранение, рекламу |
