# Hub → Google Sheets sync (через зеркало)

Слэш-команда для возврата к плану синхронизации каталога Hub → Google Sheets.

## Триггеры

Эту команду пользователь может вызвать:
- `/catalog-sheets-sync`
- «давай начнём план Hub → Sheets»
- «вернёмся к синку каталога»
- «давай начнём реализацию плана по таблице»
- «продолжаем mirror sync»

## Что делать при вызове

1. **Прочитай план:** `wookiee-hub/.planning/hub-to-sheets-sync/PLAN.md`. Не пересказывай его целиком пользователю — он сам его читал. Достаточно: статус (план готов / в процессе фазы N), что осталось согласовать.

2. **Проверь раздел 5 (10 пунктов согласования):**
   - Если в таблице есть строки с ⏳ — **пройдись с пользователем по каждой нерешённой**, спроси «согласовано или меняем?». Меняй колонку на ✅ когда подтверждено, либо переформулируй пункт и план если он сказал «по-другому».
   - Если все ✅ — переходи к шагу 3.

3. **Определи текущую фазу:**
   - Прочитай git log по ветке `feature/hub-to-sheets-sync` или ищи последний созданный файл из секции 10 (Структура файлов) в PLAN.md.
   - Если ничего из раздела 10 не создано → **Фаза 0** не начиналась. Спроси пользователя, готов ли он вручную создать зеркальную таблицу в Google Sheets, и выдай инструкцию из Фазы 0 шаг 1–4.
   - Если есть `.env` с `CATALOG_MIRROR_SHEET_ID`, но нет миграции 030 → переходим к **Фазе 1**.
   - Если миграция 030 есть, но нет `services/sheets_sync/hub_to_sheets/` → **Фаза 2**.
   - Если сервис есть, но нет роута `analytics_api/routes/catalog_sync.py` → **Фаза 3**.
   - Если роут есть, но нет `wookiee-hub/src/components/catalog/sync-mirror-button.tsx` → **Фаза 4**.
   - Если кнопка задеплоена → **Фаза 5 (тестирование)**. Открой `QA-LOG.md`, обсуди что найдено за последние дни.

4. **Работай в feature-ветке:** `feature/hub-to-sheets-sync`. Прямые коммиты в `main` блокирует workflow guard.

5. **Соблюдай правила из PLAN.md:**
   - Раздел 2 — принципы (delta-update, idempotency, anchor, mirror-first).
   - Раздел 8 — что НЕ делаем (двусторонний sync, realtime, авто-добавление колонок).
   - Раздел 7 — известные риски.

6. **На каждой фазе обновляй раздел 6 (Фазы) — отмечай выполненные шаги.** Можно «**~~зачёркнуто~~**» или добавлять «✅ DONE 2026-XX-XX commit `abc1234`».

## Если пользователь говорит «давай начнём с нуля» или «всё переделай»

- Не удаляй PLAN.md. Создай рядом `PLAN-v2.md` и в шапку PLAN.md впиши «**заменён PLAN-v2.md от 2026-XX-XX**».
- Перенеси решения из раздела 5 в новый план, чтобы не переспрашивать заново.

## Если план кажется устаревшим

- Контекст в разделе 1 может протухнуть (Hub меняется быстро). Перед стартом проверь:
  - Cron `sync_sheets_to_supabase` ещё работает? `crontab -l` на сервере `timeweb` или проверь `services/sheets_sync/runner.py` логи в `tool_runs`.
  - W10-цикл точно мерджен? `git log --oneline main | head -5`.
  - Сервис-аккаунт Google для Sheets всё ещё `wookiee-sheets@...`? `cat services/sheets_sync/credentials/sa.json | jq .client_email`.
- Если что-то изменилось — обнови раздел 1 перед стартом.
