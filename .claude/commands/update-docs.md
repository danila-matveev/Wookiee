# Update Project Documentation

Проверь актуальность документации проекта и предложи обновления.

## Шаги

1. Прочитай текущую структуру проекта:
   - `ls agents/` — какие агенты существуют
   - `ls services/` — какие сервисы существуют
   - `ls shared/clients/` — какие API-клиенты есть

2. Прочитай ключевые документы:
   - `README.md` — обзор проекта
   - `AGENTS.md` — описание компонентов
   - `docs/index.md` — навигация
   - `docs/agents/README.md` — реестр агентов
   - `docs/architecture.md` — архитектура

3. Сравни реальную структуру с документацией:
   - Все ли агенты из `agents/` перечислены в `docs/agents/README.md`?
   - Все ли сервисы из `services/` перечислены в `README.md`?
   - Актуальны ли пути в `docs/architecture.md`?
   - Есть ли сломанные ссылки в .md файлах?

4. Проверь `docs/development-history.md` — есть ли запись за последнюю неделю?

5. Покажи список расхождений и предложи конкретные правки.

6. После подтверждения пользователя — внеси изменения.

## Важно

- Не трогай правила в AGENTS.md (секции ниже "Описание проекта")
- Обновляй только описательную часть документации
- Сохраняй стиль и формат существующих документов
