# Vacancy Refinement — детальные инструкции

Цель: взять черновик пользовательской вакансии, найти топ-аналогов на рынке и предложить конкретные улучшения по 5 разрезам (зарплата, обязанности, требования, бенефиты, формат).

## Шаг 1. Получение черновика

Попроси у пользователя:
1. **Текст вакансии** — полный или ключевые секции (заголовок, обязанности, требования, бенефиты, зарплата). Можно ссылку на её активную публикацию на hh.ru — тогда первым шагом распарсить через `hh_get_vacancy` (если есть `vacancy_id` в URL) или попросить вставить текст.
2. **Регион публикации** — для корректного бенчмарка зарплат.

Если черновик неструктурирован — извлеки сам секции (заголовок, обязанности, требования, бенефиты, зарплата, формат) и подтверди разбор пользователю одним сообщением.

## Шаг 2. Идентификация роли

Из черновика извлеки:
- **Роль / job title** — для `text` параметра поиска (например: "продакт-менеджер маркетплейсы", "senior python разработчик").
- **Ключевые навыки** — 3-5 главных, для уточнения релевантности результатов поиска.
- **Уровень опыта** — junior / middle / senior → маппинг на `experience` enum.

## Шаг 3. Поиск аналогов

```
mcp__headhunter__hh_search_vacancies(
  text=<job title + 1-2 ключевых навыка>,
  area=<region_id>,
  experience=<inferred_level>,
  per_page=50,
  page=0
)
```

Если `found < 30` — расширь: убери `experience`, или попробуй вариации текста.

## Шаг 4. Подбор «лучших аналогов»

Из выдачи отбери 10 топ-аналогов по эвристике:
- известный/крупный работодатель (по имени или по `employer.trusted=true`),
- свежий `published_at` (≤ 30 дней),
- указана зарплата (`salary != null`),
- ОЭ-релевантность роли — заголовок реально соответствует целевой роли (не «оператор колл-центра» при поиске «продакт»).

Для каждого из 10 — `hh_get_vacancy(id)` для полного `description` и `key_skills`.

## Шаг 5. Сравнение по 5 разрезам

Для каждого разреза — фактологическое наблюдение + конкретное действие.

### A. Зарплата
- Бенчмарк: p25/p50/p75 `salary.from` и `salary.to` из аналогов.
- Где находится черновик: ниже p25 / в пределах вилки / выше p75.
- Действие: «поднять нижнюю границу до X для попадания в p50» или «оставить, но добавить упоминание квартальных премий».

### B. Обязанности
- Что встречается у ≥ 50% аналогов, но отсутствует у пользователя? Это типичные ожидания рынка.
- Что есть у пользователя, но редко в аналогах? Это либо уникальная ценность (оставить и подсветить), либо лишняя нагрузка (вынести в отдельные роли).

### C. Требования
- Какие технические/софт-навыки повторяются в `key_skills` аналогов?
- Где у пользователя «инфляция требований» — недостижимые комбинации опыта.
- Где недостающие критичные требования.

### D. Бенефиты
- Какие бенефиты упоминаются в `description` аналогов чаще всего? Стандартные паттерны: ДМС, обучение, гибкий график, удалёнка, опционы, премии, релокация, спорт, английский, психолог, конференции.
- Что у пользователя отсутствует из топ-5 — добавить (если реально есть в компании).

### E. Формат / график
- Распределение `schedule` среди аналогов — fullDay / remote / flexible.
- Если у пользователя fullDay, а у 60% аналогов remote/flexible — это серьёзный минус для привлечения кандидатов.

## Шаг 6. Выдача

Структура ответа:

```markdown
# Аудит вакансии: <Заголовок>

## Найдено аналогов: <N> (регион: <…>, уровень: <…>)

## Сравнение

### Зарплата
- Рынок (p25/p50/p75): <…>
- Ваша вилка: <…>
- **Действие:** <…>

### Обязанности
- Что добавить (что есть у большинства аналогов): <list>
- Что убрать/перенести: <list>

### Требования
- Критичные пробелы: <list>
- Завышенные требования: <list>

### Бенефиты
- Топ-5 у аналогов: <list с %>
- У вас отсутствуют: <list>

### Формат работы
- Распределение на рынке: <fullDay X% / remote Y% / flexible Z%>
- **Действие:** <…>

## Перепиcанные секции (готовый текст)

**Зарплата:** …

**Ключевые обязанности:**
- …

**Бенефиты (то, что точно есть в компании):**
- …
```

Сохрани в `data/hh_research/<role-slug>/<YYYY-MM-DD>-vacancy-refine.md`.

В чат — краткое резюме (3-5 буллетов с самыми важными изменениями).
