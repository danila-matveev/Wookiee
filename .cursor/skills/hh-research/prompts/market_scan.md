# Market Scan — детальные инструкции

Цель: собрать срез рынка по должности и выдать структурированный summary с зарплатной вилкой, требованиями, форматами работы и рекомендациями.

## Шаг 1. Уточнение параметров

Через AskUserQuestion собери (по необходимости — несколько вопросов):

1. **Должность / keywords** (свободный ввод) — например: "продакт-менеджер", "senior python developer", "маркетолог wb".
2. **Регион**:
   - Москва (area_id=1)
   - Санкт-Петербург (area_id=2)
   - Вся Россия (area_id=113)
   - Удалёнка (не area-фильтр, а `schedule=remote`)
   - Другой → AskUserQuestion с Other → найти через `hh_get_areas`
3. **Уровень опыта** (опционально, multiSelect):
   - noExperience — без опыта
   - between1And3 — 1-3 года
   - between3And6 — 3-6 лет
   - moreThan6 — более 6 лет
4. **Формат работы** (опционально):
   - fullDay / remote / flexible / shift

## Шаг 2. Поиск вакансий

Вызови `mcp__headhunter__hh_search_vacancies` с собранными параметрами:

```
text=<должность>
area=<area_id>            # если выбран
experience=<value>        # если один уровень; если несколько — делай несколько вызовов
schedule=<value>          # если задан
per_page=100
page=0
```

Если `found > 100` — сделай ещё запрос с `page=1` (макс 200 вакансий для агрегации).

## Шаг 3. Обогащение

Для топ-15 вакансий (отсортируй по `salary.from desc`, NULL salary в конец):
- `hh_get_vacancy(vacancy_id)` — получить полный `description`, `key_skills`, `experience`, `employment`, `schedule`.
- `hh_get_employer(employer_id)` — собрать имя/индустрию работодателя.

Для топ-3 — получить `hh_get_similar(vacancy_id)`, чтобы найти похожие позиции, не попавшие в основной поиск.

## Шаг 4. Агрегация (на стороне LLM)

Посчитай:

1. **Зарплатная вилка** (только по вакансиям с указанной зарплатой, currency=RUR):
   - p25, медиана (p50), p75, max — отдельно для `salary.from` и `salary.to`
   - доля вакансий с указанной зарплатой / всего
   - средневзвешенная середина (по правилу `.claude/rules/data-quality.md`)
2. **Топ-10 ключевых навыков** — частота из `key_skills` всех вакансий.
3. **Распределение по форматам** — fullDay / remote / flexible / shift, в %.
4. **Распределение по опыту** — какой уровень доминирует.
5. **Топ-5 работодателей** — кто публикует больше всех.
6. **География** — топ-3 регионов (если запрос был по 113).
7. **Бенефиты** — частые упоминания в `description` (ДМС, гибкий график, премии, обучение, опционы) — простой keyword-matching по русским паттернам.

Все денежные расчёты — через `Decimal` (см. `.claude/rules/python.md`).

## Шаг 5. Выдача

Сформируй markdown по шаблону `templates/market_summary.md`. Сохрани в `data/hh_research/<role-slug>/<YYYY-MM-DD>-market-scan.md` и выведи в чат краткий summary (5-7 строк): диапазон зарплат, топ-3 навыка, доля удалёнки, топ работодатель, главная рекомендация для пользователя.

## Шаг 6. Контроль качества

Перед выдачей убедись:
- Если в выборке < 20 вакансий — предупреди пользователя что выборка маленькая, цифры могут быть нерепрезентативны.
- Если зарплату указали < 30% вакансий — отдельно отметь это в отчёте.
- Если currency != RUR попадается часто — сообщи и исключи из вилки или сконвертируй.
