## [2026-02-09] ADR-001: Переход на структуру 2030ai/project_template

### Статус
Принято

### Контекст
Проект нуждался в стандартизированной AI-agent-friendly документации и структуре для публикации на GitHub.

### Решение
Принять AGENTS.md как единый источник правил, создать docs/ иерархию, применить трёхуровневую защиту секретов.

### Обоснование
- Лучшая поддержка multi-agent разработки (Claude, Cursor, Codex)
- Стандартная структура упрощает onboarding
- Documentation-first workflow

### Последствия
- (+) Единые правила для всех AI-агентов
- (+) Готовность к GitHub
- (+) Секреты вынесены из трекаемых файлов
- (-) 17 новых файлов документации

## [2026-02-09] ADR-002: database_docs/ остаётся на корневом уровне

### Статус
Superseded — при реструктуризации `database_docs/` перемещён в `docs/database/`

### Контекст
При переходе на шаблон возник вопрос — перемещать ли `database_docs/` в `docs/`.

### Решение
Оставить `docs/database/` в составе общей документации, добавить ссылки из `docs/index.md`.

### Обоснование
- Это доменная документация, а не agent docs
- На `docs/database/` есть прямые ссылки из AGENTS.md и скриптов
- Перемещение сломало бы существующие ссылки

## [2026-02-09] ADR-003: scripts/config.py переведён на python-dotenv

### Статус
Принято

### Контекст
scripts/config.py содержал хардкоженные пароли БД и Notion-токены в трекаемом файле.

### Решение
Перевести config.py на os.getenv() + python-dotenv с загрузкой из корневого .env.

### Обоснование
- Секреты не попадут в git-историю при публикации на GitHub
- Интерфейс (имена переменных) не меняется — все импортёры продолжают работать
- python-dotenv уже есть в зависимостях бота
