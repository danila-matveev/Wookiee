## 5 погрешностей (из PDF) — правила перерасчёта

### Погрешность №1: Фильтр обратной логистики

**Правило:** На листе "Расчет логистики" должны быть только строки прямой логистики:
- "К клиенту при продаже"
- "К клиенту при отмене"

Все остальные виды (зелёные и красные в PDF) — исключить. Поле в WB API: `bonus_type_name` (колонка 56 в "Детализация").

### Погрешность №2: ИЛ (Индекс Локализации) из кабинета WB

**Правило:** ИЛ должен соответствовать значениям из дашборда WB Partners → Поставки и заказы → Тарифы → Тарифы складов → Индекс локализации → История изменения индексов.

### Погрешность №3: Коэффициент склада

**Правило (из PDF):**

1. Если "Фиксированный коэффициент склада" > 0 **И** "Дата конца действия фиксации" > "Дата заказа покупателем" → использовать "Фиксированный коэффициент склада" (`dlv_prc` в WB API)
2. Если "Фиксированный коэффициент склада" > 0 **И** "Дата конца действия фиксации" ≤ "Дата заказа покупателем" → использовать Коэффициент из раздела "Тарифы" → файл `Тарифы на логискику.xlsx` (по дате заказа + название склада)
3. Fallback: если ни фиксация, ни файл не дали результат → `dlv_prc` из отчёта

**Формат файла `Тарифы на логискику.xlsx`:**
- Лист "Тарифы короб": [Дата, Склад, Коэф.логистики %, Коэф.хранения %]
- 39,296 строк: 768 дат × ~51 склад
- Коэффициент в процентах (Коледино = 195% → coef = 1.95)

### Погрешность №4: Базовые тарифы по периодам + sub-liter

**Формула расчёта базового тарифа на первый литр:**

**Если дата заказа > дата конца фиксации** (фиксация неактивна) — период определяется по дате заказа:

| Период (дата заказа) | 1-й литр (руб) | Доп. литр (руб) |
|---|---|---|
| 14.08.2024 — 10.12.2024 | 33 | 8 |
| 11.12.2024 — 27.02.2025 | 35 | 8.5 |
| 28.02.2025 — 21.09.2025 | 38 | 9.5 |
| ≥ 22.09.2025 | 46 | 14 |

**Если дата заказа < дата конца фиксации** (фиксация активна) — период определяется по дате НАЧАЛА фиксации (те же периоды, но смотрим на fix_tariff_date_from).

**Sub-liter тарифы** (если дата заказа ≥ 22.09.2025 **И** объём < 1 литра):

| Объём (Л) | 1-й литр (руб) | Доп. литр (руб) |
|---|---|---|
| 0.001 — 0.200 | 23 | 0 |
| 0.201 — 0.400 | 26 | 0 |
| 0.401 — 0.600 | 29 | 0 |
| 0.601 — 0.800 | 30 | 0 |
| 0.801 — 1.000 | 32 | 0 |

**Наш расчёт использовал хардкод: base=46, extra=14 для всех строк.**

### Погрешность №5: Отрицательные разницы

**Правило:** Строки с отрицательной разницей (WB удержал меньше чем должен был) — полностью исключить из листа "Переплата по логистике". В лист должны попасть ТОЛЬКО строки с положительной переплатой.

## Формула расчёта (для каждой строки)

```
calculated_cost = base_cost × coef × IL
overpayment = delivery_rub − calculated_cost

где:
  base_cost = base_1l            (если volume ≤ 1)
  base_cost = base_1l + (volume − 1) × extra_l  (если volume > 1)
  
  base_1l, extra_l = из таблицы периодов (Погрешность №4)
  coef = из 3-tier логики (Погрешность №3)
  IL = из дашборда WB (Погрешность №2)
```

## Что нужно создать

### 1. Скрипт перерасчёта `services/logistics_audit/recalculate_ooo.py`

**Вход:** Excel-файлы (аудит ООО + тарифы + опционально Фисанов)
**Выход:** Новый Excel + декомпозиция по фиксам в консоль

**Логика:**
1. Загрузить сырые данные из "Детализация" (фильтр: `supplier_oper_name == "Логистика"`)
2. Загрузить тарифы из `Тарифы на логискику.xlsx` → `dict[(date, warehouse_name), coef_pct]`
3. Загрузить объёмы из "Габариты в карточке" (nm_id → volume)
4. Рассчитать переплату 6 раз:
   - **Baseline:** old logic (tariff=46/14, ktr из Excel, все bonus_type, с негативами)
   - **+Fix1:** filter forward delivery only
   - **+Fix1+2:** period tariffs
   - **+Fix1+2+3:** warehouse coef from tariff file
   - **+Fix1+2+3+4:** IL from WB dashboard
   - **+Fix1+2+3+4+5:** exclude negatives
5. Декомпозиция: delta между каждым шагом = вклад фикса в рублях
6. Target: сумма дельт ≈ 94,062 руб

**Выходной Excel (4 листа):**
- "Переплата (исправленный)" — строки с overpayment > 0
- "Сравнение" — старый vs новый расчёт построчно
- "Декомпозиция" — таблица: фикс → вклад (руб) → строк
- "ИЛ" — значения ИЛ по неделям

**CLI:**
```bash
# Перерасчёт ООО
python3 -m services.logistics_audit.recalculate_ooo

# Проверка на Фисанове
```
