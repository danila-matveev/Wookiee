# Василий — Agent Specification

## Статус
- **Версия**: 3.0
- **Готовность**: Production (80%)
- **Последнее обновление**: 2026-02-14

## Миссия
Оптимизация распределения товаров между складами Wildberries. Расчёт индекса локализации, рекомендации по перемещениям и дозаказам для снижения логистических издержек.

## Архитектура

### Тип агента
**Детерминированный batch-скрипт** (БЕЗ LLM, без agent loop)

### Модели
- **LLM**: нет (полностью детерминированный алгоритм)
- **Средняя стоимость запроса**: $0 (нет LLM)

### Компоненты
- **Интерфейс**: CLI (`python -m agents.vasily`) + API (VasilyService)
- **Хранилище**:
  - SQLite (`agents/vasily/data/vasily.db`) — история расчётов
  - Excel (`agents/vasily/data/reports/`) — отчёты по каждому расчёту
  - Google Sheets (ID: `1TMadxTXPYnuGTvnMZT9Ny7eL4KbvsjK2XsuRjpRmnM0`) — публичная версия отчёта
- **Уведомления**: Bitrix24 чат `chat415` ("Wookiee Склад МСК")
- **Цикл**: Линейный pipeline (не loop)

### Pipeline
```
CronTrigger (понедельник 08:00 МСК)
    ↓
WB API (остатки + заказы)
    ↓
Трансформация → df_stocks + df_regions
    ↓
Анализ (run_analysis):
  - Индекс локализации по SKU
  - Перемещения между складами (с защитой доноров)
  - Дозаказы с собственного склада (МойСклад)
    ↓
Excel-отчёт + JSON-результат + SQLite-история
    ↓
Google Sheets экспорт (11 листов)
    ↓
Bitrix24 уведомление
```

## Индекс локализации

**Формула:**
```
Индекс локализации = Локальные заказы / Всего заказов × 100%
```

**Целевой показатель:** ≥ 75% по каждому SKU и региону

**Локальный заказ** — заказ, выполненный со склада в том же регионе, где находится покупатель.

### Приоритеты

| Приоритет | Условие | Категория | Действие |
|-----------|---------|-----------|----------|
| 🔴 Критично+ | Индекс <50% + заказы ≥10 | 1_Критичные | Делать сразу |
| 🔴 Критично | Индекс <50% | 1_Критичные | Высокий приоритет |
| 🟠 Важно+ | Индекс <75% + заказы ≥10 | 2_Важные | Средний приоритет |
| 🟠 Важно | Индекс <75% | 2_Важные | Обычный приоритет |
| 🟢 OK | Индекс ≥75% | 3_Остальные | Поддерживать |

### Два кабинета

## Типы действий

### 1. Перемещения (moves)
Перенос товара с одного склада WB на другой.
- **Донор** — склад, откуда забираем (должен иметь высокий индекс, ≥ `min_donor_localization`)
- **Получатель** — склад, куда перемещаем (низкий индекс, высокий спрос)
- **Защита доноров**: не забирать так много, чтобы индекс донора упал ниже порога

**Алгоритм защиты:**
1. Рассчитать безопасный минимум донора = спрос на `safety_days` дней
2. Доступный излишек = остаток - безопасный минимум
3. Если доступного излишка не хватает → дефицит уходит в допоставки

### 2. Допоставки (supplies)
Отправка товара с собственного склада (МойСклад) на склад WB.
- Используются, когда на складах WB не хватает товара для перемещений
- Приоритет: склады с максимальным спросом и минимальным индексом
- Ограничения:
  - Не поставлять товар с оборачиваемостью > `max_turnover_days`
  - Не поставлять товар со статусом «стоп» или «распродажа»
  - Ограничение по факту: `min(нужно, макс_по_обороту, на_своём_складе)`

## Запуск

### Автоматический (планировщик)
```bash
python -m agents.vasily
```
Первый расчёт — сразу. Далее — по расписанию (понедельник 08:00 МСК).

### Ручной (однократный)
```python
import asyncio
from agents.vasily.scheduler import VasilyScheduler
scheduler = VasilyScheduler()
asyncio.run(scheduler.run_once())
```

### CLI (прямой вызов скрипта)
```bash
# API-режим (автоматическое получение данных)
python agents/vasily/run_localization.py --cabinet ip --days 30

# Локальный режим (из Excel-файлов)
python agents/vasily/generate_localization_report_v3.py "Data 08.02" --compare-with "Data 04.02"
```
