# AI-агенты Wookiee

## Что такое "агент"

Агент — это **не бот**. Бот (Telegram, WhatsApp) — это лишь интерфейс для взаимодействия с агентом. Агент — это автономная система с четырьмя компонентами:
- **Мозг** (LLM) — языковая модель для рассуждений и принятия решений
- **Playbook** — правила, контекст и бизнес-логика
- **Tools** — функции для работы с данными и внешними системами
- **Memory** — контекст сессии, история, долгосрочные знания

Каждый агент решает конкретную бизнес-задачу и может работать автономно или через интерфейс.

### Статусы

- **Активен** — работает в production, используется командой
- **В разработке** — ручные скрипты работают, автоматизация в процессе
- **Планируется** — инфраструктура подготовлена, бизнес-логика не реализована

## docs/agents/ vs docs/ (корневой уровень)

| | `docs/agents/` (эта папка) | `docs/` (руководства, ADR, шаблоны) |
|---|---|---|
| **Аудитория** | Пользователи, разработчики, стейкхолдеры | AI-агенты разработки (Claude, Cursor) |
| **Содержание** | Бизнес-описания + техническая документация модулей | Руководства по разработке, ADR, DoD, шаблоны |
| **Вопрос** | "Что делает этот модуль и как его использовать?" | "Как правильно писать код в этом проекте?" |

## Шаблон описания агента

При добавлении нового агента используйте следующую структуру:

```markdown
# [Название агента]

## Бизнес-описание
- Назначение
- Статус (Активен / В разработке / Планируется)
- Какие задачи решает
- Кто использует

## Технические детали
- Стек технологий
- Архитектура (диаграмма потока)
- Ключевые файлы
- Конфигурация (.env переменные)
- Зависимости от других компонентов

## Запуск и использование
- Как запустить
- Примеры команд
- Ожидаемый результат

## Ссылки
- Исходный код
- Связанная документация
```
