## 3. Как WB считает стоимость логистики

WB определяет стоимость доставки каждого товара по формуле из оферты:

```
Стоимость = Базовый тариф × Коэффициент склада × Индекс Локализации (ИЛ)
```

### Компоненты формулы

| Компонент | Что это | Откуда берётся |
|---|---|---|
| **Базовый тариф** | Стоимость доставки за 1 литр объёма | Тарифы WB (API), калиброван обратным расчётом: **33.60 ₽** |
| **Доп. литры** | Для товаров >1L: `(объём − 1) × 14 ₽` за каждый дополнительный литр | API тарифов WB |
| **Коэффициент склада** | Множитель, зависящий от склада отправки (поле `dlv_prc` в отчёте) | Отчёт реализации |
| **ИЛ (Индекс Локализации)** | Множитель, зависящий от % локализации продавца. Пересчитывается каждую неделю | Рассчитан из заказов |

### Для товаров ≤ 1 литра

```
Стоимость = 33.60 ₽ × коэф_склада × ИЛ
```

### Для товаров > 1 литра

```
Стоимость = (33.60 + (объём − 1) × 14) × коэф_склада × ИЛ
```

### Переплата

```
Переплата = Списано WB (delivery_rub) − Расчётная стоимость
```

## 4. Как рассчитали Индекс Локализации (ИЛ)

**ИЛ** — это множитель, который WB применяет к стоимости логистики в зависимости от того, насколько «локально» продавец отправляет заказы. Чем больше заказов отправляется со складов в том же регионе, куда едет товар — тем ниже ИЛ и дешевле логистика.

WB пересчитывает ИЛ **каждую неделю**.

### Шаги расчёта

1. **Загрузка заказов.** Из WB API загружены все заказы за период: **68 130 заказов**
2. **Классификация заказов.** Для каждого заказа определяется:
   - Федеральный округ (ФО) склада отправки
   - Федеральный округ доставки покупателю
   - Если ФО совпадают — заказ **локальный**, не совпадают — **не локальный**
3. **Группировка по неделям.** Заказы группируются по ISO-неделям (пн–вс)
4. **Расчёт % локализации для каждой недели:**
   ```
   Локализация = Локальные заказы / Все заказы × 100%
   ```
5. **Маппинг на коэффициент ИЛ** по таблице WB (см. ниже)

## 5. Калибровка базового тарифа

### Проблема

API WB (`/api/v1/tariffs/box`) возвращает **текущий** тариф: 46 ₽ за первый литр. Но обратный расчёт по фактическим данным отчёта показывает, что реальная база ниже.

### Метод

Для всех строк логистики с объёмом ≤ 1 литра мы делаем обратный расчёт:

```
base × ИЛ = delivery_rub / dlv_prc
```

Берём **медиану** по всем таким строкам → получаем `base × ИЛ = 33.60 ₽`.
Делим на медианный ИЛ (1.00) → **base = 33.60 ₽**.

### Итог

| Объём товара | Базовый тариф | Как получен |
|---|---|---|
| ≤ 1L | **33.60 ₽** | Медиана обратного расчёта по факту |
| > 1L | 46 ₽ + 14 ₽ за доп. литр | API тарифов WB |

## 11. Важные замечания

1. **Отрицательная переплата** (когда расчёт дороже факта) означает, что WB списал меньше. Это нормально в пределах 1–3%.

2. **Калибровка базового тарифа** — ключевой момент. API показывает 46 ₽, но медиана обратного расчёта для мелких товаров = 33.60 ₽. Мы используем фактическую базу.

3. **ИЛ = 1.00** — минимальный множитель «без штрафа». Если локализация упадёт ниже 65%, ИЛ начнёт расти и логистика подорожает.

4. **Фиксированные ставки** (50 ₽ за возвраты и отмены от клиента) исключены из расчёта переплаты — их нельзя проверить по формуле.

5. **КТР из Личного Кабинета ≠ реальный ИЛ.** КТР в ЛК (1.04) — это прогнозное значение WB. Реальный ИЛ рассчитывается по фактическим заказам и у нас равен 1.00.

6. **С 23.03.2026 формула меняется.** WB вводит ИРП (Индекс Распределения Продаж) — дополнительный % от цены товара. Данный аудит покрывает только период **до** смены формулы.
