### ✅ C1. Табличный вид интеграций

Сортировка по дате и стоимости. Фильтры те же что в C2.

### ✅ C2. Фильтры на странице Интеграций
Нет фильтров совсем. Нужны:
- По блогеру (поиск по handle)
- По маркетологу (выпадающий список)

API уже поддерживает все эти фильтры (`IntegrationListParams`). Нужно только UI.

(например Instagram + Telegram одновременно).
**Требует изменения схемы БД**: поле `channel` → массив `channels text[]`.
Миграция + апдейт ETL + апдейт форм.

## Блок E — Данные и ETL

### ✅ E1. ETL не перезаписывает вручную выставленные стадии
Сейчас ETL при каждом запуске перезаписывает `stage` значением из Google Sheets.
Если менеджер вручную перетащил карточку в канбане — при следующем ETL стадия сбросится.
**Решение**: в upsert SET-клаузе убрать `stage` из обновляемых полей.
Стадия обновляется только при создании (INSERT), но не при UPDATE.
Файл: `services/sheets_etl/run.py` — логика upsert.

### ✅ E2. Календарь показывает все интеграции
После загрузки 689 интеграций календарь по-прежнему показывает только несколько.
Скорее всего — CalendarPage запрашивает интеграции с другим лимитом или фильтром даты.
Нужно проверить `src/routes/calendar/CalendarPage.tsx` — как формируется запрос.
