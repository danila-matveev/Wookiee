### ✅ `logistics-audit` — Расчёт переплат за логистику WB

Сравнивает фактические списания WB с расчётными по формуле Оферты. Pipeline: загрузка отчётов WB → загрузка тарифов → расчёт по габаритам/весу каждого артикула → генерация Excel (11 листов: детализация, сводка по артикулам/моделям, обратная логистика, штрафы, сверка с финансистом). Входные данные: кабинет (ООО/ИП), период.

**Как работает:** Pipeline:
1. Загрузка отчётов WB (reportDetailByPeriod API) за указанный период
2. Загрузка тарифов WB (актуальная тарифная сетка по габаритам/весу)
3. Загрузка данных о габаритах и весе каждого артикула
4. Расчёт «правильной» стоимости логистики по формуле Оферты WB
5. Сравнение с фактическими списаниями → определение переплаты/недоплаты
6. Генерация Excel (11 листов):
   - Детализация по каждой операции
   - Сводка по артикулам
   - Сводка по моделям
   - Калибровка ИЛ
   - Разбивка по тарифам
   - Обратная логистика
   - Штрафы
   - Сверка с данными финансиста
   
Входные данные: кабинет (OOO/IP), дата начала, дата конца
Выходные данные: Excel файл + JSON с суммарной переплатой

Формула: тариф_базовый × коэф_ИЛ × коэф_КТР × (вес_или_объём) - фактическое_списание = переплата

### ✅ `content-kb` — Индексатор фотобазы бренда (YaDisk → pgvector)

Обходит папки на Yandex.Disk (фото товаров, лукбуки, каталожные), генерирует Gemini multimodal embeddings (3072 dim), сохраняет в pgvector (Supabase). ~10K изображений. Используется скиллом /content-search для semantic search по текстовому запросу.

**Как работает:** Pipeline:
1. Обходит папки на Yandex.Disk (фото товаров, лукбуки, каталожные)
2. Скачивает новые/изменённые изображения
3. Генерирует Gemini multimodal embeddings (3072 dim) для каждого фото
4. Сохраняет в pgvector (Supabase) с метаданными (модель, коллекция, тип фото)
5. Предоставляет API поиска (vector similarity search)

Используется скиллом /content-search для поиска фото по текстовому запросу.
MCP-сервер для прямой интеграции с Claude.
~10K изображений проиндексировано.
