# /calendar — создание событий в календарях

**Запуск:** `/calendar` (или фразы: «поставь встречу», «созвон с», «запиши в календарь», «событие на», «schedule meeting»).

## Что делает

Создаёт события в нужном календаре по запросу пользователя. Поддерживает 4 источника:

- **Google Calendar** — через `gws` CLI.
- **Bitrix24 Calendar** — через REST API.
- **Yandex Calendar** — через CalDAV (`YANDEX_OAUTH_TOKEN`).
- **Apple Calendar** — через `osascript`.

Двухфазный подход:
1. **Фаза A (Sonnet):** парсит запрос, выбирает календарь, проверяет конфликты через API, готовит превью-карточку (с двумя таймзонами) и финальный API-вызов.
2. **Фаза B (Opus):** показывает превью + конфликты, принимает подтверждение/правки, выполняет API-вызов.

Скилл знает справочник команды (для приглашения участников) и Calendar IDs.

## Зависимости

- `gws` CLI (Google Workspace) — для Google Calendar.
- `.env`:
  - `Bitrix_rest_api` — для Bitrix24 Calendar.
  - `YANDEX_OAUTH_TOKEN` — для Yandex Calendar (срок действия 365 дней; обновляется через OAuth-flow).
- `osascript` (macOS) — для Apple Calendar.

## Связанный скилл

Только создание/редактирование событий. Просмотр расписания (мульти-источник) — отдельный скилл `/schedule` (пока только локально).
