# /bitrix-analytics — Пульс команды WOOKIEE

**Запуск:** `/bitrix-analytics` (или фразы: «отчёт по Bitrix», «пульс команды», «что происходит в компании», «аналитика битрикс»).

## Что делает

Еженедельный аналитический отчёт по активности команды в Битрикс24. Собирает 6 источников за выбранный период (по умолчанию 7 дней):

1. Задачи Битрикса
2. Групповые чаты (включая обсуждения внутри задач)
3. Календарь каждого штатного сотрудника
4. Личные диалоги Данилы с каждым штатным сотрудником
5. Notion — планёрка прошлой недели (per-person отчёты, транскрипт, фоллоуапы)
6. Notion — планёрка текущей недели (черновик на понедельник)

На выходе — публикация в Notion с callout'ами, цветовыми акцентами и таблицами по направлениям/блокерам/перформансу.

## Зависимости

- `modules/bitrix-analytics/fetch_data.py` — сбор сырых данных, кладёт JSON в `/tmp/bitrix_report_data.json`.
- `modules/bitrix-analytics/config.py` — справочник `STAFF` (ID → ФИО + департамент), `CEO_ID`, `EXCLUDED_IDS`, ключевые групповые чаты.
- `shared.tool_logger.ToolLogger` — телеметрия.
- `.env`: `Bitrix_rest_api`, Notion-токен.

## Запуск

```bash
cd modules/bitrix-analytics && python3 fetch_data.py --days 7
```

Затем основной флоу скилла читает JSON и публикует отчёт в Notion. Локальная копия — в `modules/bitrix-analytics/reports/YYYY-MM-DD.md` (папка под git трекается через `.gitkeep`, сами отчёты не коммитятся).

## Замечание для команды

Справочник `STAFF` в `config.py` хардкодит текущий состав команды. При найме/увольнении — обновить руками.
