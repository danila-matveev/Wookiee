# /finolog — управление ДДС в Финологе

**Запуск:** `/finolog` (или фразы: «запиши расход», «расход на», «перевод между счетами», «перевод с … на …», «сводка по счетам», «баланс», «покажи остатки», «кассовый разрыв», «прогноз по деньгам», «ДДС», «сколько на счетах», «движение денег»).

## Что делает

Работа с движением денежных средств через Finolog API. Четыре режима:

1. **Расход** — записать операцию (с подтверждением и превью).
2. **Перевод** — между счетами компании (с подтверждением и превью).
3. **Сводка** — балансы по всем счетам.
4. **Прогноз** — кассовый разрыв вперёд по времени.

Делегирование Sonnet:
- **Сводка/Прогноз** (только чтение) — полностью на Sonnet, результат показывается пользователю как есть.
- **Расход/Перевод** (запись) — двухфазный: Sonnet парсит и собирает превью + curl-команду, Opus подтверждает и выполняет.

## Связанный скилл

`/finolog-dds-report` — это другой скилл (еженедельный/месячный аналитический отчёт по ДДС). `/finolog` — про оперативную запись и сводку.

## Зависимости

- `shared.tool_logger.ToolLogger` — телеметрия запусков (Supabase).
- Finolog API token в `.env`.

## Запуск из корня репо

Tool Logging-блок использует `PYTHONPATH=.` — скилл предполагает запуск из корня `Projects/Wookiee/`.
