   Известные расхождения (учтены в PLAN.md):
   - Telegram routes — могут быть в `routes/telegram.py` + `handlers/*.py`
   - Cookie-чек скрипт — рабочее имя `scripts/telemost_check_cookies.py`, точное имя имплементер найдёт через `crontab -l` или `grep`
   - Последняя существующая миграция: `005_meetings_calendar_uniq.sql`. Новая для T7 → `006_voice_trigger_candidates.sql`.

## Как использовать

1. Открой новую сессию Claude Code в корне репо.
2. Скопируй блок выше (между ``` и ```).
3. Вставь как первое сообщение.
4. Дальше отвечаешь только на вопросы оркестратора (например: «начни с T1», «approved», «откатывай»).
5. После каждой завершённой задачи получишь отчёт.
6. После всех 7 — финальный сводный runbook.

## Контроль с твоей стороны

- Между задачами читай PR description перед merge.
- Если в отчёте есть `BLOCKED` или вопросы — отвечай.
- env-флаги выставляешь сам по графику из PLAN.md «Что делаем после merge всех 7 задач».
