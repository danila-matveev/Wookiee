# Funnel Report v2: добавление OZON — Design Spec

**Дата:** 2026-04-13
**Статус:** Утверждён

---

## 1. Цель

Добавить секцию OZON в существующий `/funnel-report`, чтобы отчёт покрывал оба канала (WB + OZON) в одном документе.

## 2. Доступные данные OZON

### Что есть

| Источник | Таблица | Метрики |
|---|---|---|
| Реклама агрегат | `adv_stats_daily` | показы, клики, заказы, расход, CTR, CPC (по кампаниям, по дням) |
| Реклама по SKU | `ozon_adv_api` | клики, **to_cart** (корзина!), заказы, расход, CTR, CPC (по SKU) |
| Финансы | `abc_date` | заказы, выручка, маржа, комиссии, логистика (по артикулам) |
| Органика (расч.) | `get_ozon_organic_estimated()` | organic_orders = total_orders - ad_orders (по моделям) |
| Ad ROI по моделям | `get_ozon_model_ad_roi()` | расход, заказы, выручка, маржа, ДРР, ROMI (по моделям) |
| Склейки OZON | `skleyki_ozon` (Supabase) | группировка артикулов в карточки |

### Чего нет

- Органические переходы (показы карточки) — `search_stat.hits_view = 0`
- Органические корзины — `search_stat.hits_tocart = 0`
- Органические заказы напрямую — только расчётные (total - paid)
- Buyouts (выкупы) — в abc_date OZON нет отдельного поля

### Ключевое отличие от WB

WB имеет `content_analysis` с полной органической воронкой (переходы→корзина→заказы→выкупы).
OZON — только рекламная воронка (показы→клики→корзина→заказы) + расчётная органика по заказам.

## 3. Архитектура

### Что меняется

1. **Data layer** — добавить `get_ozon_content_by_model()` для финансов по моделям (аналог WB)
2. **Collector** — добавить `ozon_content_by_model` блок в `collect_traffic()`
3. **SKILL.md** — добавить Stage 3b (OZON model analyst)
4. **Промпты** — обновить detector, model-analyst, synthesizer для OZON-секций

### Что НЕ меняется

- Общая архитектура (3 waves + model analyst + verifier + synthesizer)
- WB-секции остаются без изменений
- Формат таблиц (pipe-tables)
- Notion публикация

### Секция: OZON per-model toggle

Для каждой модели с заказами > 0 на OZON:

```
## ▶ OZON: {Model} — заказы {+/-n}%
  ### Экономика
  | Метрика | Значение |
  |---|---|
  | Заказы | {n} (пред. {n}, {+/-n}%) |
  | Выручка | {n} ₽ (пред. {n} ₽) |
  | Маржа | {n} ₽ ({n}%) |
  | Рекл. расход | {n} ₽ |
  | ДРР | {n}% (пред. {n}%) |
  | ROMI | {n}% |
  | Органич. заказы (расч.) | {n} ({n}%) |
  ### Анализ
  {Тренд: рост/падение заказов, изменение маржинальности, ДРР}
```

Примечание: органическая воронка (переходы→корзина) НЕ доступна для OZON. Показываем только заказы + реклама + финансы.

## 6. Промпт изменения

### detector.md

Добавить секцию "OZON":
- Сканировать ozon_finance_by_model: заказы Δ%, маржа Δ%, маржинальность Δ п.п.
- Сканировать ozon_ad_funnel_total: CTR, CPO, CR каждого шага
- Порог: Δ заказов > 15%, Δ ДРР > 2 п.п.

### model-analyst.md

Добавить OZON-шаблон (упрощённый — без органической воронки):
- Toggle: `## ▶ OZON: {Model} — заказы {+/-n}%`
- Экономика: заказы, выручка, маржа, ДРР, ROMI, доля органики
- Анализ: 1-2 предложения о тренде

### synthesizer.md

Добавить 3 секции между WB-моделями и Выводами:
- OZON: Обзор канала
- OZON: Рекламная воронка
- OZON: per-model toggles

### verifier.md

Добавить check #11: OZON данные корректны (заказы, CTR, ДРР совпадают с raw data)

## 7. Ограничения

1. **Органическая воронка OZON недоступна** — search_stat пуст. Показываем только расчётную органику (total - paid).
2. **Нет выкупов (buyouts)** — abc_date OZON не имеет buyout-поля. Не показываем CRP.
3. **Нет per-article воронки** — только per-model агрегат.
4. **Halo-эффект OZON** — skleyki_ozon есть в Supabase, но пока не подключаем (можно добавить позже).
