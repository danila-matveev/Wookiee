# Analytics Report Skill — Design Spec

**Дата:** 2026-04-07
**Статус:** Утверждён для реализации
**Источник:** `docs/superpowers/specs/2026-04-07-analytics-report-full-spec.md`

---

## 1. Обзор

Скилл `/analytics-report` — единая команда из 12 субагентов (8 аналитиков + 3 верификатора + 1 синтезайзер), которая анализирует ВСЕ аспекты бизнеса Wookiee за любой период.

**Вызов:**
```
/analytics-report 2026-04-05                    → дневной (vs вчера)
/analytics-report 2026-03-30 2026-04-05          → недельный
/analytics-report 2026-03-01 2026-03-31          → месячный
```

**Замена:** Oleg v2 (Python-native агент) → Claude Code skill (SKILL.md + prompt templates)

---

## 2. Иерархия анализа

Каждый аналитический блок анализируется на ВСЕХ уровнях иерархии:

```
Бренд (WB + OZON суммарно)
├── Канал: WB / OZON
│   ├── Юрлицо (если применимо)
│   │   ├── Модель (LOWER-нормализованная)
│   │   │   ├── Статус: Продается / Выводим / Архив / Новинки
│   │   │   └── Артикул/цвет/размер (глубина зависит от периода)
```

**Правило:** каждый аналитик обязан предоставить данные минимум на 3 уровнях: Бренд → Канал → Модель. Артикульный уровень — по адаптивной глубине (день: топ/анти-топ, неделя: средне, месяц: все).

```

         │

         │  каждый получает: свой data_slice + summary остальных блоков
         │
         ├─ Wave 2: 3 верификатора параллельно (Agent tool × 3)

         ▼
   Stage 3: Synthesizer (Agent tool × 1)

         │

         ├─ Notion → database 30158a2b-...-000b83c6b747

```

## 5. Субагенты-аналитики (Wave 1)

Все 8 запускаются параллельно. Каждый получает:
- `{{DATA_SLICE}}` — свой блок данных из JSON
- `{{SUMMARY}}` — краткий summary остальных блоков (ключевые цифры для cross-reference)
- `{{DEPTH}}` — day/week/month (для адаптивной глубины)
- `{{QUALITY_FLAGS}}` — известные ограничения данных
- `{{PERIOD_LABEL}}` — "30 марта — 05 апреля 2026" (для отображения)
- `{{PREV_PERIOD_LABEL}}` — "23-29 марта 2026"

### Субагент 1: Финансовый аналитик (financial.md)

**Вход:** `finance` блок + summary advertising, inventory
**Иерархия:** Бренд → Канал → Модель → (Артикул для week/month)
**Выход:**
- Полная финансовая воронка (15 строк): Выручка до/после СПП → Комиссия → Логистика → Хранение → Себестоимость → Реклама внутр/внешн → НДС → Штрафы → Удержания → Маржа
- ВСЕ метрики: ₽ + доля от выручки (%) + Δ vs пред. период + Δ п.п.
- Сведение ΔМаржи: какая статья как повлияла на маржу
- Группировка по статусам: Продается / Выводим / Архив / Новинки
- Драйверы / Антидрайверы маржи (модели)

**Правила:**
- WB маржа = `SUM(marga) - SUM(nds) - SUM(reclama_vn) - COALESCE(SUM(reclama_vn_vk), 0) - COALESCE(SUM(reclama_vn_creators), 0)`
- OZON маржа = `SUM(marga) - SUM(nds)`
- Единая маржа — никаких M-1/M-2
- GROUP BY всегда LOWER()

### Субагент 2: Аналитик внутренней рекламы (internal-ads.md)

**Вход:** `advertising` блок + summary finance (выручка по моделям)
**Иерархия:** Бренд → Канал → Модель → Кампания
**Выход:**
- ДРР с разбивкой: внутренняя + внешняя ОТДЕЛЬНО
- Двойной KPI: ДРР продаж + ДРР заказов (CPO)
- ROMI, CPC, CTR, CPO, CPM по моделям
- Матрица эффективности: Growth / Harvest / Optimize / Cut
- Чёрные дыры (ROMI < 100%)
- Связка реклама → заказы
- Дневная динамика
- Бюджет vs факт

**Бенчмарки:** CTR >5% отлично, CPC <5₽ отлично, CPM <100₽ отлично, CR >5% отлично

### Субагент 3: Аналитик внешнего маркетинга (external-marketing.md)

**Вход:** `external_marketing` блок (из Sheets)
**Иерархия:** Канал (Блогеры / ВК / Яндекс / SMM) → Размещение
**Выход:**
- Блогеры: расход, трафик, ROI (с учётом лага 3-7 дн)
- ВК/Яндекс: ежедневная динамика
- SMM: охваты, вовлечённость
- Эффективность каждого канала

**Запрещённые формулировки:**
- "Внешняя реклама прекратилась" → "Сегодня без внешней рекламы"
- "Экономия на внешней рекламе" → нельзя считать отсутствие несистемного расхода экономией
- ДРР в день размещения блогера → "Размещение X₽. ДРР считать рано"

### Субагент 4: Аналитик трафика и воронки (traffic-funnel.md)

**Вход:** `traffic` блок + summary advertising (для платный vs органик)
**Иерархия:** Бренд → Канал → Модель → (по статусам)
**Выход:**
- Воронка: показы → карточка → корзина → заказ → выкуп
- CR на каждом шаге + Δ + бенчмарки
- ASCII-визуализация воронки
- Органик vs Платное: НЕ суммировать
- Ad-dependent vs органические модели
- ⚠️ Выкупы — лаговый показатель
- ⚠️ Трафик ~20% gap с PowerBI — только тренд

**Диагностика аномалий:**
- CR карточка→корзина <5% → контент/фото, цена
- CR корзина→заказ <15% → ценовая конкурентоспособность
- Переходы ↓ >20% → позиции, бюджет, сезонность

### Субагент 5: Аналитик остатков (inventory.md)

**Вход:** `inventory` блок + summary finance (продажи по моделям)
**Иерархия:** Бренд → Канал → Модель → Склад (WB FBO / OZON FBO / МойСклад)
**Выход:**
- Остатки: WB FBO + OZON FBO + МойСклад + в пути
- Оборачиваемость = stock / daily_sales (дней)
- Рекомендации: пополнить / ок / распродать / FREEZE / ликвидация
- Тренды продаж (рост/падение/стабильно)

**Пороги:** <14д ДЕФИЦИТ, 14-60 OK, 60-90 ВНИМАНИЕ, 90-250 OVERSTOCK, >250 МЁРТВЫЙ

### Субагент 6: Аналитик цен (pricing.md)

**Вход:** `pricing` блок + summary inventory (оборачиваемость), finance (маржа по моделям)
**Иерархия:** Бренд → Канал → Модель → Артикул
**Выход:**
- Динамика цен + СПП
- Гипотезы "что если" с ₽ эффектом
- Годовой ROI = margin% × (365 / turnover_days)
- Ценовые рекомендации с confidence (HIGH/MEDIUM/LOW)

**Confidence discipline:** HIGH (|r|>0.5, >30д данных) → CUT допустим. MEDIUM → только HOLD или тест. LOW → HOLD.

**Ценовая логика:** Overstock+маржа>15% → снизить. Дефицит+маржа ок → повысить. Маржа<15% → не снижать. Мёртвый сток → маркдаун.

### Субагент 7: Аналитик план-факт (plan-fact.md)

**Вход:** `plan_fact` блок
**Иерархия:** Бренд → Канал → Модель/Артикул
**Выход:**
- План (месяц), Факт MTD, План MTD (пропорциональный), Выполнение MTD %, Прогноз, Прогноз vs План %
- Метрики: заказы шт/₽, продажи шт/₽, выручка, маржа, реклама внутр/внешн
- Статусы: ✅ >105% | ⚠️ 95-105% | ❌ <95%
- Рекомендации действий

### Субагент 8: Детектор аномалий (anomaly-detector.md)

**Вход:** summary ВСЕХ блоков (не отдельный slice, а обзор всего)
**Иерархия:** cross-cutting по всем уровням
**Выход:**
- Флаги: доля метрики изменилась >3 п.п., себестоимость скачок, логистика аномалия
- Все аномалии с ₽ эквивалентом
- Data quality: retention==deduction дубликаты, fan-out JOIN
- Severity: high (>5 п.п. или >100К₽), medium (3-5 п.п.)
- Глубокая диагностика модели при Δ >10% заказов

## 6. Субагенты-верификаторы (Wave 2)

Все 3 запускаются параллельно после Wave 1. Каждый получает `{{ALL_FINDINGS}}` — конкатенацию output всех 8 аналитиков.

### Субагент 9: Эксперт по маркетплейсам (marketplace-expert.md)

**Проверяет:**
- СПП не управляемый → нет рекомендаций по СПП
- Выкупы лаговые → не причина в дневном анализе
- Комиссия фиксирована → нет рекомендаций по комиссии
- WB склейки — заказы связанных артикулов
- Корректность формулировок по внешней рекламе

### Субагент 10: CFO (cfo.md)

**Проверяет:**
1. Точность цифр: текст = данные (±1%)
2. Покрытие: все warning/critical → рекомендации
3. Направление: действие соответствует проблеме
4. KB-правила: нет конфликтов
5. Реалистичность: ожидаемый эффект достоверен

**Verdict:** APPROVE / CORRECT / REJECT (макс 1 retry)

### Субагент 11: Data Quality Critic (data-quality-critic.md)

**Проверяет:**
1. Арифметика: сумма моделей = итого (±1%)
2. ДРР: с разбивкой внутр/внешн
3. СПП: средневзвешенный (не арифметическое среднее)
4. GROUP BY LOWER(): нет дублей моделей
5. Единая маржа: ВСЯ реклама включена
6. Выкуп %: не причина в дневном
7. Доли складываются: ≈100% (±2%)
8. Нет артефактов: "0%", "—", "NaN"

## 7. Synthesizer (Stage 3)

Получает: CFO output + все findings + corrected findings + quality flags

Производит **два формата:**
1. `final_document_md` — стандартный markdown (для git)
2. `final_document_notion` — Notion-enhanced (toggle headers, colored tables, callouts)

### 7.1 Структура отчёта (13 секций)

**Секция 0: Executive Summary (НОВАЯ — синтез ВСЕХ findings)**
- 10-15 строк: ключевые цифры бренда
- Маржинальность: текущая, Δ, тренд
- Скорость развития: динамика заказов/выручки, доля каналов
- План-факт: статус выполнения ключевых KPI
- Здоровье бренда: остатки, оборачиваемость, ценовая позиция
- ТОП-3 риска + ТОП-3 возможности
- Формат: callout-блок, сухой тон финансового директора

**Секция 1: Топ-выводы и действия**
3-5 находок с ₽ эффектом. Формат: [₽ эффект] Что → Гипотеза → Действие

**Секция 2: План-факт** (ОБЯЗАТЕЛЕН ВСЕГДА)
Таблица: Метрика | План (месяц) | Факт MTD | План MTD | Выполнение MTD % | Прогноз | Статус

**Секция 3: Ключевые изменения (Бренд)**
ПОЛНАЯ таблица 15 строк (без сокращений)

**Секция 4: Цены, СПП, ценовые гипотезы**

**Секция 5: Сведение ΔМаржи (Reconciliation)**
Таблица: Статья | Текущий | Прошлый | Δ | Влияние на маржу (₽)

**Секция 6: По маркетплейсам**
- 6.1 WB: объём, модельная декомпозиция (ВСЕ модели), по статусам, воронка, структура затрат, реклама
- 6.2 OZON: аналогично

**Секция 7: Маркетинг**
Исполнительная сводка, органик vs платное, внешняя реклама, матрица эффективности, чёрные дыры, daily dynamics

**Секция 8: Оборачиваемость и остатки**
Таблица: Модель | Остаток шт | Оборач. дн | Проблема | Действие

**Секция 9: Модели — драйверы/антидрайверы**

**Секция 10: Гипотезы → Действия → Метрики контроля**
Мин 5-10 гипотез (день: 3-5). Таблица: Приоритет | Объект | Гипотеза | Действие | Метрика | Эффект | Окно | Риски

**Секция 11: Рекомендации Advisor**
🔴 Критичные / 🟡 Внимание / 🟢 Позитивные

**Секция 12: Итог**
10-20 строк: что + почему + что двигало + что делать (1-3 действия)

## 10. Правила аналитики (полный свод)

### Жёсткие правила (передаются ВСЕМ субагентам)
1. GROUP BY модели = `LOWER(SPLIT_PART(article, '/', 1))`
2. Процентные метрики при объединении каналов = ТОЛЬКО средневзвешенные
3. ДРР = ВСЕГДА с разбивкой: внутренняя + внешняя ОТДЕЛЬНО
4. Единая маржа = включает ВСЮ рекламу
5. WB маржа = `SUM(marga) - SUM(nds) - SUM(reclama_vn) - COALESCE(SUM(reclama_vn_vk), 0) - COALESCE(SUM(reclama_vn_creators), 0)`
6. OZON маржа = `SUM(marga) - SUM(nds)`
7. Органик + Paid = НЕ суммировать
8. Числа — ТОЛЬКО из данных
9. Факт vs Гипотеза — ВСЕГДА разделять
10. Рекомендации = только через ЦЕНУ или МАРКЕТИНГ
11. Формат чисел: `1 234 567 ₽`, `24.1%`, `8.0% → 11.2% (+3.2 п.п.)`
12. Даты: "17 марта 2026" в тексте, "17.03.2026" в таблицах

### Лаговые/неуправляемые
- Выкуп % — лаг 3-21 дн. ЗАПРЕЩЁН как причина в дневном.
- СПП — НЕ управляемый.
- Комиссия — НЕ управляемая.

### Аналитические принципы
- Системное мышление: каскадные эффекты
- Evidence-based: гипотеза начинается с факт-триггера
- Эффект в деньгах: ₽ по конкретной модели
- Последствия 2-го порядка: заполнять "Риски"
- Приоритизация по ₽
- Тон: сухой финансовый директор

## 12. Технические ограничения

- **Размер данных:** JSON ~1-2MB. Для субагентов — slice ~200-400KB каждый.
- **Context window:** Каждый субагент работает в своём контексте Agent tool. Промпт + data slice должны вмещаться в ~100K токенов.
- **Время выполнения:** Коллектор ~30с, Wave 1 ~5-8м, Wave 2 ~3-5м, Synthesis ~5-10м, Publication ~2м. Итого ~15-25 минут.
- **Error handling:** Если коллектор не собрал блок — пропускаем аналитика, отмечаем в quality_flags. Если >3 блоков не собрано — STOP.
