# Marketing Hub — контекст для создания мокапа

> Этот документ — входные данные для создания UI-мокапа в Claude Artifacts.
> Задача: спроектировать раздел **«Маркетинг»** в Wookiee Hub с двумя страницами:
> 1. **Промокоды** — реестр + аналитика по неделям
> 2. **Поисковые запросы** — реестр + аналитика по неделям (частота, переходы, заказы)

---

## Контекст приложения

**Wookiee Hub** — внутренний инструмент команды e-commerce (WB/OZON). Тёмный интерфейс, Tailwind v4, shadcn/ui, lucide-react.

Навигация — вертикальный icon-bar слева (48px), при клике раскрывается sub-sidebar (200px) с подпунктами. Контент занимает оставшееся пространство.

Уже есть разделы: Операции, Коммуникации, Influence CRM, Аналитика. Добавляем **Маркетинг** с иконкой `Tag` (или `Percent`) и двумя подпунктами: «Промокоды» и «Поисковые запросы».

## Страница 1: Промокоды `/marketing/promo-codes`

### Что показываем

Два блока на одной странице:
- **Реестр промокодов** — таблица/список всех кодов с возможностью добавить новый
- **Аналитика** — недельная динамика выбранного кода (продажи, заказы)

### Данные реестра (`crm.promo_codes`)

```csv
id,code,channel,discount_pct,valid_from,valid_until,status
1,CHARLOTTE10,Соцсети,10%,–,–,active
2,OOOCORP25,Корп,25%,–,–,active
3,UFL6BFH9_AUDREY_TG10,Блогер,10%,–,–,active
8,WB:050B58B0-04F5-423C-9DA7-1402FABB5D44,–,–,–,–,active
9,AUDREY3,ЯПС,3%,06.05.2026,31.05.2026,active
10,AUDREY,ЯПС,3%,07.05.2026,31.05.2026,active
```

Каналы (enum): `Соцсети`, `Блогер`, `Корп`, `ЯПС` (Яндекс.Прямые продажи)

### Данные аналитики (`marketing.promo_stats_weekly`)

```csv
code,week_start,sales_rub,orders_count,returns_count,avg_discount_pct
CHARLOTTE10,2026-03-02,10845,7,0,10%
CHARLOTTE10,2026-03-09,1588,1,0,10%
OOOCORP25,2026-03-23,696,1,0,25%
UFL6BFH9_AUDREY_TG10,2026-04-06,2295,1,0,0%
WB:050B58B0-...,2026-04-27,1702,1,0,5%
```

Пока данных мало (5 строк — 5 промокодов × 1-2 недели). AUDREY3 и AUDREY только стартовали.

### UX Промокодов

**Реестр (верхняя часть страницы):**
- Таблица: `Код` | `Канал` | `Скидка` | `Дата старта` | `Дата окончания` | `Статус` | `Заказов (всего)` | `Продаж (всего, ₽)` | ` ⋮ `
- Кнопка «Добавить промокод» → Drawer справа с формой:
  - Поле: Название кода (text, required)
  - UUID WB (text, optional — для связи с API WB)
  - Канал (select: Соцсети / Блогер / Корп / ЯПС / другое)
  - Скидка % (number)
  - Дата старта / окончания (date pickers, optional)
- Строка кликабельна → открывает тот же Drawer в режиме редактирования

**Аналитика (нижняя часть / вкладка):**
- Выбор промокода (select или клик по строке реестра)
- Линейный/столбчатый график: ось X = недели, ось Y = `orders_count` + второй ряд `sales_rub`
- Если данных < 2 недель → заглушка «Данные появятся после следующего понедельника»
- Итого карточки: общие продажи, заказы, возвраты

## Страница 2: Поисковые запросы `/marketing/search-queries`

### Что показываем

Таблица артикулов в рекламных кампаниях + график динамики показателей WB для выбранного артикула.

### Данные реестра (`marketing.search_queries`)

87 записей. Каналы:

```csv
channel,count
creators,43   — продвижение через блогеров (WB Creators)
adblogger,13  — рекламные блогеры
yandex,17     — Яндекс.Директ
vk_target,5   — ВКонтакте
other,9       — прочее
```

Примеры записей:
```csv
id,code,channel,stat_weeks,last_week,avg_frequency,total_orders
71,Wendy/dark_beige_S,creators,40,2026-04-27,37321,1994
60,компбел-ж-бесшов/чер_S,creators,40,2026-04-27,5417,651
68,Audrey/nude_S,creators,40,2026-04-27,10653,520
13,Wendy/brown,adblogger,18,2026-04-27,2210,142
```

Формат кода: `Модель/цвет` (внутренний идентификатор артикула).

### Данные аналитики (`marketing.search_query_stats_weekly`)

2565 строк за 40 недель (с июля 2025). Поля:
- `frequency` — сколько раз пользователи искали этот артикул на WB
- `transitions` — переходы на карточку
- `additions` — добавления в корзину
- `orders` — заказы

Пример исторических данных лучшего артикула (`Wendy/dark_beige_S`, creators):
```csv
week_start,frequency,transitions,additions,orders
2025-07-28,15527,1784,138,21
2025-08-18,56110,6276,449,35
2025-10-20,148174,6246,1694,186
2025-11-10,289900,13207,3245,279   ← пик
2026-04-27,6408,315,63,8
```

### UX Поисковых запросов

**Реестр:**
- Фильтр по каналу (Creators / AdBlogger / Yandex / VK / Other)
- Таблица: `Артикул` | `Канал` | `Среднее frequency` | `Всего заказов` | `Последнее обновление`
- Поиск по названию артикула
- Кнопка «Добавить артикул» → Drawer: Код артикула + Канал

**Аналитика (нижняя часть / Drawer):**
При клике на строку → открывается Drawer или нижняя панель с графиком:
- 4 метрики на одном графике: частота / переходы / в корзину / заказы
- Переключатель периода: 4 нед / 13 нед / все время
- Воронка: frequency → transitions → additions → orders (конверсионная воронка за последнюю неделю)

## Дизайн-ориентиры

**Цветовая схема:** тёмный фон (`bg-background`), карточки на `bg-card`, акцент `--primary` (синий/фиолетовый).

**Паттерны из существующего Hub:**
- Sub-sidebar слева при клике на иконку раздела
- Таблицы с hover + selected state
- Drawer справа (300-480px) для форм и деталей
- Карточки-метрики вверху страницы (4 штуки в ряд)
- Recharts для графиков (уже используется в РНП-разделе)

**Компоненты shadcn/ui:** Table, Drawer, Select, DatePicker, Badge, Card, Tabs, Input, Button.
