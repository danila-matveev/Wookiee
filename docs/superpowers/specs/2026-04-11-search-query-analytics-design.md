# Аналитика поисковых запросов WB — Дизайн скрипта

**Дата:** 11 апреля 2026
**Статус:** Approved
**Подход:** A — Python-скрипт, повторяющий логику GAS, запись в новую Google Sheets

## Бизнес-цель

Еженедельный мониторинг видимости бренда Wookiee в поиске Wildberries: по каким запросам покупатели находят наши товары, открывают карточки, кладут в корзину и заказывают.

## Три категории отслеживаемых запросов

1. **Брендированные** — названия брендов/моделей на русском и английском, включая ошибки написания ("вуки", "wookiee", "Miafull", "Bella", "белла")
2. **Артикулы внешнего лида** — номенклатуры WB (nmId), используемые в рекламных каналах (Яндекс, Таргет ВК, Adsblogger)
3. **Подменные артикулы (WW)** — UTM-метки внутри WB: уникальный артикул, выданный конкретному креатору/блогеру для отслеживания эффективности. Один nmId → несколько WW-артикулов. Люди могут искать с ошибками (без WW, с одной W, с тремя W)

## API

- **Endpoint:** `POST https://seller-analytics-api.wildberries.ru/api/v2/search-report/product/search-texts`
- **Auth:** `Authorization: {WB_API_KEY}` (без Bearer)
- **Ограничения:** max 50 nmId за запрос, 3 req/min → пауза 21 сек между запросами
- **Два кабинета:** ООО (лимит ответа: 100 слов) и ИП (лимит: 30 слов)

## Архитектура

### Входные данные (из конфиг-листа Google Sheets)

**Лист "Аналитика по запросам":**
- Колонка A (строка 3+): поисковые запросы (бренды, артикулы, WW-коды)
- Колонка B (строка 3+): nmId основного артикула (для фильтрации переходов)

**Лист "nmIds":**
- Колонка A: nmId товара
- Колонка B: кабинет ("ООО Вуки" или "ИП Медведева П.В.")

### Алгоритм

```
1. Прочитать список запросов и маппинг (запрос → nmId) из листа
2. Прочитать все nmId с привязкой к кабинету
3. Для каждого кабинета:
   a. Разбить nmId на пакеты по 50
   b. Для каждого пакета: POST → API → получить items
   c. Пауза 21 сек между пакетами
   d. Для каждого запроса из списка: найти совпадения (case-insensitive includes)
   e. Суммировать: frequency, openCard (с фильтром по маппингу), addToCart, orders
4. Объединить результаты ООО + ИП (суммирование)
5. Записать 4 колонки (Частота, Переходы, Добавления, Заказы) в следующие пустые колонки
6. Записать даты периода в строку 2
```

### Фильтрация переходов (openCard)

Переходы считаются только для артикулов из маппинга (колонка B). Если для запроса указан nmId в колонке B — openCard учитывается только когда API вернул тот же nmId. Это отсекает переходы в чужие карточки.

### Выходные данные (Google Sheets)

**Лист "Аналитика по запросам":**
- Строка 1: заголовки (Частота, Переходы, Добавления, Заказы) — в следующих 4 пустых колонках
- Строка 2: даты периода (DD.MM.YYYY)
- Строка 3+: данные, привязанные к запросам из колонки A

**Лист "Аналитика по запросам (поартикульно)":**
- Плоская таблица: артикул продавца, nmId, название, запрос, метрики + dynamics

### Режимы запуска

- `auto` — прошлая неделя (Пн-Вс)
- `manual` — указать даты вручную
- `sheet` — прочитать даты из ячеек A1/B1
- `next_week` — найти последний загруженный период, взять следующую неделю

## Зависимости

- `httpx` — HTTP-клиент
- `gspread` — Google Sheets API
- `shared.clients.sheets_client` — обёртки для gspread
- `services.sheets_sync.config` — кабинеты, ключи, spreadsheet ID

## Rate Limits

- Между пакетами одного кабинета: 21 сек
- Между кабинетами: 1 сек
- При HTTP 429: пауза 60 сек, retry
