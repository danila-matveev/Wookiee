5. **P&L сверка**: Сумма плановых продаж по моделям = 35.8М. Сверить с трендом Q1 и планом финансиста (40.1М). Разрыв 4.3М — откуда?

## Важные правила

- **GROUP BY по модели — ВСЕГДА с LOWER()**: `LOWER(SPLIT_PART(article, '/', 1))`
- **СПП** — средневзвешенный при объединении каналов
- **Выкуп %** — лаговый (3-21 день), информационный только
- **ДРР** — всегда разбивка: внутренняя vs внешняя
- **Атрибуция WB**: внешний трафик на Wendy → покупки Audrey/Lana/Charlotte (склейки). Не оценивать эффективность модели только по её прямым заказам
- Маппинг моделей: `shared/model_mapping.py` → `map_to_osnova()`. vuki2→Vuki, moon2→Moon, компбел-ж-бесшов→Vuki
- Данные `content_analysis` (трафик): расхождение ~20% с PowerBI. Использовать как индикатор тренда, не абсолютные значения
- Все DB-запросы — через `shared/data_layer/`. Не писать SQL напрямую
