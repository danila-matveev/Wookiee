## Важные правила

- **GROUP BY по модели — ВСЕГДА с LOWER()**: `LOWER(SPLIT_PART(article, '/', 1))`
- **Даты**: используй `date < '2026-04-01'` (НЕ `date <= '2026-03-31'` — они дают разный результат при timestamp полях)
- **СПП** — средневзвешенный при объединении каналов: `sum(spp_amount) / sum(revenue_before_spp) * 100`
- **Выкуп %** — лаговый (3-21 день), информационный только
- **ДРР** — ВСЕГДА разбивка: внутренняя vs внешняя
- **Маржа WB** (`WB_MARGIN_SQL`): `marga - nds - reclama_vn - COALESCE(reclama_vn_vk,0) - COALESCE(reclama_vn_creators,0)` — это Маржа-2
- **Маржа OZON**: `marga - nds` — это Маржа-1 (OZON не имеет reclama_vn_vk/creators)
- **Атрибуция WB**: внешний трафик на Wendy → покупки Audrey/Lana/Charlotte (склейки)
- Маппинг моделей: `shared/model_mapping.py` → `map_to_osnova()`
- Все DB-запросы — через `shared/data_layer/`. НЕ писать SQL напрямую
- При расхождении с PowerBI — указать оба числа и объяснить причину расхождения
