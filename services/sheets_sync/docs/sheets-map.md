# Карта листов Google Sheets

Описание всех листов, задействованных в системе синхронизации: расположение чекбоксов, заголовков, данных и колонок.

---

## WB остатки

**Модуль:** `sync_wb_stocks.py`
**Источник:** WB Analytics API (асинхронный отчёт по складским остаткам)

| Ячейка / Диапазон | Содержимое |
|-------------------|------------|
| `A1` | Дата последнего обновления |
| `A2` | Время последнего обновления |
| `C1` | **Чекбокс-триггер** — GAS выставляет `TRUE` при нажатии кнопки |
| Строка 3 | Заголовки колонок |
| Строки 4+ | Данные |

**Колонки данных:**

| Колонка | Поле |
|---------|------|
| A | Баркод |
| B | Артикул |
| C | Размер |
| D | NMID |
| E | Категория |
| F | Бренд |
| G | Объем |
| H | Кабинет |
| I+ | Остатки по складам (динамически, по одной колонке на каждый склад) |

Список складов формируется динамически из ответа API — количество колонок зависит от набора активных складов.

## Ozon остатки и цены

**Модуль:** `sync_ozon_stocks_prices.py`
**Источник:** Ozon Seller API

| Ячейка / Диапазон | Содержимое |
|-------------------|------------|
| `C1` | **Чекбокс-триггер** |

**Источник SKU:** модуль читает список SKU из листа **"Все товары"**, колонка `FBO OZON SKU ID`. SKU разделяются по значению в колонке `Импортер` (ИП / ООО).

Лист содержит **30 колонок**, включая название кабинета (ИП/ООО) как отдельное поле для каждой строки.

## Аналитика по запросам

**Модуль:** `sync_search_analytics.py`
**Источник:** WB Search API

Данные записываются в два связанных листа:

### Лист "Аналитика по запросам"

Агрегированные данные по поисковым запросам.

### Лист "Аналитика по запросам (поартикульно)"

Детализация по каждому артикулу в разрезе запросов.

**Колонки (оба листа):**

| Колонка | Поле |
|---------|------|
| A | Поисковый запрос |
| B | nmID |
| C | Переходы |
| D | Добавления в корзину |
| E | %Open→Cart (конверсия из перехода в корзину) |
| F | Заказы |
| G | %Cart→Order (конверсия из корзины в заказ) |
| H | Дата начала периода |
| I | Дата конца периода |
| J | Кабинет |

## Фин данные NEW

**Модуль:** `sync_fin_data_new.py`
**Источник:** PostgreSQL

| Ячейка / Диапазон | Содержимое |
|-------------------|------------|
| `B1` | Дата начала периода |
| `C1` | Дата конца периода |
| `D1` | **Чекбокс-триггер** |
| Колонки A–G | Справочные данные, управляемые пользователем вручную |
| Колонки H–V | **15 метрик**, заполняемые синхронизацией |

**Пользовательские колонки (A–G):**

Заполняются и редактируются пользователем — синхронизация их не перезаписывает.

**Метрики синхронизации (H–V, 15 колонок):**

| Колонка | Метрика |
|---------|---------|
| H | Себестоимость |
| I | Ср. чек до СПП |
| J | Ср. чек после СПП |
| K | Заказов/день |
| L | Выкуп % |
| M | Маржа ₽ |
| N | Маржа до СПП % |
| O | Логистика |
| P | Прочие расходы |
| Q | НДС |
| R | Реклама |
| S | Маржа без ДРР % |
| T | ДРР общий % |
| U | ДРР внутренний % |
| V | ДРР внешний % |

> Процентные метрики рассчитываются как средневзвешенные значения за период.

## Статус синхронизации

**Модуль:** `status.py`

Служебный лист. Обновляется автоматически после каждого завершения синхронизации (успешного или с ошибкой).

**Колонки:**

| Колонка | Поле |
|---------|------|
| A | Скрипт (название sync-модуля) |
| B | Лист (название листа в Google Sheets) |
| C | Последний запуск (дата и время) |
| D | Статус (`OK` / `ERROR`) |
| E | Строк (количество записанных строк данных) |
| F | Длительность (время выполнения в секундах) |
| G | Ошибка (текст ошибки, если статус `ERROR`) |
