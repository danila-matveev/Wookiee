# Конфигурация и секреты

## Переменные окружения (.env)

Все переменные задаются в файле `.env` в корне проекта. Шаблон — `.env.example`.

### Wildberries

| Переменная | Описание |
|---|---|
| `WB_API_KEY_IP` | JWT токен кабинета ИП (Wildberries Seller API) |
| `WB_API_KEY_OOO` | JWT токен кабинета ООО (Wildberries Seller API) |

### Ozon

| Переменная | Описание |
|---|---|
| `OZON_CLIENT_ID_IP` | Client ID кабинета ИП (Ozon Seller API) |
| `OZON_API_KEY_IP` | API ключ кабинета ИП |
| `OZON_CLIENT_ID_OOO` | Client ID кабинета ООО |
| `OZON_API_KEY_OOO` | API ключ кабинета ООО |

### МойСклад

| Переменная | Описание |
|---|---|
| `MOYSKLAD_TOKEN` | Bearer token для МойСклад JSON API v2 |

### Google Sheets

| Переменная | Описание | По умолчанию |
|---|---|---|
| `GOOGLE_SERVICE_ACCOUNT_FILE` | Путь к JSON файлу Service Account | `credentials/google_sa.json` |
| `SPREADSHEET_ID` | ID целевой Google Таблицы (из URL) | — |
| `SYNC_TEST_MODE` | `true` — пишет в листы с суффиксом `_TEST`, `false` — в продакшен листы | `true` |

> Пока `SYNC_TEST_MODE=true`, все данные записываются в листы с суффиксом `_TEST` (например, `wb_prices_TEST`). Это позволяет проверять работу синков без риска затронуть основные данные.

### PostgreSQL (для fin_data синков)

Используется для синков, которые тянут данные из локальных PostgreSQL баз (финансовые данные, выгрузки).

| Переменная | Описание | По умолчанию |
|---|---|---|
| `DB_HOST` | Хост PostgreSQL сервера | — |
| `DB_PORT` | Порт | `5432` |
| `DB_USER` | Пользователь | — |
| `DB_PASSWORD` | Пароль | — |
| `DB_NAME_WB` | Имя БД для данных Wildberries | `pbi_wb_wookiee` |
| `DB_NAME_OZON` | Имя БД для данных Ozon | `pbi_ozon_wookiee` |

### Логирование

| Переменная | Описание | По умолчанию |
|---|---|---|
| `LOG_LEVEL` | Уровень логирования (`DEBUG`, `INFO`, `WARNING`, `ERROR`) | `INFO` |

## Script Properties (Google Apps Script)

GAS-файлы в папке `apps_script/` не читают `.env` — вместо этого они используют встроенный механизм `PropertiesService.getScriptProperties()`.

### Необходимые ключи

| Ключ | Значение |
|---|---|
| `WB_API_KEY_IP` | JWT токен WB кабинета ИП |
| `WB_API_KEY_OOO` | JWT токен WB кабинета ООО |
| `OZON_CLIENT_ID_IP` | Client ID Ozon кабинета ИП |
| `OZON_API_KEY_IP` | API ключ Ozon кабинета ИП |
| `OZON_CLIENT_ID_OOO` | Client ID Ozon кабинета ООО |
| `OZON_API_KEY_OOO` | API ключ Ozon кабинета ООО |
| `MOYSKLAD_TOKEN` | Bearer token МойСклад |

### Как добавить

1. Открыть Apps Script проект (через меню Расширения → Apps Script в Google Таблице)
2. В левом меню нажать шестерёнку **⚙️ Настройки проекта**
3. Прокрутить до раздела **Свойства скрипта**
4. Нажать **Добавить свойство**, ввести ключ и значение
5. Сохранить

> Ключи хранятся в зашифрованном хранилище Google и не попадают в историю коммитов — в отличие от хардкода в коде скрипта.

## Google Service Account

Service Account используется Python-синками для записи данных в Google Таблицу через API.

### Создание

1. Открыть [Google Cloud Console](https://console.cloud.google.com/)
2. Выбрать или создать проект
3. Перейти в **API и сервисы → Включённые API**, включить:
   - **Google Sheets API**
   - **Google Drive API**
4. Перейти в **IAM и администрирование → Сервисные аккаунты**
5. Создать новый сервисный аккаунт (название произвольное, роли не нужны)
6. Открыть созданный аккаунт → вкладка **Ключи** → **Добавить ключ → JSON**
7. Скачать JSON файл → разместить по пути `credentials/google_sa.json`

### Доступ к таблице

Открыть целевую Google Таблицу, нажать **Поделиться** и добавить email сервисного аккаунта (вида `name@project-id.iam.gserviceaccount.com`) с правами **Редактор**.

> Файл `credentials/google_sa.json` содержит приватный ключ — он добавлен в `.gitignore` и не должен попадать в репозиторий.

## Два кабинета: ИП и ООО

В системе настроены два отдельных аккаунта на каждом маркетплейсе:

- **ИП** — индивидуальный предприниматель
- **ООО** — общество с ограниченной ответственностью

Каждый кабинет имеет собственные API ключи для Wildberries и Ozon. Данные из обоих кабинетов объединяются в одних и тех же листах Google Таблицы — каждая строка содержит поле `cabinet` или аналогичный маркер, позволяющий различать источники. Это позволяет видеть консолидированную картину по всем юрлицам в одном месте.
