# wb-bundles — Склейки WB

## Назначение

Обновляет цены и скидки для товаров на листе "Склейки WB". Использует тот же WB Prices API, что и `sync_wb_prices.py`. Читает nmID из колонки D листа, строит словарь поиска по nmID, затем дописывает цены в колонки S–V (не перезаписывая остальные данные листа).

---

## API источник

| Параметр | Значение |
|---|---|
| Host | `discounts-prices-api.wildberries.ru` |
| Endpoint | `GET /api/v2/list/goods/filter` |
| Auth | `Authorization: Bearer {WB_API_KEY}` |

### Параметры запроса

| Параметр | Значение |
|---|---|
| `limit` | `1000` |
| `offset` | `0`, `1000`, `2000`, ... |

Полная документация по API: см. [wb-prices.md](wb-prices.md).

---

## Целевой лист

- **Название листа:** `Склейки WB`
- Скрипт не имеет собственного чекбокса запуска — запускается в составе общего пайплайна или вручную.
- **Чтение nmID:** колонка `D` (строки с данными, начиная с первой строки без заголовка)
- **Запись результатов:** колонки `S`–`V`

---

## Колонки (записываемые)

| Колонка | Заголовок | Источник |
|---|---|---|
| S | Цена | `sizes[0].price` |
| T | Скидка% | `sizes[0].discount` |
| U | Цена со скидкой | `sizes[0].discountedPrice` |
| V | Клубная скидка% | `sizes[0].clubDiscount` (если есть) |

---

## Логика обработки

```
# Шаг 1 — Загрузка цен с WB (оба кабинета)
nmid_map = {}  # nmID → { цена, скидка, цена со скидкой, клубная скидка }

для каждого кабинета (ООО, ИП):
    offset = 0
    цикл:
        GET /api/v2/list/goods/filter?limit=1000&offset={offset}
        для каждого товара:
            nmid_map[nmID] = { price, discount, discountedPrice, clubDiscount }
        если listGoods пуст — выйти
        offset += 1000

# Шаг 2 — Чтение nmID с листа
прочитать колонку D листа "Склейки WB"
собрать список (row_index, nmID) для непустых ячеек

# Шаг 3 — Сопоставление и запись
для каждой строки (row_index, nmID):
    если nmID найден в nmid_map:
        записать значения в ячейки S{row}, T{row}, U{row}, V{row}
    иначе:
        оставить ячейки пустыми (очистить)
```

---

## Обработка ошибок

| Ситуация | Поведение |
|---|---|
| HTTP 401 | Логировать "Неверный токен кабинета X", пропустить кабинет |
| nmID не найден в nmid_map | Очистить ячейки S–V для данной строки |
| Колонка D пустая | Завершить выполнение без записи |
| Лист не найден | Бросить исключение с понятным сообщением |
| Сетевая ошибка | До 3 повторных попыток с экспоненциальной паузой |
