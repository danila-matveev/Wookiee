# ozon-stocks-prices — Ozon Остатки и Цены

## Назначение

Выгружает остатки и цены с Ozon для всех SKU из справочника. SKU читаются с листа "Все товары", разделяются по кабинетам (ООО / ИП), после чего для каждого кабинета формируется асинхронный отчёт через Ozon Report API и скачивается в формате CSV.

---

## API источник

| Параметр | Значение |
|---|---|
| Host | `api-seller.ozon.ru` |
| Auth | Заголовки `Client-Id: {OZON_CLIENT_ID}` и `Api-Key: {OZON_API_KEY}` |
| Content-Type | `application/json` |

### Используемые эндпоинты

| Эндпоинт | Метод | Назначение |
|---|---|---|
| `/v1/report/products/create` | POST | Создать асинхронный отчёт |
| `/v1/report/info` | POST | Проверить статус отчёта |
| `{download_url}` | GET | Скачать CSV-файл отчёта |

### Тело запроса `/v1/report/products/create`

```json
{
  "language": "DEFAULT",
  "offer_id": ["SKU1", "SKU2", ...],
  "visibility": "ALL"
}
```

Батчи: максимум 1000 SKU на один запрос.

### Формат ответа `/v1/report/info`

```json
{
  "result": {
    "status": "success",
    "file": "https://..."
  }
}
```

Статус опрашивается каждые 10 секунд до получения `"success"` или `"failed"`.

## Целевой лист

- **Название листа:** `Ozon остатки и цены`
- **Чекбокс запуска:** ячейка `C1` — скрипт стартует только если значение `true`, после завершения сбрасывает в `false`.

### Источник SKU

- Лист `Все товары`, колонка `FBO OZON SKU ID`
- Разбивка по кабинетам: колонка `Импортер` — значение определяет, к какому кабинету относится SKU

## Логика обработки

```
# Шаг 1 — Чтение SKU
прочитать лист "Все товары"
разбить SKU по кабинетам на основе колонки "Импортер"

# Шаг 2 — Создание отчётов
для каждого кабинета (ООО, ИП):
    разбить SKU на батчи по 1000
    для каждого батча:
        POST /v1/report/products/create → получить code отчёта
        цикл опроса (10с):
            POST /v1/report/info → проверить статус
            если "success" — скачать CSV по file URL
            если "failed" — бросить исключение

# Шаг 3 — Обработка CSV
разобрать CSV → добавить колонку "Кабинет" к каждой строке

# Шаг 4 — Запись
объединить все батчи
записать заголовок + строки на лист "Ozon остатки и цены"
очистить чекбокс C1
```

---

## Обработка ошибок

| Ситуация | Поведение |
|---|---|
| HTTP 401 | Логировать "Неверные учётные данные кабинета X", пропустить кабинет |
| Статус отчёта `"failed"` | Бросить исключение, прервать батч |
| Лист "Все товары" не найден | Бросить исключение с понятным сообщением |
| Пустой список SKU | Записать только заголовки, не создавать отчёт |
| CSV не парсится | Залогировать предупреждение, пропустить батч |
| Сетевая ошибка | До 3 повторных попыток с экспоненциальной паузой |
