# ozon-bundles — Склейки Ozon

## Назначение

Обновляет цены и остатки FBO для товаров на листе "Склейки Озон". Читает кабинет и артикул из листа, группирует товары по кабинетам, создаёт отдельный отчёт Ozon Report API для каждого кабинета, затем дописывает цены и остатки в колонки R, S и V (не перезаписывая остальные данные листа).

---

## API источник

| Параметр | Значение |
|---|---|
| Host | `api-seller.ozon.ru` |
| Auth | Заголовки `Client-Id: {OZON_CLIENT_ID}` и `Api-Key: {OZON_API_KEY}` |
| Content-Type | `application/json` |

Используются те же эндпоинты, что и в `sync_ozon_stocks_prices.py`. Полная документация по API: см. [ozon-stocks-prices.md](ozon-stocks-prices.md).

---

## Целевой лист

- **Название листа:** `Склейки Озон`
- Скрипт не имеет собственного чекбокса запуска — запускается в составе общего пайплайна или вручную.
- **Чтение данных:** начиная со строки 3 (строки 1–2 — заголовки/служебные)
  - Колонка `A` — кабинет (ООО / ИП)
  - Колонка `E` — артикул (offer_id / vendorCode)
- **Запись результатов:** колонки `R`, `S`, `V`

---

## Колонки (записываемые)

| Колонка | Заголовок | Источник из CSV |
|---|---|---|
| R | Цена | `price` |
| S | Цена со скидкой | `marketing_price` / `price_after_discount` |
| V | FBO остаток | `fbo_present_stock` |

---

## Логика обработки

```
# Шаг 1 — Чтение листа
прочитать строки начиная с row 3
собрать список (row_index, cabinet, artikul) для непустых строк

# Шаг 2 — Группировка по кабинетам
cabinet_groups = { "ООО": [(row, artikul), ...], "ИП": [(row, artikul), ...] }

# Шаг 3 — Создание отчётов
для каждого кабинета в cabinet_groups:
    offer_ids = [artikul for (row, artikul) in cabinet_groups[cabinet]]
    разбить на батчи по 1000
    для каждого батча:
        POST /v1/report/products/create
            body: { offer_id: [...], visibility: "ALL" }
        цикл опроса (10с):
            POST /v1/report/info
            если "success" → скачать CSV
            если "failed" → бросить исключение

# Шаг 4 — Построение словаря поиска
sku_map = { artikul: { цена, цена_со_скидкой, fbo_остаток } }

# Шаг 5 — Сопоставление и запись
для каждой строки (row_index, cabinet, artikul):
    если artikul найден в sku_map:
        записать R{row} = цена
        записать S{row} = цена со скидкой
        записать V{row} = fbo остаток
    иначе:
        оставить ячейки без изменений (залогировать предупреждение)
```

---

## Обработка ошибок

| Ситуация | Поведение |
|---|---|
| HTTP 401 | Логировать "Неверные учётные данные кабинета X", пропустить кабинет |
| Статус отчёта `"failed"` | Бросить исключение, пропустить батч |
| Артикул не найден в CSV | Оставить ячейки R, S, V без изменений, залогировать предупреждение |
| Колонка A или E пустая в строке | Пропустить строку |
| Лист не найден | Бросить исключение с понятным сообщением |
| Сетевая ошибка | До 3 повторных попыток с экспоненциальной паузой |
