# wb-stocks — WB Остатки

## Назначение

Синхронизирует складские остатки с Wildberries по всем кабинетам. Выгружает остатки по каждому складу в виде отдельных колонок, специальные склады ("В пути до получателей" и т.п.) выводятся первыми.

---

## API источник

| Параметр | Значение |
|---|---|
| Host | `seller-analytics-api.wildberries.ru` |
| Endpoint | `POST /api/v1/warehouse_remains` |
| Auth | `Authorization: Bearer {WB_API_KEY}` |

### Схема запроса (асинхронная, двухфазная)

1. **Создать задачу** — `POST /api/v1/warehouse_remains` с телом запроса:
   ```json
   { "groupByBrand": false }
   ```
   Ответ: `{ "data": { "taskId": "..." } }`

2. **Опрос статуса** — `GET /api/v1/warehouse_remains/tasks/{taskId}/status`
   - Цикл: до 100 попыток × 15 секунд ожидания между запросами.
   - Когда статус `"done"` — переходить к шагу 3.
   - При статусе `"error"` — прерывать с исключением.

3. **Пауза** — подождать 60 секунд после получения статуса `"done"`.

4. **Скачать результат** — `GET /api/v1/warehouse_remains/tasks/{taskId}/download`
   Ответ: JSON-массив объектов остатков.

## Целевой лист

- **Название листа:** `WB остатки`
- **Чекбокс запуска:** ячейка `C1` — скрипт стартует только если значение `true`, после завершения сбрасывает в `false`.

---

## Колонки

Фиксированные колонки (1–8), затем динамические колонки складов:

| № | Заголовок | Источник поля |
|---|---|---|
| 1 | Баркод | `barcode` |
| 2 | Артикул | `vendorCode` |
| 3 | Размер | `techSize` |
| 4 | NMID | `nmId` |
| 5 | Категория | `category` |
| 6 | Бренд | `brand` |
| 7 | Объем | `volume` |
| 8 | Кабинет | Имя кабинета (ООО / ИП) |
| 9+ | [Склад 1], [Склад 2], ... | Динамически — остаток по конкретному складу |

Специальные склады размещаются в начале динамической части:
- `В пути до получателей`
- `В пути от получателей`
- Прочие транзитные статусы

## Логика обработки

```
для каждого кабинета (ООО, ИП):
    создать задачу → опросить статус (15с × 100) → пауза 60с → скачать

объединить результаты обоих кабинетов по ключу barcode

собрать полный список складов из всех записей
    → специальные склады первыми
    → остальные — в порядке появления

для каждой записи:
    фиксированные поля [1–8] + значения остатков по каждому складу (0 если склад отсутствует)

записать заголовок + строки на лист "WB остатки"
очистить чекбокс C1
```

---

## Обработка ошибок

| Ситуация | Поведение |
|---|---|
| HTTP 401 | Логировать "Неверный токен кабинета X", пропустить кабинет |
| Статус задачи `"error"` | Бросить исключение, прервать выполнение для данного кабинета |
| Превышено число попыток опроса (100) | Бросить исключение `TaskTimeoutError` |
| Сетевая ошибка | До 3 повторных попыток с экспоненциальной паузой |
| Лист не найден | Бросить исключение с понятным сообщением |
