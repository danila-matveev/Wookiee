# search-analytics — Аналитика запросов WB

## Назначение

Выгружает поисковую аналитику с Wildberries по обоим кабинетам в два листа: сводный отчёт по ключевым словам и детализированный отчёт по артикулам. Лимиты запросов различаются по кабинетам: ООО — 100 ключевых слов, ИП — 30.

---

## API источник

| Параметр | Значение |
|---|---|
| Host | `seller-analytics-api.wildberries.ru` |
| Auth | `Authorization: Bearer {WB_API_KEY}` |

### Используемые эндпоинты

| Эндпоинт | Метод | Отчёт |
|---|---|---|
| `/api/v2/nm-report/detail/history` | POST | Аналитика по артикулам (поартикульно) |
| `/api/v2/nm-report/grouped/history` | POST | Аналитика агрегированная по запросам |

### Формат тела запроса

```json
{
  "nmIDs": [123456, 789012],
  "period": {
    "begin": "2026-03-01 00:00:00",
    "end": "2026-03-30 23:59:59"
  },
  "timezone": "Europe/Moscow",
  "aggregationLevel": "day"
}
```

### Лимиты по кабинетам

| Кабинет | Максимум nmID в запросе |
|---|---|
| ООО | 100 |
| ИП | 30 |

---

## Целевые листы

### Лист 1: Аналитика по запросам

- **Название листа:** `Аналитика по запросам`
- Агрегированный отчёт: показатели сгруппированы по ключевому слову.

### Лист 2: Аналитика по запросам (поартикульно)

- **Название листа:** `Аналитика по запросам (поартикульно)`
- Детализированный отчёт: показатели в разрезе каждого артикула.

Оба листа обновляются в рамках одного запуска скрипта.

## Логика обработки

```
# Шаг 1 — Формирование списка nmID
прочитать nmID с листа (или из конфигурации)

# Шаг 2 — Разбивка по кабинетам и батчинг
для кабинета ООО: батчи по 100 nmID
для кабинета ИП: батчи по 30 nmID

# Шаг 3 — Запросы к API (агрегированный отчёт)
для каждого кабинета, для каждого батча:
    POST /api/v2/nm-report/grouped/history
    собрать строки для листа "Аналитика по запросам"

# Шаг 4 — Запросы к API (поартикульный отчёт)
для каждого кабинета, для каждого батча:
    POST /api/v2/nm-report/detail/history
    собрать строки для листа "Аналитика по запросам (поартикульно)"

# Шаг 5 — Запись
записать заголовки + строки на оба листа
```

## Обработка ошибок

| Ситуация | Поведение |
|---|---|
| HTTP 401 | Логировать "Неверный токен кабинета X", пропустить кабинет |
| HTTP 429 (rate limit) | Пауза 60 секунд, повторить запрос |
| Пустой ответ по батчу | Залогировать предупреждение, продолжить с следующим батчем |
| Превышен лимит nmID для кабинета | Автоматически разбить на батчи нужного размера |
| Лист не найден | Бросить исключение с понятным сообщением |
| Сетевая ошибка | До 3 повторных попыток с экспоненциальной паузой |
