# fin-data — Финансовые данные

## Назначение

Рассчитывает и записывает финансовые метрики по товарам (баркодам) из базы данных PostgreSQL. Объединяет данные WB и Ozon по баркодам, вычисляет 15 метрик и записывает их в колонки H–V листа. Колонки A–G (справочные данные пользователя) не затрагиваются.

> **Внимание:** `sync_fin_data.py` устарел и не используется. Актуальный скрипт — `sync_fin_data_new.py`.

---

## Источник данных

| Параметр | Значение |
|---|---|
| Тип источника | PostgreSQL |
| Доступ | Через `shared/data_layer.py` (шим: `scripts/data_layer.py`) |

### Используемые функции data_layer

| Функция | Назначение |
|---|---|
| `get_wb_fin_data_by_barcode(barcodes, date_from, date_to)` | Финансовые данные WB по баркодам |
| `get_ozon_fin_data_by_barcode(barcodes, date_from, date_to)` | Финансовые данные Ozon по баркодам |
| `get_barcode_mapping()` | Таблица соответствия баркодов WB ↔ Ozon |

## Целевой лист

- **Название листа:** `Фин данные NEW`
- **Чекбокс запуска:** ячейка `D1` — скрипт стартует только если значение `true`, после завершения сбрасывает в `false`.
- **Дата начала:** ячейка `B1`
- **Дата окончания:** ячейка `C1`

## Колонки

### Сохраняемые колонки A–G (пользовательские данные)

Колонки A–G содержат справочные данные, заполненные вручную. Скрипт их не читает для расчётов и не перезаписывает.

| Колонка | Назначение |
|---|---|
| A | Баркод (ключ для запроса к БД) |
| B–G | Произвольные поля пользователя |

### Записываемые колонки H–V (15 метрик)

| Колонка | Заголовок | Описание |
|---|---|---|
| H | Себестоимость | Средняя закупочная цена |
| I | Ср.чек до СПП | Средняя цена заказа до применения скидки постоянного покупателя |
| J | Ср.чек после СПП | Средняя цена заказа после СПП |
| K | Заказов/день | Среднесуточное количество заказов за период |
| L | Выкуп% | Доля выкупленных заказов |
| M | Маржа₽ | Маржа в рублях на единицу товара |
| N | Маржа до СПП% | Маржинальность до учёта СПП |
| O | Логистика | Расходы на логистику на единицу товара |
| P | Прочие расходы | Прочие операционные расходы |
| Q | НДС | Сумма НДС |
| R | Реклама | Расходы на рекламу (ДРР) |
| S | Маржа без ДРР% | Маржинальность без учёта рекламных расходов |
| T | ДРР общий% | Доля рекламных расходов от выручки (общий) |
| U | ДРР внутренний% | ДРР внутренней рекламы WB/Ozon |
| V | ДРР внешний% | ДРР внешней рекламы |

## Логика обработки

```
# Шаг 1 — Чтение параметров
прочитать date_from из B1, date_to из C1
прочитать список баркодов из колонки A (строки без заголовка)

# Шаг 2 — Запрос к БД
wb_data   = get_wb_fin_data_by_barcode(barcodes, date_from, date_to)
ozon_data = get_ozon_fin_data_by_barcode(barcodes, date_from, date_to)
mapping   = get_barcode_mapping()

# Шаг 3 — Объединение по баркоду
для каждого баркода:
    найти соответствующий баркод Ozon через mapping (если есть)
    объединить wb_data[barcode] + ozon_data[ozon_barcode] → combined

# Шаг 4 — Расчёт метрик
для каждого баркода из combined:
    рассчитать 15 метрик (H–V)
    все процентные метрики — средневзвешенные (см. AGENTS.md)

# Шаг 5 — Запись
записать только колонки H–V (A–G не трогать)
очистить чекбокс D1
```

## Обработка ошибок

| Ситуация | Поведение |
|---|---|
| Баркод не найден в БД (WB) | Записать 0 / пустые значения в H–V |
| Баркод не найден в БД (Ozon) | Использовать только WB-данные |
| Нет маппинга баркода | Использовать данные только по оригинальному баркоду |
| Некорректный формат дат B1/C1 | Бросить исключение с понятным сообщением |
| Лист не найден | Бросить исключение с понятным сообщением |
| Ошибка подключения к БД | Логировать детали, завершить выполнение |
