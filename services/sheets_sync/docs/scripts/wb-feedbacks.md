# wb-feedbacks — Отзывы WB

## Назначение

Агрегирует отзывы покупателей с Wildberries по обоим кабинетам (ООО и ИП) в разрезе nmID и рейтинга. Фильтрует отзывы по заданному диапазону дат, считает количество отзывов каждого рейтинга (1–5 звёзд) и записывает сводную таблицу на соответствующий лист.

---

## API источник

| Параметр | Значение |
|---|---|
| Host | `feedbacks-api.wildberries.ru` |
| Endpoint | `GET /api/v1/feedbacks` |
| Auth | `Authorization: Bearer {WB_API_KEY}` |

### Параметры запроса

| Параметр | Значение |
|---|---|
| `isAnswered` | `true` (первый проход) / `false` (второй проход) |
| `take` | `5000` (максимум на страницу) |
| `skip` | `0`, `5000`, `10000`, ... (пагинация) |

Оба значения `isAnswered` обрабатываются для полноты выборки.

### Формат ответа

```json
{
  "data": {
    "feedbacks": [
      {
        "id": "...",
        "nmId": 123456789,
        "productValuation": 5,
        "createdDate": "2026-03-01T12:00:00Z"
      }
    ],
    "countUnanswered": 42
  }
}
```

---

## Целевой лист

| Кабинет | Лист |
|---|---|
| ООО | `Отзывы ООО` |
| ИП | `Отзывы ИП` |

- **Чекбокс запуска:** ячейка `C1` — скрипт стартует только если значение `true`, после завершения сбрасывает в `false`.
- **Диапазон дат:** ячейка `A5` — дата начала, ячейка `B5` — дата окончания.

---

## Колонки

| № | Заголовок | Описание |
|---|---|---|
| 1 | Кабинет | Имя кабинета (ООО / ИП) |
| 2 | nmID | Артикул WB |
| 3 | Рейтинг | Средний рейтинг (взвешенный) |
| 4 | 5★ | Количество отзывов с оценкой 5 |
| 5 | 4★ | Количество отзывов с оценкой 4 |
| 6 | 3★ | Количество отзывов с оценкой 3 |
| 7 | 2★ | Количество отзывов с оценкой 2 |
| 8 | 1★ | Количество отзывов с оценкой 1 |

## Логика обработки

```
# Шаг 1 — Чтение параметров
для каждого листа (Отзывы ООО, Отзывы ИП):
    прочитать дату начала из A5 и дату окончания из B5

# Шаг 2 — Загрузка отзывов
для каждого кабинета (ООО, ИП):
    для isAnswered в [true, false]:
        skip = 0
        цикл:
            GET /api/v1/feedbacks?isAnswered={isAnswered}&take=5000&skip={skip}
            отфильтровать по createdDate (между датой начала и окончания)
            добавить отфильтрованные отзывы в список
            если получено < 5000 записей — выйти из цикла
            skip += 5000

# Шаг 3 — Агрегация
для каждого nmId:
    подсчитать количество отзывов каждого рейтинга (1–5)
    вычислить средний взвешенный рейтинг

# Шаг 4 — Сортировка и запись
отсортировать по общему количеству отзывов (по убыванию)
записать заголовок + строки на соответствующий лист
очистить чекбокс C1
```

## Обработка ошибок

| Ситуация | Поведение |
|---|---|
| HTTP 401 | Логировать "Неверный токен кабинета X", пропустить кабинет |
| Пустой `feedbacks` | Записать только заголовок без строк |
| Некорректный формат даты в A5/B5 | Бросить исключение с понятным сообщением |
| Лист не найден | Бросить исключение с понятным сообщением |
| Сетевая ошибка | До 3 повторных попыток с экспоненциальной паузой |
