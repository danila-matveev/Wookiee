## Погрешность №1: Обратная логистика в расчёте

**Что было:** В лист "Расчет логистики (короб)" попадали строки обратной логистики (возвраты, брак). У них фиксированный тариф 50 руб, их нельзя аудировать на переплату.

**Что стало:** Добавлен whitelist — в расчёт попадают только:
- "К клиенту при продаже"
- "К клиенту при отмене"

Все остальные типы (возврат брака, возврат неопознанного товара, возврат по инициативе продавца, от клиента при отмене/возврате) — исключены.

**Поле в отчёте WB:** "Виды логистики, штрафов и корректировок ВВ" (`bonus_type_name`)

**Файлы:**
- `models/report_row.py` — константа `FORWARD_DELIVERY_TYPES`, свойство `is_forward_delivery`
- `calculators/logistics_overpayment.py` — проверка `is_forward_delivery` перед расчётом
- `runner.py` — передача флага в калькулятор

## Погрешность №3: Неверный коэффициент склада

**Что было:** Всегда использовался `dlv_prc` из отчёта WB.

**Что стало:** 3-уровневый приоритет:
1. **Фиксированный коэффициент** — если фикс. коэф. > 0 И дата конца фиксации > дата заказа
2. **Supabase wb_tariffs** — исторические данные из ETL (Тарифы складов → Тарифы на остаток, по дате и складу)
3. **dlv_prc** — fallback из отчёта (помечается как "не верифицирован")

**Важно:** Коэффициент должен быть подтянут по дате и по складу для каждой строки. ETL-сбор тарифов: `python -m services.logistics_audit.etl.tariff_collector --backfill 30`

**Файлы:**
- `calculators/warehouse_coef_resolver.py` — `resolve_warehouse_coef()`, `load_supabase_tariffs()`
- `runner.py` — загрузка Supabase-тарифов, вызов резолвера

## Погрешность №4: Неверные базовые тарифы

**Что было:** Хардкод `base_tariff_1l = 46`, `base_tariff_extra_l = 14` для всех строк.

**Что стало:** Тариф определяется для каждой строки по формуле:

### Стоимость 1-го литра

**Если дата заказа > дата конца фиксации** (фиксация неактивна) — по дате заказа:

| Период | 1-й литр | Доп. литр |
|---|---|---|
| 14.08.2024 — 10.12.2024 | 33 | 8 |
| 11.12.2024 — 27.02.2025 | 35 | 8.5 |
| 28.02.2025 — 21.09.2025 | 38 | 9.5 |
| >= 22.09.2025 | 46 | 14 |

**Если дата заказа < дата конца фиксации** (фиксация активна) — по дате начала фиксации, те же периоды.

### Sub-liter тарифы (если дата заказа >= 22.09.2025 И объём < 1 литра)

| Объём (Л) | 1-й литр | Доп. литр |
|---|---|---|
| 0.001 — 0.200 | 23 | 0 |
| 0.201 — 0.400 | 26 | 0 |
| 0.401 — 0.600 | 29 | 0 |
| 0.601 — 0.800 | 30 | 0 |
| 0.801 — 1.000 | 32 | 0 |

**Файлы:**
- `calculators/tariff_periods.py` — `get_base_tariffs()` с таблицами периодов и sub-liter тиров
- `models/audit_config.py` — удалены поля `base_tariff_1l`, `base_tariff_extra_l`
- `runner.py` — вызов `get_base_tariffs()` для каждой строки

## Погрешность №5: Отрицательные разницы в итогах

**Что было:** Строки с отрицательной переплатой (WB удержал меньше, чем должен был) включались в лист и в итоговую сумму, завышая переплату.

**Что стало:** Строки с отрицательной разницей полностью исключены из листа "Переплата по логистике" (по рекомендации юристов). В СВОД суммируются только положительные переплаты.

**Файлы:**
- `output/sheet_overpayment_values.py` — строки с overpayment < 0 пропускаются
- `output/sheet_overpayment_formulas.py` — колонка "Включено в итог" (Да/Нет формула)
- `output/excel_generator.py` — СВОД агрегирует только overpayment >= 0
