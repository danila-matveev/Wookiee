# Как читать отчёт по оптимизации времени доставки OZON

Отчёт `Отчет_время_доставки_OZON_DD-MM-YYYY.xlsx` содержит 14 листов. Ниже — описание каждого листа, на что обращать внимание и как принимать решения.

---

## Общая логика отчёта

Алгоритм работает так:
1. Берёт данные OZON о времени доставки каждого артикула по маршрутам (кластер отгрузки → кластер доставки).
2. Находит артикулы, у которых среднее время доставки выше целевого (29 часов).
3. Для каждого такого артикула определяет, куда его нужно переместить, чтобы сократить время.
4. Проверяет, есть ли на других кластерах избыточные остатки, которые можно использовать как донора.
5. Генерирует рекомендации по перемещениям с расчётом экономического эффекта.

**Ключевой принцип**: если товар лежит на кластере, где его никто не покупает (или покупают мало), а спрос идёт из далёкого кластера — перемещаем товар ближе к спросу. Время доставки падает, наценка за нелокальную продажу снимается, конверсия растёт.

---

## 1. Сводка

**Что это**: Главный дашборд отчёта. Все ключевые метрики одним взглядом.

**На что смотреть**:
- **Текущее средневзвешенное время** — основная метрика (цель: ≤29 часов).
- **Прогноз после перемещений** — ожидаемое время после выполнения рекомендаций.
- **Распределение артикулов по статусу** — сколько артикулов в зоне OK / Внимание / Критично.
- **Общее количество перемещений** — сколько единиц товара нужно переместить.
- **Количество направлений ≥200 шт** — сколько заявок на перемещение реально можно подать (минимум OZON — 200 шт на одно направление склад→склад).
- **Экономия** — суммарная экономия в рублях от перемещений (наценка за нелокальную продажу + экономия логистики OZON).

---

## 2. 1_Критичные

**Что это**: Перемещения с наивысшим приоритетом. Артикулы со средним временем доставки >90 часов.

**Колонки**:
| Колонка | Описание |
|---------|----------|
| Артикул / SKU / Название | Идентификация товара |
| Откуда_кластер | Кластер, ГДЕ товар сейчас лежит (донор) |
| Куда_кластер | Кластер, КУДА нужно переместить (ближе к спросу) |
| Кол_во | Сколько единиц переместить |
| Спрос_нед | Среднее количество отгрузок в неделю на этом маршруте |
| Время_текущее | Текущее нормативное время доставки по маршруту (часы) |
| Время_новое | Прогнозное время после перемещения (обычно 28ч — локальная доставка) |
| Экономия_времени | На сколько часов сократится доставка |
| Среднее_время_артикула | Средневзвешенное время доставки этого артикула по ВСЕМ маршрутам |
| Стоимость_перемещения_руб | Стоимость перемещения этой партии (38 ₽/шт для товаров ≤1 литра) |
| Приоритет / Балл | Категория приоритета (Критично+ = 35, Критично = 30) |

**Как действовать**: Эти перемещения делать в первую очередь. Каждый час экономии на критичных артикулах даёт максимальный эффект.

---

## 3. 2_Важные

**Что это**: Перемещения средней важности. Артикулы со временем 30–90 часов или с высоким объёмом отгрузок.

Формат идентичен листу «1_Критичные». Приоритеты: Важно+ (25 баллов), Важно (20 баллов).

**Как действовать**: Делать после критичных. Здесь часто больше всего единиц — основной объём перемещений.

---

## 4. 3_Остальные

**Что это**: Перемещения низкой важности. Артикулы с небольшим объёмом отгрузок и умеренным временем. Сюда же попадают артикулы из «фазы дополнения» — добавленные, чтобы набрать минимум 200 шт по направлению.

Приоритет: «Можно улучшить» (15 баллов).

**Как действовать**: Включать в заявки, если направление уже набрало 200 шт (см. лист «Направления»).

---

## 5. Все перемещения

**Что это**: Полный список всех рекомендованных перемещений (объединение листов 1–3). Удобен для фильтрации и поиска конкретного артикула.

---

## 6. Направления

**Что это**: Агрегация перемещений по направлениям (кластер→кластер). Это ключевой лист для принятия решений о подаче заявок.

**Важные колонки**:
| Колонка | Описание |
|---------|----------|
| Всего_единиц | Суммарное кол-во единиц по направлению |
| Не_хватает_до_200 | Сколько ещё нужно добрать до минимума (0 = можно подавать) |
| Статус_заявки | «Можно подать заявку» / «Добрать до 200 шт» |
| Артикулы_список | Список артикулов в этом направлении |
| Экономия_наценка_всего | Суммарная экономия на наценке за нелокальную продажу (руб/нед) |

**Как действовать**:
1. Отфильтровать строки со статусом «Можно подать заявку».
2. Это и есть направления, по которым можно реально подать заявку в OZON.
3. Учитывать, что один кластер = несколько складов. Направление кластер→кластер может разбиться на несколько заявок склад→склад (см. файлы заявок в папке «Заявки на перемещение»).

---

## 7. Допоставки

**Что это**: Артикулы, для которых НЕ нашлось подходящего донора. Товар нужен на кластере, но взять его неоткуда — нужно делать новую поставку на этот кластер.

**Типичные причины** (колонка «Причина»):
- «Нет донора» — товар есть только на том кластере, куда нужно, или на международных складах.
- «Донор защищён» — товар есть на другом кластере, но там он тоже нужен (Дефицитный/Очень популярный).

---

## 8. Допоставки_кластеры

**Что это**: Сводка допоставок по кластерам. Показывает, на какие кластеры нужно поставить товар и в каком объёме.

---

## 9. Рекомендации по кластерам

**Что это**: Итоговая рекомендация по каждому кластеру-получателю.

**Колонки**:
| Колонка | Описание |
|---------|----------|
| Перемещений_шт | Сколько единиц можно переместить из других кластеров |
| Допоставка_шт | Сколько единиц нужно довезти новой поставкой |
| Итого_шт | Общее кол-во единиц для покрытия спроса |
| До_200_шт | Сколько не хватает до минимума заявки |
| Рекомендация | Текстовая рекомендация: «Подать заявку» / «Добрать» / «Допоставка» |

---

## 10. Анализ_артикулов

**Что это**: Детальная статистика по каждому артикулу.

**На что смотреть**:
- **Среднее_время** — средневзвешенное время доставки (цель ≤29ч).
- **Статус** — OK / Внимание / Критично.
- **Быстро_шт / Средне_шт / Долго_шт** — разбивка отгрузок по скорости. Если много «Долго» — артикул явно лежит не на тех кластерах.
- **Доля_быстро_%** — какой % отгрузок доставляется быстро (≤29ч). Чем выше — тем лучше.

---

## 11. Анализ_кластеров

**Что это**: Здоровье каждого кластера.

**Колонки**:
| Колонка | Описание |
|---------|----------|
| Статус | Избыточный / Популярный / Дефицитный — на основе данных OZON |
| Среднесуточные_продажи | Общие продажи по кластеру |
| Доступно | Текущий остаток |
| Дней_до_конца | На сколько дней хватит остатка |
| Наценка_% | Наценка за нелокальную продажу (0% = локальные кластеры) |
| Международный | Да/Нет — международные кластеры исключены из перемещений |

**Как читать**: Кластеры со статусом «Избыточный» и высоким остатком — основные доноры. Кластеры с высоким спросом и малым остатком — основные получатели.

---

## 12. Защита доноров

**Что это**: Показывает, сколько товара можно безопасно забрать с каждого кластера.

**Логика защиты**:
- **Избыточный / Без продаж**: забираем всё сверх запаса на 14 дней.
- **Популярный**: забираем осторожно — запас на 21 день (14 × 1.5).
- **Дефицитный / Очень популярный**: забирать запрещено.

**Колонка «Доступно_для_трансфера»** — это максимум, который можно переместить без риска дефицита.

---

## 13. Матрица времени

**Что это**: Нормативная матрица 25×25 — время доставки в часах между любыми двумя кластерами OZON.

- По строкам — кластер отгрузки (откуда отправляется товар).
- По столбцам — кластер доставки (куда едет покупателю).
- Значение 28 на диагонали — доставка внутри кластера (локальная).
- Чем дальше от диагонали — тем больше время.

**Применение**: Используйте для проверки — если видите в перемещениях странное направление, посмотрите, сколько часов реально экономится.

---

## 14. Экономика

**Что это**: Экономический расчёт по каждому перемещению.

**Ключевые колонки**:
| Колонка | Описание |
|---------|----------|
| Экономия_наценка_нед_руб | Экономия на отмене наценки за нелокальную продажу (руб/неделю) |
| Экономия_OZON_руб_нед | Экономия на логистике OZON (из данных OZON, руб/неделю) |
| Экономия_итого_нед | Суммарная экономия (руб/неделю) |
| Стоимость_перемещения_руб | Разовая стоимость перемещения |
| ROI_годовой | Годовой возврат инвестиций (%) |
| Окупаемость_нед | За сколько недель окупится перемещение |

**Как читать**: Сортировать по ROI_годовой (убывание) — вверху будут самые выгодные перемещения. Окупаемость менее 4 недель — однозначно делать. 4–12 недель — делать, если направление набирает 200 шт.

---

## Порядок работы с отчётом

1. **Открыть «Сводка»** — оценить масштаб проблемы и потенциал улучшения.
2. **Перейти на «Направления»** — найти направления со статусом «Можно подать заявку» (≥200 шт).
3. **Проверить «Рекомендации по кластерам»** — увидеть полную картину по каждому кластеру.
4. **Просмотреть «1_Критичные»** — убедиться, что критичные артикулы попали в подаваемые заявки.
5. **Открыть файлы заявок** из папки «Заявки на перемещение» — там уже готовые данные в формате шаблона OZON.
6. **При необходимости**: «Допоставки» — спланировать новые поставки на кластеры, которые нельзя закрыть перемещениями.
7. **Для бизнес-обоснования**: лист «Экономика» — конкретные суммы экономии и окупаемости.

---

## Ограничения

- Данные по спросу основаны на 1 неделе отгрузок. Для более точных рекомендаций нужен отчёт за 4–8 недель.
- Минимум OZON — 200 единиц на одну заявку (суммарно по всем артикулам в направлении склад→склад). Не по кластеру, а по конкретному складу.
- Один кластер может содержать несколько складов (например, Москва = 9 складов). Направление кластер→кластер может разбиться на несколько заявок склад→склад, каждая из которых должна набрать 200 шт.
- Перемещение занимает ~20 дней. Планируйте заранее.
- Стоимость перемещения 38 ₽/шт — для товаров объёмом до 1 литра. Для крупногабаритных товаров стоимость выше.
