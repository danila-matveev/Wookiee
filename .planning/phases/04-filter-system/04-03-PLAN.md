       Change signature:
       ```typescript
       addView: (entity: EntityType, name: string, columns: string[], filters: FilterEntry[], sort: TableSortState) => Promise<void>
       ```
       Map `FilterEntry[]` to `ViewConfig.filters`:
       ```typescript
       const configFilters = filters.map(f => ({
         field: f.field, op: "in", value: f.values
       }))
       const configSort = sort.field ? [{ field: sort.field, dir: sort.order }] : []
       ```
       Pass into `matrixApi.createView({ entity_type, name, config: { columns, filters: configFilters, sort: configSort } })`.

    2. **Add `loadView` action to views-store.ts:**
       ```typescript
       loadView: (view: SavedView) => void
       ```
       This needs to set:
       - Hidden columns: derive from `view.config.columns` (set difference against all field defs)
       - Active filters: map `view.config.filters` back to `FilterEntry[]`
         ```typescript
         const filterEntries: FilterEntry[] = view.config.filters.map(f => ({
           field: f.field,
           label: f.field,  // Will be resolved by UI from filterableDefs
           values: Array.isArray(f.value) ? f.value as number[] : [f.value as number],
           valueLabels: [],  // Will be resolved from lookupCache by UI
         }))
         ```
       - Sort: extract from `view.config.sort[0]` if present
       - Call `useMatrixStore.getState().clearFilters()` then set new filters via `addFilter` for each entry. Or better: add a `setFilters(entries: FilterEntry[])` bulk setter to matrix-store.

       **Add `setFilters` to matrix-store.ts:** `setFilters: (filters: FilterEntry[]) => set({ activeFilters: filters })`

       The `loadView` action should call `useMatrixStore.getState().setFilters(filterEntries)` to restore filters.
       For sort restoration, the page component needs to accept an external sort override — or `loadView` returns the parsed config and the page component applies it.

       Simpler approach: `loadView` stores the loaded view config in views-store state (`loadedViewConfig`), and page components react to it via a useEffect.
