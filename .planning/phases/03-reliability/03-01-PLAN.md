    - `@dataclass GateResult`: `name: str`, `passed: bool`, `detail: str = ""`, `is_hard: bool = True`
    - `@dataclass CheckAllResult`: `gates: List[GateResult]`, `target_date: date = None`, properties `hard_failed`, `soft_warnings`, `can_run` (exactly as in RESEARCH.md pattern)
    - `class GateChecker`:
      - `__init__(self)` — no params, uses shared/data_layer connections internally
      - `def check_all(self, marketplace: str = "wb", target_date: date = None) -> CheckAllResult`
      - Hard gates (3):
        1. `wb_orders_freshness`: `SELECT MAX(dateupdate) FROM abc_date` via `_get_wb_connection` — must be >= target_date
        2. `ozon_orders_freshness`: `SELECT MAX(date_update) FROM abc_date` via `_get_ozon_connection` — must be >= target_date
        3. `fin_data_freshness`: `SELECT MAX(dateupdate) FROM fin_data` via `_get_wb_connection` — must be >= target_date
      - Soft gates (3):
        1. `advertising_data`: `SELECT COALESCE(SUM(cost), 0) FROM advertising WHERE date = target_date` > 0
        2. `margin_fill_rate`: count of articles with margin > 0 / total articles > 0.5
        3. `logistics_data`: `SELECT COALESCE(SUM(delivery_rub), 0) FROM logistics WHERE date = target_date` > 0
      - Each gate query uses `_db_cursor(conn_factory)` pattern from shared/data_layer/_connection.py
      - CRITICAL: normalize datetime -> date using `try: update_date = val.date() except AttributeError: update_date = val` (per D-02, Pitfall 1)
      - `target_date` defaults to `date.today()` if None
      - Gate checker also collects summary metrics for pre-flight Telegram message (per D-05): orders count, revenue %, margin % — returned as `summary_metrics: dict` on CheckAllResult
    - `def format_preflight_message(result: CheckAllResult, report_names: list[str]) -> str` — formats the exact Telegram message per D-05:
      ```
      ✅ Данные за {date} готовы
      WB: | заказов {N} | выручка {X}% | маржа {Y}%
      OZON: | заказов {N} | выручка {X}% | маржа {Y}%
      📊 Запускаю: {report_names joined by ", "}
      ```
      If can_run is False, format as: `❌ Данные за {date} не готовы: {reasons}`
