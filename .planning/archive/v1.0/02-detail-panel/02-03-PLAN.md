**detail-panel.tsx** — Add edit state management:

1. `const [isEditing, setIsEditing] = useState(false)` — toggled by header button.
2. `const [editState, setEditState] = useState<Record<string, unknown>>({})` — clone of entity data when entering edit mode.
3. `const [saving, setSaving] = useState(false)`
4. `const [error, setError] = useState<string | null>(null)`

**Entering edit mode:** When "Редактировать" clicked: `setEditState({...data})`, `setIsEditing(true)`. Also trigger lookup prefetch (see below).

**Lookup prefetch:** On panel open (or when entering edit mode at latest), check `lookupCache` in Zustand for all 6 lookup tables. For each table NOT in cache, call `matrixApi.getLookup(table)` and store result via `setLookupCache(table, items)`. Use `Promise.all` for parallel fetching. Tables to prefetch: `kategorii`, `kollekcii`, `fabriki`, `statusy`, `razmery`, `importery` (from `Object.values(LOOKUP_TABLE_MAP)`).

**onChange handler:** `(fieldName: string, value: unknown) => setEditState(prev => ({ ...prev, [fieldName]: value }))`.

**handleSave:**
- Diff `editState` against original `data`: only include fields where value changed.
- If no changes: `setIsEditing(false); return`.
- Filter out `_name` fields and `id` field from the diff.
- Call correct PATCH based on entityType: `updateModel`, `updateArticle`, or `updateProduct`.
- On success: update local data (refetch or merge response), `setIsEditing(false)`.
- On failure: show error toast or inline error, do NOT close edit mode.
- Show `saving` state during the PATCH call.

**handleCancel:** `setEditState({}); setIsEditing(false); setError(null)`.

**Close panel while editing:** Show a confirmation or auto-cancel (Claude's discretion — recommend auto-cancel with no unsaved-changes guard for v1).

Pass `isEditing`, `editState`, `onChange`, `lookupCache` down to PanelBody.

**panel-body.tsx** — Extend to support edit mode:
- Accept new props: `editState`, `onChange`, `lookupCache` (from Zustand).
- For each PanelFieldRow, pass `editValue={editState[def.field_name]}` and `lookupOptions={lookupCache[LOOKUP_TABLE_MAP[def.field_name]]}` when applicable.
- Determine if field has lookup: `LOOKUP_TABLE_MAP[def.field_name]` exists.

**panel-header.tsx** — Wire edit toggle:
- "Редактировать" button calls `onToggleEdit`. When `isEditing` is true, hide the edit button (Save/Cancel in bottom bar replaces it).
- Optionally show "Редактирование..." indicator text when in edit mode.

Render PanelSaveBar at the bottom of SheetContent (inside detail-panel.tsx), conditionally when `isEditing`.

IMPORTANT: Only send changed fields on PATCH (diff against original data). Do NOT send all fields — backend uses `exclude_none=True` but frontend should also minimize payload.
