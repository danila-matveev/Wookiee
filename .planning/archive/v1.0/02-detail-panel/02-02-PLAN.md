**panel-field-row.tsx**:
- READ MODE ONLY in this plan. Props match `PanelFieldRowProps` from `panel/types.ts`, plus additional optional props:
  - `inherited?: boolean` — whether this field is inherited from a parent entity
  - `parentEntityType?: string` — parent entity type for navigation (e.g. "models" for an Artikul's inherited fields)
  - `parentEntityId?: number` — parent entity ID for navigation
  - `parentData?: Record<string, unknown>` — parent entity data for popover preview
- Render: Label (muted-foreground, ~40% width) + Value (foreground, right side). Notion-style 1-column stacked layout: label on left, value on right in a single row.
- If `def.is_system` or `IMMUTABLE_FIELDS.has(def.field_name)` AND `isEditing` is true: show a lock icon or "системное" badge next to the value. In read mode (isEditing=false), show value normally but with a subtle lock icon for system fields.
- For `field_type === "select"` in read mode: display the resolved `_name` value (not the raw `_id`). The parent component should pass the resolved name as `value`.
- Handle null/undefined/empty values: display "—" dash.
- Filter out computed fields: if `COMPUTED_FIELD_PATTERN.test(def.field_name)`, do NOT render this row (the resolved name is shown via the original field row instead).

**Inherited field popover** (per CONTEXT.md locked decision):
- When `inherited === true`, render the field value wrapped in a `Popover` (from `@/components/ui/popover` which uses `@base-ui/react/popover`).
- The field row should have a subtle visual indicator that it is inherited (e.g., a small arrow-up icon or lighter background tint).
- The field value is NOT editable (even in edit mode) — inherited fields are always read-only at the child level.
- **First click** on the inherited field value opens a `PopoverContent` showing a mini-preview of the parent entity: display the parent's title (e.g., model kod) and 3-4 key fields from `parentData`. Style as a compact card (padding, border, rounded).
- **Second click** (a "Перейти к [parent type]" link/button inside the popover): calls `openDetailPanel(parentEntityId, parentEntityType)` to navigate to the parent entity in the panel.
- Use the existing `Popover`, `PopoverTrigger`, `PopoverContent` from `@/components/ui/popover`.
