**ФОКУС:** Прибыльная компания с рентабельностью от 15% по чистой прибыли. 

1. **Маржинальная прибыль (₽) и маржа (%)** — ядро ежедневного управления (сколько зарабатываем на продажах).
2. **Годовая доходность** (годовой ROI) — возврат на вложенный капитал.
3. **Косвенные расходы (₽ и % от выручки)** — главный рычаг общей эффективности (не дать «съесть» маржу).

**Главное правило:**

> Любое управленческое решение должно улучшать маржинальную прибыль и/или чистую прибыль.
> 

**На январь 2026 года:** 
Mинимальная цель для БУ - 5 млн маржинальной прибыли при марже в 20% → 25 млн выручки → 38,5 млн суммы заказов
Средняя цель - 6,5 млн маржинальной прибыли при марже в 23% → 28,3 млн выручки → 43,5 млн суммы заказов
Высокая цель - 8 млн маржинальной прибыли при марже в 25%
Cупер цель - 10 млн маржинальной прибыли при марже в 30% (нужно регулярное расширение товранйо матрицы)
*при косвенных расходах на уровне 3.75 млн.

## Ключевые метрики и их значение

**Ежедневная аналитика**

| **Показатель** | **Пояснение показателя** | **Регулярность** | **Срезы** |
| --- | --- | --- | --- |
| Маржинальная прибыль, ₽ | Главный показатель эффективности продаж: сколько денег осталось после переменных затрат (комиссия, логистика, реклама, себестоимость и пр.) | Каждый день | Бренд → маркетплейс → модель → артикул + по статусам (”продается” / “выводим”) |
| Маржа, % (до СПП) | Показывает «качество» выручки. | Каждый день | Те же срезы |
| Выручка, ₽ | База для всех долей (реклама, комиссия, логистика). Без выручки долевые метрики искажаются. | Каждый день | Те же срезы |
| Сумма заказов, ₽ | Ранний индикатор спроса и будущей выручки. Нужна для контроля воронки и планов. | Каждый день | Те же срезы |
| Средний чек (до СПП) | Рычаг для маржи в % до СПП.  | Каждый день | Те же срезы |
| Средний чек (после СПП) |  | Каждый день | Те же срезы |
| ДРР рекламы (итого + по каналам) | Управляет маржой и спросом напрямую. | Каждый день | Бренд → маркетплейс → модель + разрез внутренняя/внешняя/ВК/блогеры |
| ROMI/вклад рекламы в марж.прибыль | ДРР может быть в норме, но маржи нет. Нужно смотреть, сколько маржинальной прибыли приносит реклама. | Каждый день (упрощенно), еженедельно (точно) | Канал рекламы → модель |
| Логистика и хранение, ₽ и % от выручки + ₽/ед | Часто «тихий убийца» маржи. Важна динамика: % и стоимость на единицу. | Раз в неделю | Маркетплейс → модель → артикул |
| Себестоимость, ₽ и % + ₽/ед | Если себестоимость по модели сильно отклоняется от средней, тогда смотрим себестоимость артикулов. | Каждый день и еженедельно | Маркетплейс → модель → артикул |
| СПП, % | Для понимания, чем реально достигнут объем (спросом или скидкой). При росте СПП пересматриваем цену. | Каждый день и еженедельно | Маркетплейс → модель → артикул при необходимости |
| Выкуп, % | Управляет истинной выручкой и логистикой. Падение выкупа = скрытое падение прибыли. | Еженедельно | Маркетплейс → модель → артикул при необходимости |
| Конверсии воронки (CTR → корзина → заказ) | Находит узкое место: трафик/карточка/цена/наличие. Без этого реклама «льется» в дырявую воронку. | Каждый день | Модель/артикул; источник трафика |
| Оборачиваемость, дни | Связка с «годовой доходностью»: деньги в товаре должны быстро возвращаться. | Еженедельно | Маркетплейс → модель |
| Косвенные расходы факт, ₽ | Еженедельно фиксируем косвенные расходы. Нельзя узнавать об отклонении от плана в конце месяца. | Еженедельно / ежемесячно (ОПИУ) | Статьи косвенных расходов |

**Ежедневное простое правило контроля:**

- Если **маржинальная прибыль** за день/неделю ниже плановой → искать причину **только по 5 рычагам**:
    
    **цена до СПП** → **СПП, %** → **ДРР**  → **логистика** → **выкуп**.
    
    *Дополнительная быстрая проверка себестоимости на ед. товара - не должно быть больших отклонений.*
    

- **Управление ценами**
    
    ### Ежедневные правила
    
    1. **Если растет СПП — поднимаем цену**
        - Если **СПП** вырос, то **поднимаем базовую цену минимум на величину роста СПП**, чаще — **с запасом** (чтобы маржа не просела).
        - Применяется **одинаково для WB и Ozon**.
    2. **Цена не имеет права ухудшать маржу**
        - Любое снижение цены на товары со статусом “продаются” допустимо только если **маржа до СПП** не падает ниже целевого уровня для модели.
    3. **Контроль “цена → маржа → объем”**
        - После изменения цены на следующий день проверяем: **заказы (₽), ДРР**.
        - Если заказы упали — считаем “потенциальную” маржинальную прибыль по усредненному проценту выкупа, чтобы оценить, компенсирует ли цена падение заказов → маржинальная прибыль должна увеличиться.
    4. **Запрет “отыгрывать” просадку заказов скидкой без расчета**
        - Если упали заказы, нельзя автоматически снижать цену. Сначала проверяем: **видимость/реклама/остатки/конверсии**.
    
    ---
    
    ### Еженедельные правила
    
    1. **Еженедельная переоценка по ABC (по маржинальной прибыли)**
        - **A-товары:** не допускаем падения маржи → цена корректируется первой.
        - **B-товары:** тесты цен (ступенчато) для перевода в A.
        - **C-товары:** цена либо под цель, либо распродажа/вывод (без бессмысленной рекламы).
    2. **Правило “защиты маржи” по каналам**
        - Раз в неделю пересчитываем целевую цену с учетом факта: **комиссия %**, **логистика ₽/ед**, **ДРР**, **СПП и тд**.
        - Если что-то стало “дороже” (комиссия/логистика) — **цена должна это компенсировать**.
        - Рост ДРР не означает автоматического увеличения цены - сначала проверять конверсии, источники трафика (внутренняя или внешняя реклама).
    3. **Пороговые решения по результату недели**
        - Если **маржинальная прибыль** по артикулу/модели ниже плана при нормальном объеме и плановом ДРР — значит проблема в цене → **корректируем цену**, а не “доливаем рекламу”.
        - Если объем упал, но маржа и маржинальная прибыль выросли — **не откатываем цену автоматически**.
    4. **Календарь ценовых изменений**
        - Ограничение: не более **1–2 изменений цены в неделю на SKU**, чтобы видеть причинно-следственную связь.
- **Управление внутренней рекламой**
    1. Если растет ДРР и CPL, останавливаем РК для оценки ситуации: 
    - проверяем изменения CR, CTR, 
    - позиции по важным ключам, 
    - проверяем в каких ключах произошло изменение (снизились конверсии)
    - проверяем остатки и распределение на ключевые склады
    - смотрим отзывы на негатив
    - смотрим что происходит у конкурентов и в категории
    - в случае, если все с предыдущими пунктами нормально, пересматриваем ассортимент на продвижение. Возможно просто сменился тренд.
    2. Если CR норм, CPL норм, клики ↓, CTR ↓ — конкуренты обходят по позиции, CTR или цене. 
    - оцениваем ситуацию, что необходимо и мы можем изменить: главную, цену или войти в акцию
    3. Необходимо следить за уровнем органического трафика в динамике. При продвижении  внутренними инструментами количество органических переходов должно расти в рамкам ёмкости рынка (не плохо было бы нам её посчитать).
    4. Продвижение новых моделей может происходит с повышенным ДРР, при этом органические позиции карточек и органический трафик должны расти, а ДРР снижаться в динамике.
- **Правила ABC-анализа**
    
    **Правило выбора метрики ABC:**
    
    1. Основной ABC: **по маржинальной прибыли, ₽**
    2. Доп. ABC: по **выручке, ₽** (для спроса) и по **оборачиваемости/остаткам** (для капитала)
    
    **Регулярность ABC:**
    
    - **Еженедельно**: быстрый ABC по маржинальной прибыли.
    - **Ежемесячно**: полный ABC + пересборка ассортимента.
    
    **Пороговая логика:**
    
    - **A**: товары, дающие ~70**%** маржинальной прибыли
    - **B**: следующие ~20**%**
    - **C**: оставшиеся ~10**%**
    
    **Правила действий по группам:**
    
    **A (ядро прибыли):**
    
    - Держать **наличие** и **позиции в выдаче**.
    - Любые эксперименты (цена/контент) — аккуратно, с лимитом потерь.
    - Цель: растить вклад A, не роняя маржу.
    
    **B (зона роста):**
    
    - Тесты: цена, реклама, карточка, комплекты.
    - Перевод B → A через улучшение контента/рекламы.
    - Если B не растет в марже — не «доливать» рекламу автоматически.
    
    **C (кандидаты на чистку):**
    
    - Запрет на «поддерживающую рекламу» без доказанной маржи.
    - Решение: **изменить цену / рекламу / вывести из ассортимента**.
    - Если у категории C большой % денег в остатках и не дает продажи и маржу — это минус годовой доходности.
    
    **Мини-набор показателей для ABC:**
    
    По каждой позиции (модель/артикул):
    
    - вклад в маржинальную прибыль, ₽
    - маржа %, до СПП
    - ДРР %
    - логистика ₽/ед
    - выкуп %
    - оборачиваемость, остаток ₽
    - годовой ROI
- **Правила анализа ОПИУ (P&L)**
    
    **Регулярность:**
    
    **Еженедельно:** факт по маржинальной прибыли и косвенным
    
    **Ежемесячно:** финальный ОПИУ (закрытие) + разбор отклонений.
    
    **Обязательные показатели ОПИУ:**
    
    1. Выручка
    2. Прямые расходы (комиссия, логистика, хранение)
    3. Доля прямых расходов в выручке, %
    4. Маржинальная прибыль
    5. Маржа, %
    6. Косвенные расходы
    7. Прямые расходы
    8. Рекламные расходы
    9. Доля рекламных расходов в выручке, %
    10. Чистая прибыль
    11. Чистая рентабельность
    12. Годовая доходность
    
    **Правила отклонений**
    
    Любое отклонение чистой прибыли объясняется только тремя блоками:
    
    **маржинальная прибыль** → **косвенные** **расходы** → **прочее (налоги/штрафы/разовые)**.
    
    1. Если маржинальная прибыль «в плане», а чистая нет → проблема **в косвенных**.
    2. Если выручка «в плане», а маржинальная прибыль нет → проблема **в юнит-экономике** (цена/ДРР/комиссия/логистика).
