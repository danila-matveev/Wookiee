# /hygiene-resolve

Когда пользователь вставляет команду — ты (Claude в локальной сессии) делаешь следующее.

## Стиль общения (КРИТИЧНО)

Пользователь — **не разработчик**. Никаких слов «PR», «коммит», «ветка», «merge», «refactor», «migration». Заменяй на: «изменение», «папка», «база», «правка». Аналогии из жизни приветствуются. Объясняй ЧТО и ЗАЧЕМ простым языком. См. memory `feedback_plain_language.md`.

## Алгоритм

1. **Прочитать `.hygiene/queue.yaml`**. Если файл не существует или пустой:
   > Готово, вопросов нет. Можешь закрывать.

   Завершиться.

2. **Сгруппировать вопросы** — если 2 находки про одну папку/тему, объединить в один вопрос с подвопросом. Иначе — отдельные блоки.

3. **Для каждого вопроса (не более 3 за раз)**:

   - Расскажи ЧТО нашли: 1 предложение.
   - Расскажи ЗАЧЕМ это важно: 1 предложение (аналогия — ок).
   - Дай **рекомендацию**: «я бы сделал X, потому что...»
   - Дай 2 варианта ответа: например `удалить (рекомендую) / оставить`.
   - **Будь честным про неопределённость**: если ты не на 100% уверен — скажи «я проверил X и Y, но абсолютной гарантии нет».
   - Жди ответа пользователя.

4. **На каждый ответ**:
   - Записать решение в `.hygiene/decisions.yaml` через `shared.hygiene.decisions.record_decision(finding_id, decision, reason, expires=+90d)`.
   - Применить решение **сейчас** (используя то же `shared.hygiene.autofix.apply_finding()` что и night-coordinator):
     - `delete` → `git rm`
     - `keep` → no-op (только запись в memory)
     - `exclude` → добавить в `.gitignore` правило
     - `archive` → `git mv` в `docs/_archive/YYYY-MM/`
   - Удалить пункт из `.hygiene/queue.yaml`.

5. **После всех ответов**:
   - Если ничего реального не сделал (всё было `keep`) → просто запиши decisions и скажи «Готово, запомнил».
   - Если что-то изменилось → создай ветку `hygiene-resolve/YYYY-MM-DD-HH-MM`, коммит, push, открой PR через `gh pr create`, включи auto-merge `gh pr merge --auto --squash --delete-branch`.
   - Скажи пользователю на простом языке:
     > Готово. Изменения попадут в основную папку через 5 минут — нужна автоматическая проверка. Память записал, второй раз про эти штуки спрашивать не буду до августа.

## Что **не делать**

- Не задавай больше 3 вопросов за раз. Остальное останется в `queue.yaml` до завтра — пользователь придёт ещё раз.
- Не показывай технические термины. «PR» можно только в скобках как пояснение: «(автоматическая проверка, по-английски PR)».
- Не пытайся объяснять что такое git, autofix, mirror — это не его задача.
- Не делай force-push. Не делай `--no-verify`. Не пиши в main напрямую — всегда через PR.
