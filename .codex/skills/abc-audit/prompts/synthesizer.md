# Структура отчёта v4 (13 секций)

**Принцип v4:** Секции I-VI — детальный v2-стиль (помесячные таблицы, причинно-следственные цепочки, полные вердикты). Секции VII-XI — компактный v3-стиль (таблицы вместо списков). Секции XII-XIII — сводные выходы.

## ▶ I. Executive Summary

Три блока:
1. **Ключевые метрики** — таблица m1/m2/m3 + тренд: Выручка, Маржа, Маржинальность, ДРР (с разбивкой internal/external), артикулов Продается, ABC-распределение, флаги проблем.
2. **Ключевые возможности** — 2-3 callout-блока с главными инсайтами в цифрах.
3. **Ключевые риски** — 2-3 callout-блока.

## ▶ II. Качество данных

Два блока:
1. **Покрытие данных** — таблица: иерархия, финансы, выкупы, остатки, статус покрытия.
2. **Флаги качества** — таблица: флаг, кол-во, комментарий.
3. **Проблемы целостности** — таблица: проблема, детали.

## ▶ III. Полная артикульная таблица

Ссылка на Google Sheets. Описание колонок листа "Все артикулы".

## ▶ IV. Топ по моделям

**ВСЕ** модели с активными продажами (Продается + Выводим), отсортированы по абсолютной марже m3. Для каждой — топ-2 артикула с ABC, маржинальностью и ДРР.

Формат: таблица с колонками # | Модель | Арт. | Rev m3 | Margin m3 | Margin% | ДРР% | ABC | Топ-1 | Топ-2

Не обрезать список — показывать ВСЕ модели.

## ▶ V. Матрица ABC × ROI

Сводная таблица 3×3 (A/B/C × Green/Yellow/Orange/Red/NoData):
- Количество артикулов и сумма маржи в каждой ячейке
- Перечисление ключевых артикулов ячейки
- После таблицы — интерпретационный callout

## ▶ VI. Приоритетные действия

Все P0, P1, P2 вердикты Арбитра. Для каждого:
- Заголовок: `### P{N}-{#}. Краткое описание`
- **Приоритет** | **Эффект** (₽/мес) | **Confidence** (%)
- **Причинно-следственная цепочка** — развёрнутая, с помесячной динамикой в таблице
- **Разрешённое противоречие** (если есть, из арбитра)
- **Действия** — нумерованный список с ответственными

Правило: "тестировать", не "остановить" (кроме явно убыточных). "Возврат из вывода" — с оговоркой "требует проверки причины".

## ▶ VII. Склейки и halo-эффект

Таблица всех 27 склеек WB: Склейка | Арт. | Rev m3 | Margin m3 | Adv m3 | ДРР m3.
Показывать только активные (Rev m3 > 0) полностью, неактивные — одной строкой.

## ▶ VIII. Находки Товароведа

Таблицы (НЕ списки) по блокам:
- Дефицит запасов (stock < 500, sales > 20/мес): Артикул | Сток | Sales m3 | Дней до OOS | Rev m3 | Margin m3 | ABC | MOQ мес
- Оверсток (> 24 мес, > 10K шт): Артикул | Сток | Sales m3 | Мес запаса | Rev m3 | ABC | Замороженный капитал
- Отрицательная маржа (Продается, m3): Артикул | Rev m3 | Margin m3 | Margin% m3 | ДРР m3 | ABC | Тренд
- Color Code анализ: спред маржи по BE

## ▶ IX. Находки Финансиста

Таблицы:
- Топ-10 unit economics (m3): Артикул | ABC | Rev/шт | Margin/шт | Adv/шт | Margin% | ДРР% | Buyout%
- Ценовые аномалии (маржа <15% + declining/flat): помесячная динамика
- Замороженный капитал топ-10: Артикул | Сток шт | Себ./шт | Капитал | Оборач. дн | Margin%
- ИП vs ООО: сравнительная таблица

## ▶ X. Находки Маркетолога

Таблицы:
- Артикулы с высоким ДРР (>20%, Продается): Артикул | Модель | Склейка | Rev m3 | Margin m3 | ДРР m3 | Adv int | Adv ext | ABC
- Недоинвестированные звезды (высокая маржа, низкий ДРР)
- Перераспределение бюджета: Откуда | Сумма | Куда | Эффект
- Помесячная динамика ДРР: топ-аномалии

## ▶ XI. Критические противоречия

Таблица разрешённых противоречий арбитра: ID | Тип | Суть | Решение | Conf.

## ▶ XII. Сводная таблица действий

Все приоритетные действия, сортировка по эффекту внутри каждого приоритета.

Формат:
```
| # | Приоритет | Действие | Артикул / Группа | Эффект ₽/мес | Срок | Ответственный |
```

## ▶ XIII. Итог

10-20 строк: период + протокол + вердикт, главные цифры m3 (выручка, маржа, маржинальность, замороженный капитал), главные 3 возможности с эффектом, главные 3 риска, следующие шаги (эта неделя).

# Формат вывода — Notion Enhanced Markdown

**ВАЖНО:** Отчёт генерируется в Notion Enhanced Markdown формате и публикуется через Notion MCP. НЕ использовать plain markdown pipe-таблицы — они не отображают цвета в Notion.

## Таблицы — ТОЛЬКО `<table>` формат

```
<table fit-page-width="true" header-row="true">
<tr color="gray_bg">
<td>**Заголовок 1**</td>
<td>**Заголовок 2**</td>
</tr>
<tr>
<td>Значение</td>
<td>Значение</td>
</tr>
<tr color="red_bg">
<td>Критическая строка</td>
<td>Выделена красным</td>
</tr>
</table>
```

### Цвета строк — обязательная семантика:
- `color="red_bg"` — убыточные, критические (маржа <0, ДРР >30%, OOS <5 дней)
- `color="orange_bg"` — предупреждения (ДРР 15-30%, declining маржа, дефицит <30 дней)
- `color="yellow_bg"` — внимание (статус некорректный, мониторинг, пограничные)
- `color="green_bg"` — позитивные (итоги, лучшие показатели, рост)
- `color="blue_bg"` — информационные заголовки (модели, склейки)
- `color="gray_bg"` — заголовки таблиц (header row)
- Без цвета — обычные строки

### Заголовки таблиц P0/P1/P2:
- P0 таблица: `<tr color="red_bg">` для header
- P1 таблица: `<tr color="orange_bg">` для header
- P2 таблица: `<tr color="yellow_bg">` для header

<callout icon="💡" color="blue_bg">

	Ссылка на Google Sheets
</callout>

<callout icon="⏰" color="yellow_bg">

## Секции

- Секция 1 (Executive Summary): `## 1. Executive Summary` — БЕЗ toggle, всегда развёрнута
- Секция 2 (Sheets ссылка): `## 2. Полная артикульная таблица` — БЕЗ toggle
- Секции 3-11: `## {N}. Название {toggle="true"}` — с toggle, контент с табуляцией
- Секции 12-13: `## 12. Сводная таблица действий` / `## 13. Итог` — БЕЗ toggle
- Подсекции: `### {N}.{M} Название`

**Toggle-контент:** Всё внутри toggle-секции начинается с TAB (`\t`).

## Терминология — только русский

## Числа
- Рубли: пробел-разделитель + ₽ (`1 234 567 ₽`, или `8,8М ₽` для крупных)
- Проценты: одна десятая (`24,1%`), изменение со знаком (`+16,9%`)
- п.п.: `+3,2 п.п.`
- Штуки: пробел (`1 234 шт`), дни: `45 дн`
- Нет данных: `—`
- Значимые изменения (>±10%) — **жирным**

## Confidence
Каждое решение сопровождается уровнем и обоснованием.

## Причинно-следственные цепочки
НЕ: "Joy/чёрный — кандидат на вывод"
ДА: "Joy/чёрный: маржинальность 18%, ROI 72% (vs медиана 176%). ДРР = 2% → недоинвестирован → низкие продажи (2 шт/д vs медиана 8) → медленная оборачиваемость (92 дн)"

# Правила контента
- ТОЛЬКО данные из вердиктов Арбитра. Никакой собственной аналитики.
- Все ячейки заполнены если данные есть. Нет данных → "—"
- Никогда не выдумывать модели или артикулы
- Если причина проблемы неизвестна — "требуется проверка", НЕ предполагать конкретную причину
- "Возврат из вывода" — с оговоркой "требует подтверждения причины вывода"
- Рекомендации по рекламе — "протестировать", не "остановить"
- Находки с >5 элементами — ВСЕГДА `<table>`, НЕ bullet-списками
- Bullet-списки допустимы только для 1-4 элементов
- Секции 8-10 (находки экспертов) — в конец, после действий
