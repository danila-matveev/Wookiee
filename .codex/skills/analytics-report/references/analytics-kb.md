## 1. Бизнес-контекст и цели

### Фокус компании

Прибыльная компания с рентабельностью от 15% по чистой прибыли. Категория: женское нижнее бельё. Каналы: WB (~85% выручки), OZON (~15%).

**Три ключевых показателя:**
1. **Маржинальная прибыль (₽) и маржинальность (%)** — ядро ежедневного управления
2. **Годовая доходность (ROI)** — возврат на вложенный капитал: `margin_pct × (365 / turnover_days)`
3. **Косвенные расходы (₽ и % от выручки)** — рычаг общей эффективности

### Целевые показатели

| Уровень | Маржа ₽ | Маржинальность % | Выручка | Сумма заказов |
|---------|---------|-------------------|---------|---------------|
| Минимум | 5 млн | 20% | 25 млн | 38.5 млн |
| Средний | 6.5 млн | 23% | 28.3 млн | 43.5 млн |
| Высокий | 8 млн | 25% | — | — |
| Супер | 10 млн | 30% | — | — |

*При косвенных расходах ~3.75 млн.*

### Главное правило

Любое управленческое решение должно улучшать маржинальную прибыль и/или чистую прибыль. Всё остальное — производное.

### Шкала маржинальности (единый стандарт)

| Маржинальность | Зона | Действие |
|---|---|---|
| > 25% | Зона роста | Масштабировать |
| 20-25% | Целевая зона | Норма |
| 15-20% | Зона внимания | Требует оптимизации |
| < 15% | Критическая зона | Stop-loss |

### Модели A-группы

- **wendy** — ключевая модель (~28-30% маржи бренда)
- **ruby** — вторая по значимости (~8-10%)
- **set_vuki** — третья (~5%)

### ABC-анализ

- **A**: ~70% маржинальной прибыли — держать наличие и позиции, эксперименты аккуратно
- **B**: ~20% — тесты цена/реклама/карточка, перевод B в A
- **C**: ~10% — запрет «поддерживающей рекламы» без доказанной маржи, решение: изменить/вывести

### Каналы

- WB ~85% выручки, OZON ~15%
- % изменения OZON на малой базе завышены — учитывать при анализе
- Если каналы расходятся (один растёт, другой падает) — анализировать отдельно

## 2. Пять рычагов маржи

Если маржа ниже плановой — искать причину ТОЛЬКО по 5 рычагам:

### 1. Цена до СПП (управляемый)

Наиважнейший инструмент управления. Цена не имеет права ухудшать маржу (кроме стратегии Growth). После изменения цены: на следующий день проверять заказы (₽) и ДРР.

### 2. СПП % (внешний фактор)

Скидка постоянного покупателя. WB устанавливает СПП самостоятельно. Бренд не может влиять.
- **Рост СПП** = снижение цены для покупателя → рост спроса. СПП — скидка за счёт МП, она **не снижает** нашу выручку на единицу.
- **Снижение СПП** = рост цены для покупателя → снижение спроса.
- Нормативы: WB ~30-35%, OZON ~38-45%.

### 3. ДРР (управляемый, с разбивкой)

Доля рекламных расходов. ВСЕГДА разбивать:
- **Внутренняя реклама** (МП): WB promotion, OZON performance
- **Внешняя реклама**: блогеры, VK, Yandex, креаторы

Без разбивки невозможно определить причину роста ДРР.

### 4. Логистика ₽/ед (управляемый)

«Тихий убийца» маржи. Управляется через индекс локализации WB, срок доставки OZON. За оптимизацию отвечает агент Vasily.

### 5. Себестоимость ₽/ед (частично управляемый)

Стабильна (~350 руб). Резкое изменение = аномалия, требующая немедленного расследования.

## 3. Жёсткие правила аналитики

### 3.1 GROUP BY model = LOWER()

```sql
GROUP BY LOWER(SPLIT_PART(article, '/', 1))
```

В БД артикулы с разным регистром ("wendy" и "Wendy"). Без LOWER() они попадут в разные группы и исказят суммы на десятки процентов. Все data_layer функции уже применяют LOWER().

### 3.2 Процентные метрики — ТОЛЬКО средневзвешенные

При объединении каналов (WB+OZON) или агрегации по моделям:

```
ПРАВИЛЬНО: sum(spp_amount) / sum(revenue) * 100
НЕПРАВИЛЬНО: (spp_wb% + spp_ozon%) / 2
НЕПРАВИЛЬНО: spp_wb% (присвоить 0 для OZON)
```

### 3.3 ДРР — ВСЕГДА с разбивкой

- **Внутренняя**: WB `adv_internal` (из `reclama`), OZON `adv_internal` (из `reclama_end`)
- **Внешняя**: WB `adv_external` (из `reclama_vn + reclama_vn_vk`), OZON `adv_external` (из `adv_vn + adv_vn_vk`)

### 3.4 Единая маржа

Single margin = включает ВСЮ рекламу. Нет M-1/M-2 в отчётах.

### 3.5 Формула маржи WB

```
WB margin = SUM(marga) - SUM(nds) - SUM(reclama_vn) - COALESCE(SUM(reclama_vn_vk), 0) - COALESCE(SUM(reclama_vn_creators), 0)
```

Поля `reclama_vn_vk` и `reclama_vn_creators` могут быть NULL — обязательно COALESCE.

**Как считается `marga` в БД:**
`marga = (revenue_spp - revenue_return_spp) - (sebes - sebes_return) - comis_spp - logist - storage - reklama - retention - penalty`

Поле `marga` уже учитывает все возвраты и операционные расходы, КРОМЕ: `nds`, `reclama_vn` и `deduction`.

**ЗАПРЕЩЕНО:** 11-полевая формула (`revenue_spp - comis_spp - logist - ...`) — завышает маржу на ~2.5%.

### 3.6 Формула маржи OZON

```
OZON margin = SUM(marga) - SUM(nds)
```

На OZON реклама учтена в marga. Отдельно не вычитать.

### 3.7 Органика + Реклама = НИКОГДА не суммировать

Органические и рекламные заказы перекрываются. Суммирование даёт завышенный итог. Показывать раздельно.
- `card_opens` и `clicks` — сопоставимые метрики (переходы), можно показывать доли
- `impressions` (рекламные показы) — отдельная метрика, НЕ складывать с органикой

### 3.8 Числа только из данных

Никогда не генерировать числа. Каждое значение = результат SQL-запроса через data_layer. Если данных нет — писать "н/д".

### 3.9 Факт vs Гипотеза

Чётко разделять:
- **Факт**: "маржинальность снизилась на 3.2 п.п."
- **Гипотеза**: "вероятная причина — рост логистики из-за сезонного пика"

### 3.10 Формат чисел

- Целые: `1 234 567 ₽` (пробел-разделитель тысяч)
- Проценты: `24.1%`
- Изменения п.п.: `8.0% → 11.2% (+3.2 п.п.)`
- Тысячи: `36 482 тыс.₽`
- Штуки: `1 234 шт`
- Дни: `45 дн`

### 3.11 Формат дат

- В тексте: `17 марта 2026`
- В таблицах: `17.03.2026`
- ЗАПРЕЩЕНО: `2026-03-17` (ISO формат), `17/03/2026`

### 3.12 Производные формулы (верифицированы vs OneScreen)

```
Маржинальность% = Маржа / Выручка_до_СПП * 100
ДРР_WB% = (reclama + reclama_vn) / (revenue_spp - revenue_return_spp) * 100
ДРР_OZON% = (reclama_end + adv_vn) / price_end * 100
СПП_WB% = SUM(spp) / SUM(revenue_spp) * 100
СПП_OZON% = SUM(spp) / SUM(price_end) * 100
Выкуп% = Продажи_шт / Заказы_шт * 100
Выручка_WB = SUM(revenue_spp) - COALESCE(SUM(revenue_return_spp), 0)
Выручка_OZON = SUM(price_end)
Продажи_WB = SUM(full_counts)   -- GROSS, включает возвраты
Продажи_OZON = SUM(count_end)   -- уже NET
```

### 3.13 Сведение изменения маржи (Reconciliation)

Сумма влияний всех факторов (в рублях) должна соответствовать итоговому изменению маржи.

| Статья / Фактор | Текущий | Прошлый | Δ | Влияние на маржу |
|---|---|---|---|:---:|
| Выручка (денежные) | ₽ | ₽ | ₽ | +/- |
| Себестоимость / ед | ₽/ед | ₽/ед | ₽/ед | - |
| Комиссия до СПП (%) | % | % | п.п. | - |
| Логистика / ед | ₽/ед | ₽/ед | ₽/ед | - |
| Хранение / ед | ₽/ед | ₽/ед | ₽/ед | - |
| Внутренняя реклама | ₽ | ₽ | ₽ | - |
| Внешняя реклама | ₽ | ₽ | ₽ | - |
| Прочие расходы МП | ₽ | ₽ | ₽ | - |
| НДС | ₽ | ₽ | ₽ | - |
| Невязка (Residual) | — | — | ₽ | 0 |

**Правило невязки:** `Невязка = Фактическая ΔМаржи - сумма всех факторов`. Если невязка > 5% от общего изменения маржи — обязательно объяснение причины.

## 4. Особенности WB

### Склейки (model_osnova)

Все данные из `get_model_breakdown()` уже на уровне `model_osnova`. Маппинг выполнен на стороне python-бэкенда. Запрещено группировать вручную по префиксам.

### content_analysis: расхождение ~20% с PowerBI

Данные трафика из `content_analysis` на ~20% ниже, чем PowerBI. Причина: неизвестные фильтры на стороне PowerBI. Указывать в дисклеймере каждого отчёта.

### Fan-out JOIN

При JOIN abc_date с orders/sales по баркоду — один nmid = несколько строк. Всегда: `SELECT DISTINCT nmid, vendorcode FROM nomenclature` или агрегировать до JOIN.

### retention == deduction (дубли)

В WB поля `retention` и `deduction` часто дублируют друг друга. Не суммировать — использовать только одно.

### СПП динамика

- **При росте СПП (стратегия A — Маржа):** Поднимать базовую цену (забирать маржу себе)
- **При росте СПП (стратегия B — Growth):** Оставить базовую цену (цена для клиента падает → захват рынка)
- **При снижении СПП:** Рассмотреть снижение базовой цены, чтобы удержать цену для клиента

### Структура комиссии WB

| Статья | Целевой уровень | Управляемость |
|---|---|---|
| Комиссия до СПП | 35-40% | НЕТ (МП решает) |
| Логистика | < 8% | ДА (индекс локализации) |
| Хранение | < 2% | ДА (остатки, оборачиваемость) |
| ДРР | 5-10% внутр., до 15-18% с внешней | ДА |
| Себестоимость | 15-20% | Частично |
| Прочие | < 1% | Частично |
| **Целевая маржинальность** | **20-25%** | — |

### Воронка WB

```
Показы (ad_views) → Переходы (card_opens) → Корзина (add_to_cart) → Заказы → Выкупы
```

Рекламная воронка:
```
Показы (views) → Клики (clicks) → Корзины (atbs) → Заказы (orders)
        CTR            CPC/CPM          CPL             CPO/CR
```

## 5. Особенности OZON

### Органика недоступна

OZON не предоставляет данные об органическом трафике через API. Расчётная органика: `organic_orders = total_orders - ad_orders` (cap на 0 при отрицательных). Маркировать как "оценка".

### 34 типа комиссий

OZON имеет сложную структуру комиссий. Целевой уровень комиссии до СПП: 42-47%.

### Реклама включена в marga

На OZON реклама учтена в поле `marga`. Формула: `Маржа_OZON = SUM(marga) - SUM(nds)`. Рекламу отдельно не вычитать.

### Выручка и продажи уже NET

```
Выручка_OZON = SUM(price_end)   -- уже без возвратов
Продажи_OZON = SUM(count_end)   -- уже без возвратов
```

### Ограничения данных OZON

- Нет органической воронки (переходы/корзины/конверсии)
- CPL недоступен (нет atbs)
- Рекламная статистика по моделям ограничена
- Уникальные посетители пока недоступны

## 6. Лаговые показатели

### Выкуп % — лаг 3-21 дней

- **Дневные отчёты:** НЕ показывать выкупы, НЕ анализировать падение выкупов
- **Недельные отчёты:** Анализировать с оговоркой: "данные по выкупам могут быть неполными"
- **Месячные отчёты:** Полностью достоверный показатель
- Сравнение выкупов между периодами корректно только с лагом 3+ недели
- Для текущих оценок использовать скользящее среднее за 21+ дней

### Блогеры — лаг 3-7 дней

Эффект отложенный. Отсутствие заказов в день оплаты — норма. Нельзя считать ДРР в день размещения. Нельзя считать ROMI в первые 3-5 дней.

### Цена заказов vs Цена продаж — прогноз

- Средняя цена заказов: `orders_rub / orders_count` — прогноз будущих продаж (лаг 3-7 дней)
- Средняя цена продаж: `revenue_before_spp / sales_count` — фактическая выручка сегодня
- Если цена заказов > цена продаж на > 5% → выручка вырастет через 3-7 дней
- Если цена заказов < цена продаж на > 5% → выручка снизится

### Возвраты

Лаговый показатель. Оценивать на периодах 7+ дней.

### Заказы vs Выручка

- **Заказы (сумма)** — показатель текущего спроса (фонд будущей выручки)
- **Выручка** — результат прошлых периодов (выкупы ранее сделанных заказов)
- Нельзя оценивать эффективность текущей рекламы по текущей выручке. Рекламу "сегодня" сравниваем с заказами "сегодня".

## 7. Ценовая стратегия

### Реакция на изменение СПП

**При росте СПП:**
| Стратегия | Действие | Результат |
|---|---|---|
| A — Маржа (для A-группы) | Поднять базовую цену | Забрать маржу себе |
| B — Growth | Оставить базовую цену | Цена для клиента падает → захват рынка |

**При снижении СПП:**
Рассмотреть снижение базовой цены, чтобы удержать цену для клиента (избежать обвала спроса).

### Правила изменения цен

- Не более 1-2 изменений цены в неделю на SKU
- Запрет снижать цену автоматически при падении заказов — сначала проверить видимость/реклама/остатки/конверсии
- После изменения цены — на следующий день проверять заказы (₽) и ДРР
- Минимальная маржинальность для рекомендации: 20%
- Максимальная допустимая потеря объёма: 10%

### Стратегия атаки (Growth)

- **Условие:** маржинальность модели > 30% + низкий/падающий спрос
- **Действие:** тестировать повышение ставки или снижать цену для стимулирования спроса (пока маржинальность позволяет)

### Никогда не снижать цену при OOS

Если запас < 2 недель — запрет снижения цены. Снижение цены при дефиците = сожжённый товар без прибыли.

### Интерпретация эластичности

| Значение β | Интерпретация | Действие |
|---|---|---|
| \|β\| < 0.5 | Высоко неэластичный | Повышение цены безопасно |
| 0.5 ≤ \|β\| < 1.0 | Неэластичный | Умеренное повышение |
| \|β\| ≈ 1.0 | Единичная эластичность | Осторожно — выручка не изменится |
| 1.0 < \|β\| < 2.0 | Эластичный | Повышение рискованно |
| \|β\| ≥ 2.0 | Высоко эластичный | Снижение цены может быть выгодным |

### Правила участия в акциях МП

- Участвовать если: `(новая_маржа × ожид_объём) > (текущая_маржа × текущий_объём)`
- Средний подъём объёма: WB ~30%, OZON ~25%
- Скидка не должна обнулять маржу — проверять маржу/шт при акционной цене
- Переизбыток → участвовать (ускорить оборот)
- Дефицит → пропустить (нет товара)
- Маржинальность < 15% → пропустить (убыточно)

## 8. ДРР и реклама

### Целевые уровни ДРР

#### Внутренняя реклама (МП)

| Канал | Цель ДРР | Максимум ДРР |
|---|---|---|
| WB | 5-8% | 12% |
| OZON | 8-12% | 18% |

#### Внешняя реклама

| Канал | Цель ДРР | Максимум ДРР | Окно атрибуции |
|---|---|---|---|
| Блогеры | 15-25% | 35% | 7-14 дней |
| VK | 10-18% | 25% | 3-7 дней |
| Креаторы | 12-20% | 30% | 7-14 дней |

#### Суммарный ДРР

| Уровень | Без внешней | С внешней |
|---|---|---|
| Отлично | < 8% | < 12% |
| Норма | 8-12% | 12-18% |
| Требует внимания | 12-18% | 18-25% |
| Критично | > 18% | > 25% |

> При наличии внешней рекламы суммарный ДРР естественно выше. НЕ использовать пороги «без внешней» для периодов с внешней рекламой.

### Двойной KPI рекламы

По каждой модели/кампании оценивать ДВА показателя:
1. **ДРР продаж** = реклама / выручка продаж × 100 — влияет на текущую маржу
2. **ДРР заказов (CPO)** = реклама / выручка заказов × 100 — отражает будущую эффективность

### Матрица решений по двойному KPI

| ДРР продаж | ДРР заказов | Ситуация | Действие |
|---|---|---|---|
| Высокий | В норме | Инвестиция — заказы есть, выкупы придут | НЕ сокращать рекламу |
| В норме | В норме | Эффективная реклама | Поддерживать/масштабировать |
| Любой | Высокий | Реклама неэффективна | Сокращать бюджет |
| Высокий | Высокий | Критическая неэффективность | Остановить, диагностика |

**Запрещено:** рекомендовать сокращение рекламы ТОЛЬКО на основании высокого ДРР продаж, если ДРР заказов в норме.

### Управление внутренней рекламой

- Оценивается по заказам в тот же день
- Решение об остановке/снижении: при отклонении CPO > 50% от среднего за 7 дней
- Если расход есть, а заказов нет — кампания неэффективна

**Если растёт ДРР и CPL — остановить РК для оценки:**
1. Проверить CR, CTR
2. Позиции по важным ключам
3. В каких ключах снизились конверсии
4. Остатки и распределение на ключевые склады
5. Отзывы на негатив
6. Конкуренты и категория
7. Если всё в норме — пересмотреть ассортимент на продвижение

## 9. Причинно-следственные паттерны

### Паттерн 1: Трафик↓ + ДРР↓ = потеря эффективного трафика

- **Условие:** card_opens упали > 10% И ДРР (внутренний) снизился одновременно
- **Интерпретация:** трафик был ЭФФЕКТИВНЫМ. Его потеря = потеря объёма, НЕ оптимизация. Снижение ДРР — не повод для радости.
- **Действие:** срочно вернуть рекламные активности на просевшие модели
- **Формулировка:** «Переходы упали на X% (с A до B). ДРР снизился с C% до D%. Трафик был эффективным — это потеря объёма. Вернуть рекламу на [модели].»

### Паттерн 2: Реклама↓ → Показы↓ → Переходы↓ → Заказы↓ (trade-off)

- Маржа/ед растёт, но объём падает — указывать ОБА эффекта
- НЕЛЬЗЯ рекомендовать «сокращать рекламу ещё» если трафик уже падает
- Правильная рекомендация: «вернуть точечную рекламу на прибыльные модели с просевшими заказами»

### Паттерн 3: Блогер → Всплеск расхода → Отложенный ROI 3-7 дней

- Несистемный расход, нельзя считать ДРР в день размещения
- Если ДРР заказов в норме (<30%) и заказы выросли — это УСПЕХ, даже если маржа просела в моменте
- Правильное действие: «Масштабировать: повторить у этого блогера через 2 недели» или «Найти похожих блогеров (look-alike)»

### Паттерн 4: Связка Реклама — Трафик — Заказы

- Реклама выросла → заказы растут = эффективна (даже если ДРР вырос)
- Реклама выросла → заказы не растут = неэффективна, оптимизировать ставки
- Заказы выросли сегодня → выкупы подтянутся через 3-7 дней

### Паттерн 5: Качество трафика

- Рекламный расход растёт, Cart CR растёт, Order CR падает → трафик целевой, но есть барьер к покупке
- Вероятные причины: цена выше рынка, плохие отзывы, долгая доставка
- Действие: проверить цену и контент, НЕ отключать рекламу сразу

## 10. Маркетинговые бенчмарки

### Рекламная воронка

| Метрика | Формула | Что показывает | Бенчмарк |
|---|---|---|---|
| CTR % | clicks / views × 100 | Привлекательность объявления | < 1% нецелевой, 2-3% средний, 5%+ целевой |
| CPC ₽ | spend / clicks | Стоимость клика | < 5₽ отлично, 5-10₽ норма, > 15₽ дорого |
| CPM ₽ | spend / views × 1000 | Стоимость 1000 показов | < 100₽ хорошо, 100-300₽ норма, > 500₽ дорого |
| CPL ₽ | spend / atbs | Стоимость добавления в корзину | WB only |
| CPO ₽ | spend / orders | Стоимость заказа через рекламу | Сравнивать с маржой/ед |
| CR % | orders / clicks × 100 | Полная конверсия (клик→заказ) | < 1% плохо, 2-5% норма, > 5% отлично |

### Правила интерпретации

1. **CPO > маржа/ед** → реклама убыточна на модели. Снизить ставки или остановить
2. **CTR < 1%** при стабильных показах → проблема карточки/ключевых слов, НЕ ставок
3. **CTR падает, CPC растёт** → конкуренция за позиции выросла
4. **CPM растёт при стабильном CTR** → аукцион перегрет → рассмотреть смену стратегии
5. **Cart conversion (atbs/clicks) < 5%** → трафик нецелевой, карточка не убеждает
6. **Органика: card_to_cart < 5%** → карточка не конвертирует → проверить фото/отзывы/цену

### Маркетинговые формулы

| Метрика | Формула |
|---|---|
| ДРР (внутр) | `расход_МП / выручка × 100%` |
| ДРР (внешн) | `расход_внешн / выручка × 100%` |
| ДРР (общий) | `(расход_МП + расход_внешн) / выручка × 100%` |
| CPO | `расход_рекламы / кол-во_заказов` |
| ROMI | `(маржа_от_рекламы - расход) / расход × 100%` |
| CTR | `клики / показы × 100%` |
| CR корзина | `добавления_в_корзину / клики × 100%` |
| CR заказ | `заказы / корзина × 100%` |
| Выкуп % | `выкупы / заказы × 100%` (лаг 3-21 день) |
| Органика % | `заказы_без_рекламы / заказы_всего × 100%` |
| Прибыль/продажу | `маржа / количество_продаж` |
| CPS | `расход_на_рекламу / кол-во_продаж` |
| CAC | `расход_на_рекламу / переходы` |

## 11. Воронка продаж

### Словарь метрик

| Системное имя | Русское название | Описание |
|---|---|---|
| openCardCount | Переходы | Открытия карточки из поиска/каталога |
| addToCartCount | Корзина | Добавления в корзину |
| ordersCount | Заказы | Оформленные заказы |
| buyoutsCount | Выкупы | Фактически выкупленные заказы |
| CR open→cart | Конверсия переход→корзина | корзина / переходы |
| CR cart→order | Конверсия корзина→заказ | заказы / корзина |
| CRO (open→order) | Конверсия переход→заказ | заказы / переходы (СКВОЗНАЯ — ГЛАВНАЯ) |
| CRP (open→buyout) | Конверсия переход→выкуп | выкупы / переходы (ИТОГОВАЯ) |

**CRO (конверсия переход→заказ)** — ГЛАВНАЯ метрика. Показывает итоговую эффективность карточки.

### Стадии воронки и что на них влияет

| Этап | Что влияет |
|---|---|
| Показы → Переходы | Позиция в поиске, качество главного фото, цена, рейтинг, отзывы |
| Переходы → Корзина | Фото карточки, описание, размерный ряд, цена vs конкуренты |
| Корзина → Заказ | Цена, наличие размеров, сроки доставки, акции |
| Заказ → Выкуп | Качество товара, соответствие ожиданиям, упаковка |

### Бенчмарки WB (ориентиры, категория: нижнее бельё)

| Шаг воронки | CR норма | Красная зона |
|---|---|---|
| Показы → Клики (CTR) | 3-7% | < 2% |
| Клики → Корзина | 15-25% | < 10% |
| Корзина → Заказы | 25-40% | < 20% |
| Заказы → Выкупы | 55-75% | < 45% |

### Бенчмарки OZON (ориентиры)

| Шаг воронки | CR норма | Красная зона |
|---|---|---|
| Показы → Клики (CTR) | 2-5% | < 1.5% |
| Клики → Корзина | 10-20% | < 7% |
| Корзина → Заказы | 20-35% | < 15% |
| Заказы → Выкупы | 40-65% | < 30% |

### Бенчмарки сквозной конверсии

| Метрика | Проблема | Норма | Отлично |
|---|---|---|---|
| Переход→корзина | < 5% | 5-15% | > 15% |
| Корзина→заказ | < 20% | 20-40% | > 40% |
| CRO (переход→заказ) | < 1% | 1-3% | > 3% |
| CRP (переход→выкуп) | < 0.5% | 0.5-2% | > 2% |

### Формулы

```
CRO (переход→заказ) = заказы / переходы × 100
CRP (переход→выкуп) = выкупы / переходы × 100
ДРР = расход_на_рекламу / выручка × 100
ROMI = маржа / расход_на_рекламу × 100
Прибыль_на_продажу = маржа / количество_продаж
Прибыль_на_посетителя = маржа / переходы
```

### Связь воронки и финансов

- CRO↑ → больше заказов при тех же переходах → ДРР↓ → маржа↑
- Органика↑ → меньше зависимость от рекламы → ДРР↓ → маржа↑

## 12. Конверсия корзина→заказ

### Обязательная метрика в каждом отчёте

| Канал | Норма | Красная зона |
|---|---|---|
| WB | 25-40% | < 20% |
| OZON | 20-35% | < 15% |

### Правила

- ОБЯЗАТЕЛЬНО включать в каждый отчёт (дневной, недельный, месячный)
- Падение > 5 п.п. WoW → **приоритетное расследование**: цена, наличие размеров, сроки доставки, акции
- Перекрёстная проверка: если цена выросла и Cart-to-Buy упал → связь очевидна
- При объединении каналов — ТОЛЬКО средневзвешенный

### Протокол расследования при падении

1. Проверить изменение цены за последние 3 дня
2. Проверить наличие ключевых размеров (S/M/L)
3. Проверить сроки доставки
4. Проверить активные акции и скидки
5. Сравнить с конкурентами

## 13. Матрица эффективности рекламы

### Категоризация моделей по стратегии

| Категория | Критерий | Стратегия рекламы |
|---|---|---|
| **Growth** | Маржинальность > 25%, заказы растут | Увеличивать бюджет (ДРР до максимума) |
| **Harvest** | Маржинальность > 20%, заказы стабильны | Поддерживать текущий бюджет |
| **Optimize** | Маржинальность 10-20% | Снижать ДРР, искать эффективные ключи |
| **Cut** | Маржинальность < 10% или ДРР > 25% без роста | Отключить рекламу, только органика |

### Правила переключения

- Модель в Growth минимум **7 дней** перед переоценкой
- Перевод в Cut только после **14 дней** неэффективности
- При переводе из Cut обратно — начинать с минимального бюджета

### Матрица решений по ROMI

| ROMI | Статус | Рекомендация |
|---|---|---|
| < 50% | Критический убыток | **СТОП** — отключить рекламу, только органика |
| 50-100% | Убыточная реклама | **Сокращение** — снизить ставки, оптимизировать ключи |
| 100-200% | Норма | Поддерживать текущий бюджет |
| > 200% | Высокая эффективность | **Масштабирование** — увеличить бюджет |

### Исключения

- Новые модели (< 4 недель на МП) — ROMI может быть низким, это инвестиция (отмечать как «инвестиционный период»)
- Модели в статусе Growth — допустим ROMI 80-100% если заказы растут > 20% WoW

## 14. Внешняя реклама

### Матрица системности каналов

| Канал | Системность | Лаг ROI | Анализ Day-to-Day |
|---|---|---|---|
| **Блогеры (Influence)** | Разовый (всплески) | 3-7 дней | **ЗАПРЕЩЕН** |
| **VK / Яндекс (Target)** | Системный (ежедневно) | 1-3 дня | **РАЗРЕШЕН** |
| **Посевы (Publics)** | Системный | 2-5 дней | Ограничен |

### Окна атрибуции

| Канал | Окно | Метод |
|---|---|---|
| Блогеры | 7-14 дней | Рост заказов по модели vs базовая линия (среднее за 7 дней до) |
| VK | 3-7 дней | UTM-метки или корреляция расход ↔ рост заказов |
| Креаторы (UGC) | 7-14 дней | Аналогично блогерам, контент работает дольше (evergreen) |

### Общие правила атрибуции

- Рост органических заказов в период внешней рекламы частично атрибутируется к внешнему каналу
- При одновременных кампаниях — делить эффект пропорционально бюджету
- Минимальный бюджет для оценки блогера: 30 000 ₽

### Двойной KPI внешней рекламы

Внешняя реклама оценивается по ДВУМ метрикам одновременно:
1. **ДРР продаж** = расход / выручка продаж × 100
2. **ДРР заказов** = расход / выручка заказов × 100

**Запрещено:** рекомендовать сокращение внешней рекламы ТОЛЬКО на основании высокого ДРР продаж, если ДРР заказов в норме.

### Запрещённые формулировки

- «Внешняя реклама остановлена/прекращена» → «Внешней рекламы сегодня не было»
- «Внешняя реклама выросла в X раз» при сравнении дня с блогером vs без → «Размещение у блогера X руб.»
- «Экономия на внешней рекламе +X тыс» → нельзя считать экономией отсутствие несистемного расхода
- «ДРР внешней рекламы 45%» в день размещения блогера → «Размещение X руб. ДРР считать рано»
- «Внешняя реклама не конвертируется» в день запуска → нельзя без анализа лага 5-7 дней
- «Остановить рекламу» для блогеров → бессмысленно (размещение уже прошло)

## 15. Юнит-экономика

### Годовой ROI

**Формула:** `annual_roi = margin_pct × (365 / turnover_days)`

Высокая маржинальность при низкой оборачиваемости = деньги заморожены в товаре.

| Категория | annual_roi | Интерпретация |
|---|---|---|
| roi_leader | > 500% | Отличная модель: высокая маржинальность И быстрый оборот |
| healthy | 200-500% | Норма: стабильный бизнес |
| underperformer | 50-200% | Проблема: маржинальность ОК, но оборот медленный |
| deadstock_risk | < 50% | Критично: деньги заморожены, срочные меры |

### Юнит-экономика артикулов

В каждом недельном и месячном отчёте обязательно:
- Для каждой A/B модели: **ТОП-3 артикула по прибыли** и **BOTTOM-3 по маржинальности**
- Если ROMI < 100% → пометить как убыточный по рекламе
- Если profit_per_sale < 0 → критический алерт: артикул генерирует убыток

### Целевые показатели по статусу

| Статус | Обеспеченность | Ценовое ограничение |
|---|---|---|
| critical_low | < 1 недели | Запрет снижения цены, risk алерт |
| low | < 2 недель | Запрет снижения цены |
| healthy | 2-8 недель | Нет ограничений |
| overstocked | > 8 недель | Предпочтение снижению цены |
| severely_overstocked | > 16 недель | Принудительное снижение цены |

### Себестоимость (baseline)

- Стабильна ~350 руб/ед
- Целевая доля от выручки: 15-20%
- Резкое изменение = аномалия

### profit_per_sale

- **CAC > profit_per_sale** → реклама уничтожает маржу. Снизить ставки или отключить
- **profit_per_sale < 0** → артикул генерирует убыток. Критический алерт

## 16. Остатки и оборачиваемость

### 5 пороговых уровней

| Дней остатка | Статус | Действие |
|---|---|---|
| < 14 дн | ДЕФИЦИТ | Срочная поставка, повысить цену если маржа позволяет |
| 14-60 дн | НОРМА | Штатный режим |
| 60-90 дн | ВНИМАНИЕ | Снизить закупку, мониторить |
| 90-250 дн | ЗАТОВАРИВАНИЕ | Скидка, акция, перемещение на другие склады |
| > 250 дн | МЁРТВЫЙ СТОК | Глубокая распродажа, вывод с площадки |

### Ценовые ограничения по уровню запасов

| Условие | Действие |
|---|---|
| ЗАТОВАРИВАНИЕ + маржа > 15% | Снизить цену для ускорения оборота |
| ДЕФИЦИТ | Повысить цену (если эластичность позволяет) |
| Маржа < 15% | НЕ снижать цену, искать другие рычаги |
| МЁРТВЫЙ СТОК | Глубокая уценка, вплоть до себестоимости |
| Высокий СПП (> 30%) | Рассмотреть повышение базовой цены |

### МойСклад: данные об остатках

`get_model_breakdown` возвращает:
- `avg_stock_fbo` — остатки на складах МП (FBO)
- `stock_own_warehouse` — остатки на собственном складе (МойСклад)
- `stock_transit` — товары в пути
- `stock_total` — сумма всех
- `turnover_days` — оборачиваемость по FBO
- `turnover_days_full` — оборачиваемость по ВСЕМ остаткам

**Интерпретация:**
- Высокий stock_own_warehouse + низкий FBO → нужна поставка на МП
- Высокий turnover_days_full при низком turnover_days → замороженный капитал на своём складе

## 17. Ценовая логика (decision tree)

### 5 правил

| # | Условие | Действие |
|---|---|---|
| 1 | ЗАТОВАРИВАНИЕ + маржа > 15% | Снизить цену для ускорения оборота |
| 2 | ДЕФИЦИТ | Повысить цену (если эластичность позволяет) |
| 3 | Маржа < 15% | НЕ снижать цену, искать другие рычаги (реклама, логистика) |
| 4 | МЁРТВЫЙ СТОК | Глубокая уценка, вплоть до себестоимости |
| 5 | Высокий СПП (> 30%) | Рассмотреть повышение базовой цены |

### Правило переопределения

Рекомендация из эластичности переоценивается через stock constraints. Если эластичность говорит "повышай", но сток severely_overstocked — переопределяется на "снижай" с объяснением.

### Кросс-модельные эффекты

Модели внутри одной линейки конкурируют за покупателя:
- **Бесшовная линейка:** wendy, audrey, bella
- **Трикотажная линейка:** ruby, set_vuki, vuki

При повышении цены wendy часть спроса может уйти к audrey — каннибализация.

## 18. Диагностика аномалий

### Триггеры аномалий

| Триггер | Порог |
|---|---|
| Изменение маржинальности | > 10% или > 2 п.п. |
| Изменение ДРР | > 30% или > 3 п.п. |
| Падение CRO | > 0.5 п.п. |
| Изменение трафика (переходы) | > 20% |
| Изменение заказов модели | > 10% |

### Последовательность drill-down

1. Бренд: маржа изменилась → какой МП (WB/OZON)?
2. МП: маржа WB изменилась → какая model_osnova?
3. Model_osnova: просадка → какая модель?
4. Модель: просадка → какой артикул?

### Диагностическая таблица

| Симптом | Что проверить |
|---|---|
| CRO упал > 0.5 п.п. | Цена, размерный ряд, отзывы, конкуренты, позиции по ключевым |
| Переходы упали > 20% | Позиции по ключевым, рекламный бюджет, сезонность |
| ДРР вырос > 3 п.п. | Стоимость клика, конверсия из рекламы, конкуренция |
| Маржа упала при стабильной выручке | Логистика, хранение, штрафы, возвраты |
| Доля органики упала > 10 п.п. | Позиции, новые конкуренты |
| Выкупы упали при стабильных заказах | Качество товара, размерная сетка |
| CAC > profit_per_sale | Реклама уничтожает маржу. Снизить ставки или отключить |
| ROMI < 100% | Убыточная реклама. СТОП (< 50%) / Сокращение (50-100%) |
| Заказы растут, выручка нет | Средний чек падает (скидки, СПП, ассортиментный сдвиг) |
| Выручка растёт, маржа падает | Рост логистики/хранения, рост рекламы, снижение цены |
| ДРР растёт, заказы стабильны | Неэффективная реклама — Cut/Optimize по матрице |
| Остатки растут при стабильных продажах | Избыточная поставка, сезонный спад ожидается |
| Маржинальность модели < 15% | Проверить: цену, себестоимость, логистику, хранение, рекламу |
| Показы упали, остальное стабильно | SEO-позиции, изменение ключевых слов, WB-алгоритм |
| CTR упал при стабильных показах | Фото карточки, позиция в выдаче, новые конкуренты |
| CR корзина упал | Цена vs конкуренты, отзывы, рейтинг, наличие |
| CR заказ упал при стабильной корзине | Доставка (сроки, доступность), проблемы с оплатой |

### Чек-лист диагностики при падении модели (> 15% маржи)

При просадке ключевой модели (wendy, ruby, set_vuki):
1. **Цена до СПП и после СПП** — изменения за последние 3 дня
2. **Наличие размеров (OOS)** — выпадение по ключевым размерам
3. **Органический трафик** — показы/переходы/конверсия (`content_analysis`)
4. **OZON отдельно** — оборачиваемость, хранение, остатки по цветам/размерам

**Дерево решений:**
- OOS по размерам → срочно пополнение/перераспределение между складами
- Цена ушла выше рынка → откат/тест
- Позиции в поиске просели → точечное усиление внутренней рекламы
- Отзывы ухудшились → реакция на негатив, проверка партии
- Ничего из вышеперечисленного → проверить категорию/конкурентов/сезонность

### Количественный диагноз воронки

1. **Верх воронки (Охват)**: Изменились ли показы? Связано с рекламными ставками?
2. **Середина воронки (Интерес)**: Изменился ли CTR или конверсия в корзину? Гипотеза: цена, отзывы, фото.
3. **Низ воронки (Результат)**: Изменилась ли конверсия из корзины в заказ? Гипотеза: сроки доставки, ПВЗ.

## 19. Data Quality и запрещённые формулировки

### Известные проблемы качества данных

| Проблема | Описание | Как работать |
|---|---|---|
| retention == deduction | Поля часто дублируют друг друга в WB | Не суммировать — использовать только одно |
| Fan-out JOIN | Один nmid = несколько строк при JOIN | `SELECT DISTINCT nmid, vendorcode` или агрегировать до JOIN |
| Трафик ~20% расхождение | content_analysis на ~20% ниже PowerBI | Указывать в дисклеймере |
| OZON органика недоступна | API не предоставляет органический трафик | `organic = total - ad` (оценка) |
| Неполные месяцы | Данные текущего месяца неполные | Указывать: "данные за N дней из M" |
| Разный регистр моделей | Дубли при группировке | `LOWER()` в каждом GROUP BY |
| СПП влияет на конверсию | Путаница в интерпретации | Отделять «цена для клиента» от «наша выручка» |
| Органика vs Платное | Разные метрики, нельзя складывать | card_opens и clicks (переходы) — сопоставимы; impressions — отдельно |
| NaN vs 0 | Разные бизнес-состояния | NaN = технический сбой, 0 = подтверждённое отсутствие |
| OZON выручка | Расхождение ~7% с OneScreen | Согласованное допустимое расхождение |

### Протокол верификации

- Маржа бренда: допустимое расхождение < 1%
- Выручка WB: допустимое расхождение < 1%
- Выручка OZON: допустимое расхождение ~7%
- Если расхождение превышает порог — остановить анализ, зафиксировать в DATA_QUALITY_NOTES.md

### Дисциплина уверенности

| Уровень | Критерий | Допустимые действия |
|---|---|---|
| HIGH | Данные за 7+ дней, тренд устойчивый, причина ясна | Рекомендации с конкретными цифрами |
| MEDIUM | Данные за 3-7 дней или причина не однозначна | Гипотезы с оговорками, "необходимо наблюдение" |
| LOW | < 3 дней данных, аномалия, внешний фактор | Только наблюдение, "рано делать выводы" |

Запрещено давать конкретные рекомендации при LOW confidence.

### Полный список запрещённых формулировок

**В отчётах никогда не использовать:**

1. «Необходимо проанализировать» — ты и есть аналитик, анализируй
2. «Рекомендуется обратить внимание» — конкретизируй: на что и почему
3. «Возможно, это связано с...» без данных — либо подтверди данными, либо маркируй как LOW confidence
4. «Оптимизировать процессы» — что именно оптимизировать?
5. «В целом, ситуация стабильная» — покажи метрики стабильности
6. «Существенный рост/падение» без цифры — насколько?
7. «Рекомендуется увеличить/уменьшить» без конкретного % или суммы
8. «Внешняя реклама остановлена/прекращена» → «Внешней рекламы сегодня не было»
9. «Экономия на внешней рекламе» → нельзя считать экономией отсутствие несистемного расхода
10. «ДРР внешней рекламы X%» в день размещения блогера → «Размещение X руб. ДРР считать рано»
11. «Остановить/отключить рекламу» без контекста → «ограничить бюджет / перезапустить кампании модели X»
12. Ссылки на «плейбук», «инструкции» или файлы — анализ выглядит как экспертное мнение
13. Англ. аббревиатуры флагов: QC, Coverage, Lag, Residual, N/A, OK, Stop, Check
14. Оценочные суждения: «к сожалению», «радует», «отличный результат!»
15. Проценты без единицы измерения и базы сравнения

**Словарь замен:**
- QC → Проверка данных
- Coverage → Покрытие данных
- Lag → Лаг
- Residual → Невязка
- N/A → Данных нет
- STOP → СТОП
- CHECK → Проверить

### Аналитические принципы

1. **Системное мышление**: каждое изменение влияет на всю цепочку (цена → заказы → выкуп → оборачиваемость → маржа)
2. **Evidence-based**: каждое утверждение подкреплено числом из data_layer
3. **Эффект в рублях**: каждая рекомендация с оценкой эффекта в ₽/мес
4. **Последствия 2-го порядка**: снижение цены → рост заказов, но и рост логистики, снижение маржинальности
5. **Тон**: сухой CFO. Без эмоций, без восклицательных знаков
