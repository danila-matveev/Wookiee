# Agent 1: UX/UI Critic — Review

## Общая оценка

**5.5 / 10**

Визуально приятный дашборд с хорошей цветовой палитрой и типографикой. Однако за красивым фасадом скрываются серьёзные проблемы: дашборд показывает статическую картинку, а не инструмент оператора. Нет ни одного способа взаимодействовать с данными — нет фильтров, поиска, drill-down, временных диапазонов. Это скорее "скриншот состояния", чем операционный инструмент. Для мокапа — хорошая стартовая точка, но до рабочего MVP нужно закрыть критические пробелы.

---

## P0 — Критические проблемы

### 1. Отсутствие drill-down и навигации по данным
Клик на строку агента ведёт в никуда. Клик на аномалию "-22% vs avg" не раскрывает деталей. Клик на failed pipeline не показывает лог ошибки. **Оператор не может расследовать проблему, не выходя из дашборда.** В Grafana/Datadog каждый элемент — это ссылка на детальное представление. Здесь — глухая стена.

### 2. Нет выбора временного диапазона
Дашборд показывает "24ч" для Cost и "7 дней" для Pipeline Timeline — и это жёстко захардкожено. Оператор не может посмотреть, что было вчера, на прошлой неделе, за последний час. **Это базовая функция любого мониторинга**, без неё дашборд бесполезен для расследования инцидентов.

### 3. Нет алертов и уведомлений
Аномалия "Revenue WB -22% vs avg" с пометкой "critical" показана только в Activity Feed — на одном уровне с обычными записями. Нет баннера, нет визуального выделения на уровне страницы, нет sound/notification. **Критический алерт не должен быть 3-й строкой в ленте, которую нужно скроллить.**

### 4. Хронологическая ошибка в Activity Feed
Записи идут: 09:15, 09:12, 09:10, 08:45, **08:30**, **08:32**. Событие в 08:32 стоит ПОСЛЕ 08:30 — хронология нарушена. Retry (08:32) произошёл после ошибки (08:30), но визуально стоит ниже. Это фактическая ошибка в данных мокапа, которая сбивает с толку.

### 5. Нет auto-refresh и индикации актуальности данных
Пометка "LIVE" есть, но нет индикации "последнее обновление X секунд назад". Если WebSocket отвалится — оператор не узнает, что смотрит на устаревшие данные. В Datadog есть явный таймер "Last updated: 5s ago" с визуальным предупреждением при задержке.

## P1 — Важные улучшения

### 6. Summary Cards: "Success Rate" без контекста
87.5% — это хорошо или плохо? За какой период? Какой target? Без SLO/SLA линии на спарклайне число бессмысленно. Должен быть threshold: зелёный >95%, жёлтый 90-95%, красный <90%. Сейчас карточка зелёная, но Success Rate на самом деле падает (-4.2%).

### 7. Summary Cards: тренд "+12%" для расхода помечен классом "down" (красный)
Семантика CSS-класса `card-trend down` используется для отображения роста расхода. Это путает — "down" с точки зрения кода, но визуально показывается как рост. Нужен отдельный класс `card-trend bad` для случаев, когда рост — это плохо.

### 8. Agent Fleet: колонка "Avg" не объяснена
Что такое "Avg"? Average execution time? За какой период? Заголовок колонки должен быть самодостаточным: "Avg Time" или "Avg Duration (24h)". Tooltip хотя бы.

### 9. Нет фильтрации и поиска
При 6 агентах это терпимо. При 20 — невозможно. Нет ни search bar в хедере, ни фильтров по статусу (show only errors), ни сортировки таблицы.

### 10. Pipeline Timeline: неэффективная визуализация
Текущий формат — строки с тегами — не масштабируется. При 10+ пайплайнах строка будет переполнена. GitHub-style heatmap или Gantt-chart были бы значительно лучше. Также нет информации о **длительности** каждого пайплайна — только факт запуска.

### 11. Микс языков
"Agent Fleet", "Live Activity", "Pipeline Timeline", "Circuit Breakers", "Cost Breakdown" — английский. "Активных агентов", "Отчётов сегодня", "Расход за сутки", "Мониторинг" — русский. "Runs", "Errors" рядом с "Посл. запуск". Нужно выбрать один язык и следовать ему. Для внутреннего инструмента русский язык предпочтительнее.

### 12. Circuit Breakers: нет действий
Validator OPEN — и что? Нет кнопки "Reset CB", нет ссылки на лог ошибок, нет информации о том, когда CB автоматически закроется (cooldown показан, но нет таймера обратного отсчёта).

## P2 — Nice to have

### 13. Noise texture overlay — визуальный мусор
`body::before` с fractalNoise на opacity 0.018 — декоративный элемент, который ничего не даёт для мониторинга, но добавляет GPU нагрузку при рендере. В инструменте оператора каждый пиксель должен нести информацию.

### 14. Ambient glow — то же самое
`body::after` с radial-gradient в правом верхнем углу. Чисто декоративный элемент.

### 15. Sparklines в summary cards
Идея хорошая, но реализация через div-бары слишком грубая. 7 столбиков не дают понимания тренда. Лучше SVG sparkline с 24-48 точками (как в Grafana stat panels).

### 16. Sidebar: нет бейджей с количеством ошибок
Пункт "Logs" не показывает, что есть непрочитанные ошибки. "Health" не показывает, что есть проблемы. В Datadog на навигации всегда видны counter badges с количеством active alerts.

## Конкретные предложения по каждому блоку

### Summary Cards

**Что хорошо:** компактные, чёткая типографика, sparklines дают направление тренда.

**Что исправить:**
- Добавить target/SLO линию на sparkline (горизонтальная линия на уровне threshold)
- "Активных агентов 5/6" — добавить кликабельность, чтобы отфильтровать неактивных
- Расход $0.42 — добавить "vs бюджет" или "vs вчера" для контекста
- Сделать карточку Success Rate красной/жёлтой при значении ниже threshold, а не зелёной
- Sparklines: заменить на SVG path для плавности

### Agent Fleet Table

**Что хорошо:** читаемая таблица, аватары помогают быстро идентифицировать агентов, status badges чёткие.

**Что исправить:**
- Добавить сортировку по колонкам (кликабельные заголовки)
- Строка Validator с CB Open должна визуально выделяться сильнее — фоновый цвет строки, не только badge
- Добавить колонку "Next Run" для scheduled агентов
- Добавить мини-индикатор тренда errors (стрелка вверх/вниз)
- "Avg" переименовать в "Avg Time"
- Клик на строку -> открывать detail panel/page для агента

### Live Activity Feed

**Что хорошо:** формат "Agent -> Task -> Result" читаемый, мета-информация (время, стоимость) полезна.

**Что исправить:**
- Исправить хронологию (08:30 и 08:32)
- Добавить группировку retry с оригинальным событием (Marketer error + retry = одна группа)
- Добавить фильтр по типу: All / Errors only / Warnings only
- Critical аномалии — выделять полностью строку фоном, а не только текстом
- Добавить "Show more" / бесконечный скролл
- Кликабельность каждого события -> детальный лог

### Pipeline Timeline

**Что хорошо:** компактное представление, цветовое кодирование.

**Что исправить:**
- Заменить на heatmap grid (строки = пайплайны, столбцы = дни, цвет ячейки = статус)
- Добавить tooltip с деталями при наведении на блок
- Показать длительность выполнения (ширина блока = время)
- Добавить возможность клика -> переход к логу конкретного запуска
- Сейчас "Вс 30" показывает mkt_daily красным — но в CB секции Marketer CLOSED. Несогласованность данных мокапа.

### Circuit Breakers

**Что хорошо:** grid layout, чёткое разделение CLOSED/OPEN, progress bar для failures.

**Что исправить:**
- Добавить кнопку "Force Reset" для OPEN breakers (с confirmation dialog)
- Cooldown "2m 30s" — сделать live countdown timer
- Healthy breakers (0/3) не требуют внимания — визуально уменьшить, выделить только проблемные
- Добавить "Half-Open" состояние (стандарт CB pattern)
- Показать timestamp последней ошибки

### Cost Breakdown

**Что хорошо:** горизонтальные бары с процентами, моноширинный шрифт для чисел.

**Что исправить:**
- Добавить тренд (стрелка вверх/вниз vs вчера)
- Добавить "daily budget" линию на барах
- $0.42 total — слишком мало информации. Добавить: week total, month projection
- Кликабельность -> переход на страницу Costs с детализацией по вызовам
- Показать количество LLM-вызовов, не только стоимость
- Разбивка по tier (LIGHT/MAIN/HEAVY) была бы полезнее, чем по агенту

### Sidebar / Navigation

**Что хорошо:** минималистичный, не отвлекает, иконки читаемые.

**Что исправить:**
- Добавить badges с count на "Logs" (errors count) и "Health" (alerts count)
- "Fleet" — зачем отдельная страница, если fleet table на overview? Объяснить разницу.
- Не хватает: "Alerts" / "Incidents" — отдельная страница для алертов
- Не хватает: "Schedules" — расписание запуска пайплайнов
- "Назад в Hub" — убрать из sidebar bottom, перенести в breadcrumb хедера
- Нет индикатора текущего environment (production/staging)

## Отсутствующие паттерны

### vs Grafana
- **Variable selectors** (выбор агента, периода, environment) — полностью отсутствуют
- **Annotations** (вертикальные линии на графиках при деплоях/инцидентах) — нет
- **Panel links** (каждая панель -> детальный dashboard) — нет
- **Time range picker** — нет
- **Auto-refresh selector** (5s / 10s / 30s / off) — нет
- **Template variables** (переключение между agents) — нет

### vs Datadog
- **Alert status widget** (dedicated panel с active alerts) — нет
- **Service map** (граф зависимостей между агентами) — нет
- **SLO widget** с burn rate — нет
- **Change events** (коррелирование деплоев с ошибками) — нет
- **Notebook/investigation mode** — нет
- **Monitor status badge** в навигации — нет

### vs Linear / Incident.io
- **Incident timeline** с structured data — Activity Feed слишком свободного формата
- **Assign / Acknowledge** actions на инцидентах — нет
- **Status page** integration — нет
- **Runbook links** при ошибках — нет

### Общие паттерны, которых не хватает
1. **Keyboard shortcuts** (j/k для навигации по feed, / для поиска)
2. **Command palette** (Cmd+K) для быстрого доступа к любому агенту/логу
3. **Breadcrumbs** для навигации при drill-down
4. **Toast notifications** для real-time алертов
5. **Bulk actions** (restart all failed, reset all CBs)
6. **Export** (скачать логи, экспорт в CSV)
7. **Comparison mode** (сравнить два периода рядом)
8. **Dark/light theme toggle** (не критично, но стандарт)

## Accessibility

### Контраст
- `--text-dim: #55556a` на `--bg-surface: #16161f` — contrast ratio ~2.8:1. **Не проходит WCAG AA** (требуется 4.5:1). Это используется повсюду: роли агентов, время в feed, секционные лейблы.
- `--text-secondary: #8888a0` на `--bg-surface: #16161f` — ~4.2:1. Пограничное значение, не проходит AA для мелкого текста.
- `--border-subtle: rgba(255,255,255,0.06)` — разделители практически невидимы. На некоторых мониторах таблица будет выглядеть как сплошной блок.

### Размеры шрифтов
- `10px` используется для: секционных лейблов sidebar, feed-meta-item, legend-item, cb-failures, cost-pct, stat-mini, timeline-block. **10px — ниже минимально рекомендуемого размера 12px.** Для мониторинга, где оператор может смотреть с расстояния — неприемлемо.
- `9px` для timeline-block внутри Pipeline Timeline — за гранью читаемости.

### Кликабельные области
- Nav items: padding `9px 12px` — минимальный touch target. Для мыши приемлемо, для touch — нет (рекомендуется 44x44px).
- Timeline blocks: высота 22px — слишком мало для клика, особенно с учётом gap 4px между ними.
- Нет focus-visible стилей ни на одном элементе. Tab-навигация невозможна.

### Keyboard navigation
- Все nav-item — это `<a href="#">`, но без `tabindex` и `aria-*` атрибутов.
- Таблица fleet — строки имеют `cursor: pointer`, но нет `role="button"` или обработчика keyboard events.
- Activity feed items не фокусируемы.
- **Полное отсутствие keyboard navigation** — критическая проблема accessibility.

### Screen readers
- Summary cards — нет `aria-label`. Screen reader прочитает "5 / 6" без контекста.
- Status badges — нет `aria-label` для "Running", "Idle", "CB Open".
- Live badge "LIVE" — чисто визуальный, нет `aria-live` region для обновлений feed.
- Иконки `<svg>` без `aria-hidden="true"` и без `<title>` — screen reader будет пытаться их интерпретировать.

### Color-blindness
- **Red/Green** (Deuteranopia, ~8% мужчин): status Running (blue) vs Idle (green) — OK. Но Success (green) vs Failed (red) в pipeline timeline — **проблема**. Оба будут выглядеть как оттенки коричневого/оливкового.
- Решение: добавить иконки/символы к цветовым индикаторам. В pipeline timeline уже есть символы в stats (check/cross), но сами блоки различаются только цветом.
- Sparklines — различаются только цветом, без дополнительных паттернов.
