# Папка `.hygiene/`

Тут живёт «мозг» ночного DevOps-агента — это тот, который каждую ночь проверяет репо, чинит мелочи сам и присылает короткое Telegram-сообщение про то, что нашёл.

## Что внутри простыми словами

| Файл / папка | Что это |
|---|---|
| `config.yaml` | Главные настройки. Тут пока стоит «режим тишины» (`read_only: true`) — агент работает в холостую: смотрит, считает, шлёт предпросмотр, но **ничего не правит и не открывает PR**. Снять режим тишины можно только вручную через PR. |
| `decisions.yaml` | Память агента. Тут хранятся все ответы, которые ты дал на вопросы агента. На следующую ночь он сначала читает память — «вот это уже решали, спрашивать не буду». |
| `queue.yaml` | Очередь вопросов. Если за ночь нашлось что-то спорное — агент кладёт сюда. Команда `/hygiene-resolve` берёт оттуда вопросы по одному и спрашивает тебя на простом русском. |
| `reports/` | JSON-отчёты за каждую ночь: что нашли, что починили, что в очереди. Лежат 30 дней, потом старые удаляются. Сама папка в git, но её содержимое (`*.json`) — gitignored. Внутри лежит пустой файл `.gitkeep`, чтобы папка вообще существовала в репо. |

## Как пользоваться

### Обычный режим (первая неделя)

Просыпаешься утром → в Telegram уже лежит короткое сообщение. Дальше:
- Если сообщения нет — значит, всё спокойно, агент ничего не нашёл.
- Если есть сообщение «1 вопрос» / «3 вопроса» — открой Claude Code и вставь команду `/hygiene-resolve`. Агент задаст вопросы голосом простого русского, ты ответишь, и память обновится.

### Что делать, если что-то идёт не так

1. Если приходят странные сообщения от агента — открой `config.yaml`, поставь `read_only: true` (если ещё не стоит) и сделай PR. С этого момента агент перестанет что-либо менять, останется только наблюдать.
2. Если хочешь полностью выключить ночные cron-задачи — пиши в чат «выключи ночного агента», я отключу нужные workflow в GitHub Actions.
3. Если очередь `queue.yaml` накопила больше 5 вопросов — это нормально, через 7 дней агент сам применит безопасные дефолты по каждому вопросу. Никаких неожиданностей не будет.

### Как поменять настройки

Все настройки — в `config.yaml`. Самые важные:

- `read_only: true` — режим тишины (так стартуем).
- `read_only: false` — реальный режим, агент сам открывает PR и мерджит, когда CI зелёный.
- `coverage_min_pct: 60` — если тесты покрывают код меньше чем на 60%, мердж блокируется. Подкрутить можно вверх/вниз.
- `queue_expire_days: 7` — сколько дней вопрос ждёт твоего ответа. После — применяется безопасный дефолт.
- `heartbeat_quiet_if_zero: true` — если за ночь ноль активности, Telegram молчит. Можно поставить false, тогда каждое утро будет хотя бы «всё спокойно».

Любое изменение `config.yaml` — через PR, никак иначе. Это специально: чтобы случайно не сломать.

## Технические детали

Полный план агента: [`docs/superpowers/plans/2026-05-14-nighttime-devops-agent-impl.md`](../docs/superpowers/plans/2026-05-14-nighttime-devops-agent-impl.md).

Код, который читает/пишет эту папку:
- `shared/hygiene/config.py` — загружает `config.yaml`
- `shared/hygiene/decisions.py` — читает/пишет `decisions.yaml`
- `shared/hygiene/queue.py` — читает/пишет `queue.yaml`
- `shared/hygiene/reports.py` — читает/пишет JSON-отчёты в `reports/`
- `shared/hygiene/schemas.py` — Pydantic-модели для всех данных выше
