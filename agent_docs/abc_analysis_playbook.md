# Playbook: ABC-анализ бренда Wookiee

## Назначение

Этот документ — инструкция для LLM-агента (Claude Code), который анализирует данные ABC-анализа. Скрипт `scripts/abc_analysis_unified.py` готовит данные (JSON + таблица), а LLM пишет аналитику, следуя этому playbook.

---

## 1. Товарная матрица бренда

### 1.1 Иерархия продуктов (5 уровней)

```
model_osnova (базовая модель) — 21 уникальная
  └── model_kod (вариант по юрлицу) — vuki, vuki2, vukiw, vukin, ...
      └── артикул (модель + цвет) — vuki2/black, компбел-ж-бесшов/чер
          └── товар (артикул + размер) — SKU уровень
```

## 2. Статусы товаров

### 2.1 Определения

| Статус | Значение | Как учитывать в анализе |
|--------|---------|------------------------|
| **Продается** | Активно продаётся | Основа анализа. Проверяем: ABC подтверждает? |
| **Запуск** | Только запущен | = Новый. НЕ ставить "Неликвид" даже при слабых метриках |
| **Новый** | Недавно добавлен | = Новый. Рано судить. Комментарий о наблюдении |
| **Выводим** | Решение о выводе принято | Проверяем: оправдано ли? Если ABC=A/B — сигнал "пересмотреть" |
| **Архив** | Снят с продажи | Не анализировать. Упомянуть если маржа <> 0 (возвраты/остатки) |
| **План / Подготовка** | Ещё не на полке | Исключить из анализа |

### 2.2 Правила работы со статусами

1. **НЕ рекомендовать вывод того, что уже в "Выводим"**. Вместо этого: "Статус подтверждён данными" или "Ускорить вывод — убыток X руб/мес".
2. **"Новый" и "Запуск"**: никогда не классифицировать как "Неликвид". Допустим мягкий комментарий "рано судить" или "первые сигналы тревожные, мониторинг через 1 мес".
3. **"Продается" + ABC=C**: главный сигнал к действию. Рекомендация: "перевести в вывод".
4. **"Выводим" + ABC=A/B**: ошибка статуса. Рекомендация: "вернуть в продажу".

## 4. Методология ABC-анализа

### 4.1 Уровень анализа

**ABC-классификация делается на уровне АРТИКУЛОВ внутри каждой model_osnova.**

Не сравниваем модели между собой (wendy vs ruby). Сравниваем артикулы внутри одной модели: wendy/black vs wendy/white vs wendy/coffee.

### 4.2 Алгоритм

Для каждой model_osnova:
1. Отсортировать артикулы по Марже (руб) убывания
2. Посчитать кумулятивную долю маржи
3. Присвоить ABC-класс:
   - **A**: до 80% кумулятивной маржи (лучшие)
   - **B**: 80-95% (хорошие)
   - **C**: >95% (хвост)

### 4.3 Три метрики (ABC — необходимое, но НЕ достаточное условие)

**КРИТИЧЕСКИ ВАЖНО**: ABC-класс основан ТОЛЬКО на абсолютной марже (руб). Это необходимое, но **НЕ достаточное** условие для рекомендации. Каждую рекомендацию необходимо квалифицировать по ТРЁМ метрикам:

| # | Метрика | Что показывает | Где смотреть | Порог |
|---|---------|---------------|-------------|-------|
| 1 | **Маржа (руб)** | Абсолютный вклад | `margin` | ABC-класс (A/B/C) |
| 2 | **Маржинальность (%)** | Эффективность денег | `margin_pct` | Целевая ≥15%. Ниже 10% — проблема. Отрицательная — убыток |
| 3 | **Оборот (дни)** | Скорость оборачиваемости | `turnover_days` | > 1.5× медиана модели = высокий |

**Правила квалификации рекомендаций:**

1. **«Вернуть в продажу»** (Выводим → ABC=A/B):
   - ✅ Вернуть ТОЛЬКО если: margin_pct ≥ 15% И оборот не аномально высокий
   - ⚠️ Осторожно если: margin_pct 10-15% — «вернуть, но мониторить маржинальность»
   - ❌ НЕ возвращать если: margin_pct < 10% — маржа в рублях положительная, но модель неэффективна
   - ❌ НЕ возвращать если: margin_pct отрицательная — это убыток

2. **«Перевести в вывод»** (Продается → ABC=C):
   - Подтвердить выводом: margin_pct низкая И/ИЛИ оборот высокий
   - Если margin_pct > 20% при ABC=C — «малый объём, но эффективный; мониторинг, не выводить»

3. **«Неликвид»** — три типа:
   - Убыточный: margin < 0 (любой ABC-класс) → вывод
   - Хвост: ABC=C + margin_pct < 15% + оборот высокий → вывод
   - Малый эффективный: ABC=C + margin_pct > 20% → мониторинг, не торопиться с выводом

### 4.4 Определение высокого оборота

Внутри модели:
- Найти медиану оборота (дни)
- "Высокий оборот" = > 1.5 × медиана модели

### 4.5 Цветовая склейка (color_code) — решения на уровне цвета

**КРИТИЧЕСКИ ВАЖНО**: Решения о выводе/возврате артикулов принимаются НЕ по отдельным артикулам, а по ЦВЕТОВЫМ ГРУППАМ внутри линейки.

**Что такое склейка**: В каждой линейке (tricot, seamless_wendy, seamless_audrey) один color_code может присутствовать в нескольких моделях. Например, color_code "5" (peach) может быть в wendy/peach, bella/peach, eva/peach.

**Правила:**

1. **Выводим цвет** — если color_code = «Убрать» в color_analysis (все модели в C), рекомендация распространяется на ВСЕ артикулы этого цвета в линейке
2. **Возвращаем артикул** — проверить, что color_code этого артикула НЕ в «Убрать». Если цвет «Убрать», то артикул НЕ возвращать даже если ABC=A/B
3. **Смешанный** — если color_code = «Смешанный» (часть моделей A/B, часть C), указать конкретно в каких моделях работает, а в каких нет
4. **Формат рекомендации**: «Вывести цвет [color_code] из линейки [tricot/seamless]: артикулы X, Y, Z (суммарная маржа N руб, маржинальность M%)»

### 4.6 Финальная классификация (с учётом всех метрик)

| Категория | ABC | Маржин-ть % | Оборот | Действие |
|-----------|-----|------------|--------|----------|
| **Лучшие** | A | ≥15% | норма | Основа модели, поддерживать |
| **Хорошие** | B | ≥15% | норма | Поддерживать, мониторить |
| **Эффективный малый** | C | ≥20% | норма | Мониторинг, не торопиться выводить |
| **Большой неэффективный** | A/B | <10% | — | Пересмотреть ценообразование |
| **Неликвид** | C | <15% | высокий | Кандидат на вывод |
| **Убыточный** | любой | <0% | — | Вывод немедленно |
| **Новый** | любой | любая | любой | Рано судить, мониторинг |

### 4.7 Динамика (ABC текущий vs предыдущий период)

| Сигнал | Описание |
|--------|---------|
| **Устойчивый** | ABC-класс совпадает в обоих периодах |
| **Появился** | Ухудшение/улучшение только в последнем периоде |
| **Не подтверждён** | Разнонаправленные метрики |

## 5. Структура аналитического отчёта

### 5.1 Обязательные разделы

1. **Здоровье бренда** — общая маржа, маржинальность, ДРР vs целевые (5-10 млн/мес, 20-25%)
2. **Анализ по model_osnova** — для каждой базовой модели:
   - Суммарная маржа ВСЕХ артикулов (включая vuki2, компбел и т.д.)
   - Маржинальность (%) модели — vs целевые 20-25%
   - Структура ABC (сколько A/B/C)
   - Ключевые сигналы (что растёт, что падает)
3. **Ошибки статусов** — "Продается" → Неликвид и "Выводим" → Лучшие/Хорошие
   - **С ОБЯЗАТЕЛЬНОЙ проверкой margin_pct и оборота** (см. §4.3)
   - Для каждого артикула: маржа (руб) + маржинальность (%) + оборот (дни) + color_code
4. **Цветовой анализ** — решения по color_code в рамках линеек (см. §4.5)
   - Какие цвета «Убрать» (все модели в C) — с суммарным эффектом
   - Какие цвета «Смешанный» — где работает, где нет
5. **Модели на выводе** — подтверждение/оценка текущих решений
6. **Новые модели** — оценка старта, мягкие рекомендации
7. **Рекомендации по действиям** — только конкретные, с руб. эффектом, с учётом ВСЕХ 3 метрик

### 5.2 Запреты

- НЕ рекомендовать вывод того, что уже "Выводим"
- НЕ ставить "стоп реклама" для новых моделей (alice, lana, valery)
- НЕ называть высокий оборот vuki2/moon2 "замороженными остатками"
- НЕ путать model_kod (компбел-ж-бесшов) с model_osnova (Vuki)
- НЕ сравнивать модели между собой (только артикулы внутри модели)
- НЕ придумывать данные, которых нет
- **НЕ рекомендовать «вернуть в продажу» без проверки margin_pct** (ABC=A/B при маржинальности 3% — это НЕ повод возвращать)
- **НЕ рекомендовать вывод/возврат отдельного артикула** без проверки color_code в рамках линейки

### 5.3 Формат рекомендаций

Каждая рекомендация ОБЯЗАНА содержать:
- Конкретный артикул или цветовая группа (color_code + линейка)
- Действие (вернуть в продажу / перевести в вывод / мониторинг / пересмотреть цену)
- **Три метрики**: маржа (руб), маржинальность (%), оборот (дни)
- Color_code и его статус в линейке (Оставить/Убрать/Смешанный)
- Обоснование из данных
- Оценка эффекта в рублях (где возможно)

**Пример хорошей рекомендации:**
> wendy/green (маржа 171K, margin 22.3%, оборот 2 дн, color_code=WE003 «Оставить» в seamless_wendy) — статус «Выводим», ABC=B. **Вернуть в продажу**: маржинальность в целевом коридоре, быстрый оборот, цвет работает во всей линейке.

**Пример плохой рекомендации:**
> ~~wendy/green (маржа 171K, ABC=B) — вернуть в продажу~~ ← нет margin %, нет оборота, нет color_code
