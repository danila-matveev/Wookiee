# Агенты Wookiee Analytics

## Что такое "агент"

В контексте Wookiee Analytics **агент** — это автономный модуль, который решает конкретную бизнес-задачу. Каждый агент:
- Имеет чёткую зону ответственности
- Может работать автономно или через единый интерфейс (Telegram-бот)
- Имеет собственный стек технологий и конфигурацию
- Документирован в этой папке

## Реестр агентов

| Агент | Статус | Назначение | Документация |
|-------|--------|-----------|--------------|
| [Telegram Bot](telegram-bot.md) | Активен | AI-ассистент финансового менеджера через Telegram | `bot/` |
| [Analytics Engine](analytics-engine.md) | Активен | Генерация аналитических отчётов с confidence scoring | `scripts/` |
| [MP Localization](mp-localization.md) | В разработке | Оптимизация логистики и индекса локализации WB/OZON | `MP scripts/` |
| [Bitrix CRM](bitrix-crm.md) | Планируется | Интеграция с CRM Bitrix24 | `Bitrix/` |
| [Людмила](lyudmila-bot.md) | Активен | IEE-агент, офис-менеджер и бизнес-ассистент | `lyudmila_bot/` |

### Статусы

- **Активен** — работает в production, используется командой
- **В разработке** — ручные скрипты работают, автоматизация в процессе
- **Планируется** — инфраструктура подготовлена, бизнес-логика не реализована

## agents/ vs agent_docs/

| | `agents/` (эта папка) | `agent_docs/` |
|---|---|---|
| **Аудитория** | Пользователи, разработчики, стейкхолдеры | AI-агенты разработки (Claude, Cursor) |
| **Содержание** | Бизнес-описания + техническая документация модулей | Руководства по разработке, ADR, DoD, шаблоны |
| **Вопрос** | "Что делает этот модуль и как его использовать?" | "Как правильно писать код в этом проекте?" |

## Шаблон описания агента

При добавлении нового агента используйте следующую структуру:

```markdown
# [Название агента]

## Бизнес-описание
- Назначение
- Статус (Активен / В разработке / Планируется)
- Какие задачи решает
- Кто использует

## Технические детали
- Стек технологий
- Архитектура (диаграмма потока)
- Ключевые файлы
- Конфигурация (.env переменные)
- Зависимости от других компонентов

## Запуск и использование
- Как запустить
- Примеры команд
- Ожидаемый результат

## Ссылки
- Исходный код
- Связанная документация
```
