# MP Localization Agent

## Бизнес-описание

**Назначение:** Оптимизация логистики и индекса локализации товаров на маркетплейсах Wildberries и OZON. Помогает перемещать товары между складами для снижения времени доставки и повышения видимости товаров.

**Статус:**
- **WB:** Автоматизирован через `vasily_agent/` (расписание: пн 08:00 МСК → Google Sheets + Bitrix)
- **OZON:** В разработке (ручные скрипты)

**Полная документация WB:** [`vasily_agent/playbook.md`](../vasily_agent/playbook.md)

**Какие задачи решает:**

### Wildberries
- Анализ текущего индекса локализации по складам
- Выявление складов-доноров (излишек) и складов-реципиентов (дефицит)
- Генерация рекомендаций по перемещениям для достижения индекса 75%+
- Расчёт необходимых дополнительных поставок

### OZON
- Анализ среднего времени доставки по регионам
- Оптимизация распределения товаров по складам для снижения времени доставки
- Расчёт стоимости перемещений между складами

**Кто использует:** Логист, менеджер маркетплейсов

## Технические детали

### Wildberries: Индекс локализации (автоматизирован)

**Агент:** `vasily_agent/` — полная документация в [`playbook.md`](../vasily_agent/playbook.md)

**Что делает:**
1. Каждый понедельник в 08:00 МСК автоматически запускает расчёт
2. Берёт данные через WB API (остатки + заказы за 2 дня)
3. Анализирует индекс локализации по 6 регионам для ИП и ООО
4. Генерирует перемещения (откуда/куда/сколько) и допоставки
5. Выгружает в Google Sheets (11 листов с автоформатированием)
6. Отправляет сводку в Bitrix чат "Wookiee Склад МСК"
7. Хранит историю в SQLite для сравнения (тренд)

**Ключевые файлы:**

| Файл | Назначение |
|------|-----------|
| `vasily_agent/service.py` | Ядро расчёта (VasilyService) |
| `vasily_agent/scheduler.py` | Планировщик (APScheduler CronTrigger) |
| `vasily_agent/sheets_export.py` | Экспорт в Google Sheets |
| `vasily_agent/bitrix_notify.py` | Уведомления в Bitrix24 |
| `MP scripts/Index_localization WB/generate_localization_report_v3.py` | Алгоритм анализа v3 |
| `MP scripts/Index_localization WB/run_localization.py` | API-обёртки WB |
| `MP scripts/Index_localization WB/wb_localization_mappings.py` | Маппинги складов → регионы |

### OZON: Время доставки

**Файл:** `MP scripts/Index_localization Ozon/generate_delivery_optimization.py`

**Алгоритм:**
1. Загрузка данных из Excel (среднее время доставки, управление остатками)
2. Анализ текущего распределения по складам
3. Расчёт оптимального распределения для минимизации времени доставки
4. Генерация рекомендаций по перемещениям с расчётом стоимости

**Стоимость перемещений OZON:**
| Объём | Цена |
|-------|------|
| 0.1 - 1 л | 38 руб |
| 1.001 - 175 л | 10 руб за каждый доп. литр свыше 1 л |
| Свыше 175 л | 910 руб |

**Надбавка за нелокальную доставку (с 6 апреля 2026):**
| Регион | Надбавка |
|--------|---------|
| Москва/МО, дальние регионы | 8% от цены товара |
| СПб и СЗО | 8% |
| Краснодар, Красноярск, Омск | 6% |
| Воронеж, Екатеринбург, Тюмень, Уфа | 4% |
| Некоторые регионы | 0% |

**Входные данные:**
- `average_delivery_time_*.xlsx` — из ЛК OZON
- `Управление остатками *.xlsx` — из ЛК OZON

**Выходные данные:**
- Excel-отчёт с рекомендациями по перемещениям

## Запуск и использование

### WB Локализация (автоматический)

```bash
# Запуск агента (первый расчёт сразу, далее — по расписанию пн 08:00)
python -m vasily_agent
```

### OZON Время доставки (ручной)

```bash
# Подготовить данные:
# 1. Скачать "Среднее время доставки" из ЛК OZON
# 2. Скачать "Управление остатками" из ЛК OZON
# 3. Положить в папку DD.MM/

cd "MP scripts/Index_localization Ozon"
python generate_delivery_optimization.py
```

## Статус автоматизации

### Реализовано (WB)
- Автоматическое скачивание данных через WB API
- Регулярная генерация отчётов по расписанию (пн 08:00 МСК)
- Экспорт в Google Sheets с автоформатированием
- Уведомления в Bitrix24 чат с дельтой индекса
- SQLite-хранилище для трендов и сравнения

### Планируется
- Автоматизация OZON (аналогично WB)
- Уведомления при критическом снижении индекса (< 60%)
