# SEO-плейбук Wookiee (Wildberries)

> Живая инструкция для анализа SEO-метрик на Wildberries.
> Бренд: Wookiee (женское нижнее бельё).
> Обновляется по мере калибровки бенчмарков.

---

## 1. Словарь метрик (ТОЛЬКО русские названия в отчёте)

| Системное имя | Русское название (для отчёта) | Описание |
|---------------|------------------------------|----------|
| card_opens / openCardCount | **Переходы** | Открытия карточки из поиска/каталога |
| add_to_cart / addToCartCount | **Корзина** | Добавления в корзину |
| orders / ordersCount | **Заказы** | Оформленные заказы |
| buyouts / buyoutsCount | **Выкупы** | Фактически выкупленные заказы |
| median_position | **Позиция** | Медианная позиция в поиске (1 = топ) |
| visibility | **Видимость** | % видимости (0–100%) |
| frequency | **Частотность** | Частота поискового запроса |
| organic_share | **Доля органики** | Органические заказы / все заказы |
| CR open→cart | **Конверсия переход→корзина** | корзина / переходы |
| CR cart→order | **Конверсия корзина→заказ** | заказы / корзина |
| CR open→order | **Конверсия переход→заказ** | заказы / переходы (СКВОЗНАЯ) |

**Сквозная конверсия переход→заказ** — ГЛАВНАЯ метрика. Она показывает итоговую эффективность карточки. Даже если переход→корзина упала, но корзина→заказ компенсирует — итог может быть позитивным.

## 2. Структура отчёта

### Обзор модели (TOTAL)
Всегда начинай с модели целиком. Это рамка для анализа артикулов.

### Артикулы vs Модель
Каждый артикул анализируй в контексте модели:
- "У wendy/brown переходы -18%, при общем по модели -4%"
- "Конверсия переход→заказ у wendy/black 2.9%, среднее по модели 2.0%"

### Сквозная конверсия
ОБЯЗАТЕЛЬНО считай и анализируй:
```
Конверсия переход→заказ = заказы / переходы
```
Это ВАЖНЕЕ, чем отдельные шаги воронки. Если переход→корзина -1.5pp, но корзина→заказ +2.5pp, то:
- Итоговая конверсия ВЫРОСЛА → это хорошо
- Объясни: "Меньше людей добавляют в корзину, но те кто добавляют — чаще заказывают"

## 3. Бенчмарки

### Позиции
| Позиция | Зона | Ожидания |
|---------|------|----------|
| 1–5 | ТОП | Максимум переходов |
| 6–15 | Конкурентная | Стабильный трафик |
| 16–30 | Средняя | Трафик снижен, нужно внимание |
| 30+ | Проблемная | Товар почти не находят |

### Конверсии (категория: нижнее бельё)
| Метрика | Проблема | Норма | Отлично |
|---------|----------|-------|---------|
| Переход→корзина | < 5% | 5–15% | > 15% |
| Корзина→заказ | < 20% | 20–40% | > 40% |
| Переход→заказ (сквозная) | < 1% | 1–3% | > 3% |

## 4. Связь SEO и финансов

### Улучшение позиции (число ↓)
```
Позиция ↓ → переходы ↑ → органические заказы ↑ → ДРР ↓ → маржа ↑
```

### Ухудшение позиции (число ↑)
```
Позиция ↑ → переходы ↓ → нужно больше рекламы → ДРР ↑ → маржа ↓
```

### Нетипичные ситуации — ОБЯЗАТЕЛЬНО объяснять
| Ситуация | Возможные причины |
|----------|-------------------|
| Позиция упала, заказы стабильны | Внешний трафик (блогеры), рекламная компенсация, рост конверсии |
| Позиция улучшилась, заказы не растут | Проблема конверсии (цена, карточка, out of stock) |
| Переходы падают, конверсия растёт | Трафик стал целевее (потеряли нерелевантные показы) |
| Переходы растут, конверсия падает | Привлекается нецелевой трафик |

## 5. Гипотезы — как формулировать

### ПРАВИЛО: каждое значимое изменение → гипотеза + данные

Пример ХОРОШЕЙ гипотезы:
> "Переходы wendy/brown упали на 18%. По данным позиций, запрос «бесшовный комплект» потерял 5 позиций (с 12 на 17), его частотность 8 500/день. Это может объяснять основную часть падения переходов."

Пример ПЛОХОЙ гипотезы:
> "Переходы упали. Рекомендуем обновить фото и описание."

### Логика построения гипотез:
1. Зафиксируй факт (метрика, изменение)
2. Найди коррелирующие данные (позиции, ключевики, конверсии, доля органики)
3. Сформулируй причинно-следственную связь
4. Если данных недостаточно — честно укажи "требуется ручной анализ"

## 6. Рекомендации — ТОЛЬКО подкреплённые данными

### ЗАПРЕЩЕНО
- Общие советы без привязки к данным
- Рекомендации, не проверяемые по имеющимся данным (мы не знаем текущий заголовок, фото, описание)
- Советы типа "запросить фото-отзывы" (нерелевантно для WB)
- Указание конкретных ключевых слов для добавления без проверки их релевантности товару

### РАЗРЕШЕНО
- Указать конкретные запросы, по которым упали позиции, и предложить проверить причины
- "Доля органики 44% при ДРР 12% → если восстановить позиции по ТОП-3 запросам, органика вырастет до ~55%, ДРР снизится на ~2pp"
- "Конверсия переход→корзина упала с 9.5% до 8.0% — проверить: цену, наличие размеров, свежие отзывы"
- Рассчитать потенциальный эффект от улучшения позиции по конкретному запросу

### Формат рекомендации:
```
ФАКТ: [что произошло, цифры]
ГИПОТЕЗА: [почему, подкреплённая данными]
ДЕЙСТВИЕ: [что конкретно проверить/сделать]
ЭФФЕКТ: [ожидаемый результат в числах, если возможно]
```

## 7. Формулы

```
# Сквозная конверсия (главная метрика)
конверсия_переход_заказ = заказы / переходы * 100

# Доля органики
доля_органики = органические_заказы / (органические_заказы + рекламные_заказы) * 100

# Изменение позиции (положительное = улучшение)
дельта_позиции = прошлая_позиция - текущая_позиция

# Изменение переходов
дельта_переходов_% = (текущие - прошлые) / прошлые * 100

# ДРР
ДРР = расход_на_рекламу / выручка * 100
```
