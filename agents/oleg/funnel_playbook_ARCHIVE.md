# Плейбук оцифровки маркетинговой воронки Wookiee (WB)

> Живая инструкция для анализа маркетинговой воронки товаров бренда Wookiee (женское нижнее бельё) на Wildberries.
> Обновляется по мере калибровки бенчмарков.

---

## 1. Словарь метрик (ТОЛЬКО русские названия в отчёте)

| Системное имя | Русское название (для отчёта) | Описание |
|---------------|------------------------------|----------|
| openCardCount | **Переходы** | Открытия карточки из поиска/каталога |
| addToCartCount | **Корзина** | Добавления в корзину |
| ordersCount | **Заказы** | Оформленные заказы |
| buyoutsCount | **Выкупы** | Фактически выкупленные заказы |
| CR open→cart | **Конверсия переход→корзина** | корзина / переходы |
| CR cart→order | **Конверсия корзина→заказ** | заказы / корзина |
| CRO (CR open→order) | **Конверсия переход→заказ** | заказы / переходы (СКВОЗНАЯ — ГЛАВНАЯ) |
| CRP (CR open→buyout) | **Конверсия переход→выкуп** | выкупы / переходы (ИТОГОВАЯ) |
| DRR | **ДРР** | расход на рекламу / выручка × 100% |
| ROMI | **ROMI** | маржа / расход на рекламу × 100% |
| CPS | **CPS** | расход на рекламу / кол-во продаж |
| CAC | **CAC** | расход на рекламу / переходы |
| organic_share | **Доля органики** | органические заказы / все заказы |
| margin | **Маржа** | выручка минус все расходы |
| revenue_spp | **Выручка** | выручка с учётом СПП |

**Конверсия переход→заказ (CRO)** — ГЛАВНАЯ метрика. Она показывает итоговую эффективность карточки.

---

## 2. Маркетинговая воронка

```
Показы → Переходы → Корзина → Заказы → Выкупы → Маржинальная прибыль
```

### Что влияет на каждый этап

| Этап | Что влияет |
|------|-----------|
| Показы → Переходы | Позиция в поиске, качество главного фото, цена, рейтинг, отзывы |
| Переходы → Корзина | Фото карточки, описание, размерный ряд, цена vs конкуренты |
| Корзина → Заказ | Цена, наличие размеров, сроки доставки, акции |
| Заказ → Выкуп | Качество товара, соответствие ожиданиям, упаковка |

---

## 3. Бенчмарки (категория: нижнее бельё)

### Конверсии
| Метрика | Проблема | Норма | Отлично |
|---------|----------|-------|---------|
| Переход→корзина | < 5% | 5–15% | > 15% |
| Корзина→заказ | < 20% | 20–40% | > 40% |
| CRO (переход→заказ) | < 1% | 1–3% | > 3% |
| CRP (переход→выкуп) | < 0.5% | 0.5–2% | > 2% |

### Экономика
| Метрика | Проблема | Норма | Отлично |
|---------|----------|-------|---------|
| ДРР (стабильная карточка) | > 15% | 10% | 5–7% |
| ДРР (новая карточка, 1-3 мес) | > 50% | 30–40% | 20% |
| ROMI | < 100% | 100–200% | > 200% |
| Доля органики | < 30% | 30–60% | > 70% |
| CAC vs Прибыль/продажу | CAC > profit_per_sale | CAC < profit_per_sale | CAC < 50% profit_per_sale |

## 4. Формулы

```
# Воронка
CRO (конверсия переход→заказ) = заказы / переходы × 100
CRP (конверсия переход→выкуп) = выкупы / переходы × 100

# Экономика
ДРР = расход_на_рекламу / выручка × 100
ROMI = маржа / расход_на_рекламу × 100
CPS = расход_на_рекламу / количество_продаж
CAC = расход_на_рекламу / переходы
Прибыль_на_продажу = маржа / количество_продаж
Прибыль_на_посетителя = маржа / переходы

# Маржа
Маржа = revenue_spp - comis_spp - logist - sebes - reclama - reclama_vn
        - storage - nds - penalty - retention - deduction

# Доля органики
Доля_органики = органические_заказы / (органические_заказы + рекламные_заказы) × 100

# Средний чек
Ср_чек = revenue_spp / count_orders
```

## 5. Структура отчёта

### 5.1 Обзор модели (TOTAL)
Всегда начинай с модели целиком. Это рамка для анализа артикулов.

### 5.2 Артикулы vs Модель
Каждый артикул анализируй В КОНТЕКСТЕ модели:
- "У wendy/brown CRO 2.1%, при среднем по модели 2.0%"
- "Переходы wendy/brown -18%, при общем по модели -4%"

### 5.3 Сквозная конверсия
ОБЯЗАТЕЛЬНО считай и анализируй CRO (переход→заказ).
Если переход→корзина упала, но корзина→заказ выросла — смотри на ИТОГОВУЮ CRO:
- Если итоговая CRO выросла — это ХОРОШО, несмотря на отдельные падения
- "Переходы ↓18%, но CRO ↑0.3pp → заказы упали всего на 5%"

## 6. Связь воронки и финансов

### Улучшение CRO
```
CRO ↑ → больше заказов при тех же переходах → ДРР ↓ → маржа ↑
```

### Рост органики
```
Органика ↑ → меньше зависимость от рекламы → ДРР ↓ → маржа ↑
```

### Нетипичные ситуации — ОБЯЗАТЕЛЬНО объяснять
| Ситуация | Что проверить |
|----------|---------------|
| Переходы падают, заказы стабильны | CRO вырос? Реклама компенсирует? Внешний трафик? |
| CRO упал | Цена изменилась? Размеры out of stock? Негативные отзывы? Новый конкурент? |
| Переходы растут, CRO падает | Привлекается нецелевой трафик? Позиции по нерелевантным запросам? |
| ДРР вырос | Реклама дорожает? Конверсия падает? Конкуренция за ключевые слова? |
| Маржа падает при стабильной выручке | Расходы на логистику/хранение выросли? Больше возвратов? |
| Доля органики падает | Позиции по ключевым ухудшились? Конкуренты усилились? |

## 7. Гипотезы — как формулировать

### ПРАВИЛО: каждое значимое изменение → гипотеза + данные

Пример ХОРОШЕЙ гипотезы:
> "CRO wendy/brown упала с 2.5% до 1.8%. По данным ключевых слов, запрос «бесшовный комплект» потерял 5 позиций (с 12 на 17), его частотность 8 500/день. Если трафик по этому запросу стал менее целевым — это может объяснять падение CRO."

Пример ПЛОХОЙ гипотезы:
> "CRO упала. Рекомендуем обновить фото и описание."

### Логика построения гипотез:
1. Зафиксируй факт (метрика, изменение, конкретные цифры)
2. Найди коррелирующие данные (другие метрики, ключевые слова, экономика)
3. Сформулируй причинно-следственную связь
4. Если данных недостаточно — честно укажи "требуется ручной анализ"

## 8. Рекомендации — ТОЛЬКО подкреплённые данными и расчётом

### ЗАПРЕЩЕНО
- Общие советы без привязки к данным
- "Добавить инфографику" (у карточек уже есть инфографика)
- "Добавить хлопок в заголовок" (не проверив состав товара)
- "Запросить фото-отзывы" (нереалистично для WB)
- Любые советы типа "обычно на WB так работает" без данных Wookiee
- Выдумывать данные, которых нет в отчёте

### РАЗРЕШЕНО
- Указать конкретные проблемы с расчётом эффекта в рублях
- Предложить проверить конкретные факторы (цена, размеры, отзывы)
- Рассчитать потенциальный эффект от улучшения конкретной метрики

### Формат рекомендации:
```
ФАКТ: CRO wendy/brown 2.1% при среднем по модели 3.0%
ГИПОТЕЗА: возможно, цена не конкурентоспособна или отсутствуют ходовые размеры
ДЕЙСТВИЕ: проверить ценообразование и наличие размеров S/M/L
ЭФФЕКТ: если довести CRO до 2.5% → +48 заказов/нед × 1270₽ = +60 960₽ выручки → +12 800₽ маржи
```

## 9. Анализ аномалий (что проверять)

| Симптом | Проверить |
|---------|-----------|
| CRO упал > 0.5pp | Цена, размерный ряд, новые негативные отзывы, конкуренты. **Вызови get_search_keywords и get_keyword_positions** — проверь, не потеряны ли позиции по ключевым генераторам |
| Переходы упали > 20% | Позиции по ключевым, рекламный бюджет, сезонность |
| ДРР вырос > 3pp | Стоимость клика, конверсия из рекламы, конкуренция за ключевые |
| Маржа упала при стабильной выручке | Логистика, хранение, штрафы, возвраты |
| Доля органики упала > 10pp | Позиции, новые конкуренты, изменение алгоритма WB |
| Выкупы упали при стабильных заказах | Качество товара, размерная сетка, описание vs реальность |
| CAC > profit_per_sale | Реклама уничтожает маржу на артикуле. Рекомендация: снизить ставки или отключить рекламу |
| ROMI < 100% | Убыточная реклама. Показать расход, маржу, ROMI. Решение: стоп (<50%) / сокращение (50-100%) |
