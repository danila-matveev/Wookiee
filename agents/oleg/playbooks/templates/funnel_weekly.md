## ВАЖНО: Как работает Funnel агент

Этот шаблон описывает структуру отчёта, генерируемого `build_funnel_report` в Python. Агент Макар передаёт результат без изменений.

**Протокол работы:**
1. Вызвать `build_funnel_report(start_date, end_date)` — ПЕРВЫМ И ЕДИНСТВЕННЫМ основным инструментом
2. `build_funnel_report` возвращает готовый отчёт: `telegram_summary + brief_summary + detailed_report`
3. Агент передаёт результат без изменений
4. При необходимости drill-down — использовать `get_funnel_overview`, `get_article_funnel`, `get_article_economics`

**Инструменты в порядке вызова:**
1. `build_funnel_report` — основной (генерирует весь отчёт)
2. `get_funnel_overview` — если нужен обзор конкретной модели
3. `get_article_funnel` — если нужна детализация по артикулам
4. `get_article_economics` — если нужна юнит-экономика
5. `get_search_keywords` — если нужен анализ ключевых слов
6. `get_article_ad_attribution` — органика vs реклама per article
7. `get_keyword_positions` — позиции по ключевым словам WoW

## Ожидаемая структура detailed_report (генерируется build_funnel_report)

### Словарь метрик (только русские названия в отчёте)

| Системное имя | Русское название (для отчёта) | Описание |
|---------------|------------------------------|----------|
| openCardCount | **Переходы** | Открытия карточки из поиска/каталога |
| addToCartCount | **Корзина** | Добавления в корзину |
| ordersCount | **Заказы** | Оформленные заказы |
| buyoutsCount | **Выкупы** | Фактически выкупленные заказы |
| CR open→cart | **Конверсия переход→корзина** | корзина / переходы |
| CR cart→order | **Конверсия корзина→заказ** | заказы / корзина |
| CRO (CR open→order) | **Конверсия переход→заказ** | заказы / переходы (СКВОЗНАЯ — ГЛАВНАЯ) |
| CRP (CR open→buyout) | **Конверсия переход→выкуп** | выкупы / переходы (ИТОГОВАЯ) |
| DRR | **ДРР** | расход на рекламу / выручка × 100% |
| ROMI | **ROMI** | маржа / расход на рекламу × 100% |
| organic_share | **Доля органики** | органические заказы / все заказы |

**Конверсия переход→заказ (CRO)** — ГЛАВНАЯ метрика.

## Бенчмарки (категория: нижнее бельё WB)

| Метрика | Проблема | Норма | Отлично |
|---------|----------|-------|---------|
| Переход→корзина | < 5% | 5–15% | > 15% |
| Корзина→заказ | < 20% | 20–40% | > 40% |
| CRO (переход→заказ) | < 1% | 1–3% | > 3% |
| CRP (переход→выкуп) | < 0.5% | 0.5–2% | > 2% |
| ДРР (стабильная карточка) | > 15% | 10% | 5–7% |
| ROMI | < 100% | 100–200% | > 200% |
| Доля органики | < 30% | 30–60% | > 70% |

## Маркетинговая воронка

```
Показы → Переходы → Корзина → Заказы → Выкупы → Маржинальная прибыль
```

| Этап | Что влияет |
|------|-----------|
| Показы → Переходы | Позиция в поиске, главное фото, цена, рейтинг, отзывы |
| Переходы → Корзина | Фото карточки, описание, размерный ряд, цена vs конкуренты |
| Корзина → Заказ | Цена, наличие размеров, сроки доставки, акции |
| Заказ → Выкуп | Качество товара, соответствие ожиданиям, упаковка |

## Анализ аномалий (для drill-down через get_funnel_overview)

| Симптом | Проверить |
|---------|-----------|
| CRO упал > 0.5pp | Цена, размерный ряд, отзывы, конкуренты. get_search_keywords |
| Переходы упали > 20% | Позиции по ключевым, рекламный бюджет, сезонность |
| ДРР вырос > 3pp | Стоимость клика, конверсия, конкуренция |
| Выкупы упали при стабильных заказах | Качество товара, размерная сетка |
| CAC > profit_per_sale | Реклама уничтожает маржу |
| ROMI < 100% | Убыточная реклама |
