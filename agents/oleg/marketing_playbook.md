# Маркетинговый плейбук Wookiee

> Живая инструкция для Marketer-агента.
> Обновляется по мере калибровки целевых значений.

---

## 1. Целевые уровни ДРР

### Внутренняя реклама (МП)

| Канал | Цель ДРР | Максимум ДРР | Комментарий |
|-------|----------|--------------|-------------|
| WB    | 5-8%     | 12%          | Основной канал, высокая органика |
| OZON  | 8-12%    | 18%          | Слабая органика, зависимость от рекламы выше |

### Внешняя реклама

| Канал     | Цель ДРР | Максимум ДРР | Окно атрибуции |
|-----------|----------|--------------|----------------|
| Блогеры   | 15-25%   | 35%          | 7-14 дней после публикации |
| VK        | 10-18%   | 25%          | 3-7 дней |
| Креаторы  | 12-20%   | 30%          | 7-14 дней |

### Суммарный ДРР (внутр + внешн)

| Уровень        | ДРР    |
|----------------|--------|
| Отлично        | < 10%  |
| Норма          | 10-15% |
| Требует внимания | 15-20% |
| Критично       | > 20%  |

> Значения ориентировочные. Калибровать по факту первых 2 недель работы.

## 2. Нормы конверсии воронки

### WB (ориентиры)

| Шаг воронки         | CR норма  | Красная зона |
|----------------------|-----------|--------------|
| Показы → Клики (CTR) | 3-7%     | < 2%         |
| Клики → Корзина      | 15-25%   | < 10%        |
| Корзина → Заказы     | 25-40%   | < 20%        |
| Заказы → Выкупы      | 55-75%   | < 45%        |

### OZON (ориентиры)

| Шаг воронки         | CR норма  | Красная зона |
|----------------------|-----------|--------------|
| Показы → Клики (CTR) | 2-5%     | < 1.5%       |
| Клики → Корзина      | 10-20%   | < 7%         |
| Корзина → Заказы     | 20-35%   | < 15%        |
| Заказы → Выкупы      | 40-65%   | < 30%        |

> Выкуп % лаговый (3-21 день). Не использовать для оценки текущих кампаний.

## 3. Атрибуция внешних каналов

### Блогеры
- Окно атрибуции: **7-14 дней** после публикации
- Метод: рост заказов по конкретной модели vs базовая линия (среднее за 7 дней до)
- Учитывать: день недели, сезонность, одновременные акции МП
- Минимальный бюджет для оценки: 30 000 ₽ на размещение

### VK (таргет)
- Окно атрибуции: **3-7 дней** после показа
- Метод: UTM-метки (если есть) или корреляция расход ↔ рост заказов
- Специфика: хуже атрибутируется, эффект размазан

### Креаторы (UGC)
- Окно атрибуции: **7-14 дней**
- Метод: аналогично блогерам, но контент работает дольше (evergreen)
- Считать накопительный эффект за 30 дней

### Общие правила
- Рост органических заказов в период внешней рекламы частично атрибутируется к внешнему каналу
- При одновременных кампаниях — делить эффект пропорционально бюджету (упрощение)

## 4. Стратегия по моделям

### Категоризация моделей

| Категория   | Критерий                          | Стратегия рекламы |
|-------------|-----------------------------------|-------------------|
| **Growth**  | Маржинальность > 25%, заказы растут | Увеличивать бюджет (ДРР до максимума) |
| **Harvest** | Маржинальность > 20%, заказы стабильны | Поддерживать текущий бюджет |
| **Optimize**| Маржинальность 10-20%              | Снижать ДРР, искать эффективные ключи |
| **Cut**     | Маржинальность < 10% или ДРР > 25% без роста | Отключить рекламу, только органика |

### Правила переключения
- Модель в Growth минимум **7 дней** перед переоценкой
- Перевод в Cut только после **14 дней** неэффективности
- При переводе из Cut обратно — начинать с минимального бюджета

## 5. Словарь метрик и формулы

| Метрика | Формула | Пояснение |
|---------|---------|-----------|
| **ДРР (внутр)** | `расход_МП / выручка × 100%` | Доля рекламных расходов внутри маркетплейса |
| **ДРР (внешн)** | `расход_внешн / выручка × 100%` | Блогеры + VK + креаторы |
| **ДРР (общий)** | `(расход_МП + расход_внешн) / выручка × 100%` | Суммарная рекламная нагрузка |
| **CPO** | `расход_рекламы / кол-во_заказов` | Стоимость одного заказа |
| **ROMI** | `(маржа_от_рекламы - расход) / расход × 100%` | Возврат на маркетинговые инвестиции |
| **CTR** | `клики / показы × 100%` | Кликабельность |
| **CR корзина** | `добавления_в_корзину / клики × 100%` | Конверсия в корзину |
| **CR заказ** | `заказы / корзина × 100%` | Конверсия в заказ |
| **Выкуп %** | `выкупы / заказы × 100%` | Лаг 3-21 день |
| **Органика %** | `заказы_без_рекламы / заказы_всего × 100%` | Доля органического трафика |

### Правила расчёта
- При объединении каналов: **только средневзвешенные** для процентных метрик
- GROUP BY по модели: **всегда LOWER()** (разный регистр в данных)
- ДРР считать по выручке, не по заказам (заказы ещё не выкуплены)

## 6. Ограничения данных

| Ограничение | Влияние | Как работать |
|-------------|---------|--------------|
| OZON: нет органической воронки | `search_stat = 0` | Воронку анализировать только по WB. Для OZON — расчётная органика: `organic = total − ad` (инструмент `get_ozon_organic_estimate`) |
| OZON: уникальные посетители | Метрика пока недоступна | Мониторить OZON Seller Analytics API (`/v1/analytics/data`, метрика `hits_view_pdp`) |
| Выкуп % лаговый (3-21 дн) | Нельзя оценивать текущую рекламу | Использовать скользящее среднее за 21+ дней |
| Внешняя реклама без UTM | Неточная атрибуция | Корреляционный метод (расход ↔ рост заказов) |
| Данные по блогерам вручную | Задержка 1-3 дня | Учитывать при анализе «свежих» дней |
| Разный регистр моделей в БД | Дубли при группировке | `LOWER()` в каждом GROUP BY |
| СПП влияет на конверсию, но не на нашу выручку/ед | Путаница в интерпретации | Отделять «цена для клиента» от «наша выручка» |
| Органика vs Платное: разные метрики | Нельзя складывать card_opens и impressions | card_opens ↔ clicks (переходы), add_to_cart ↔ paid_to_cart, orders ↔ paid_orders |
| JOIN nomenclature: fan-out | Один nmid = несколько строк | `SELECT DISTINCT nmid, vendorcode FROM nomenclature` |

## 7. Правила отображения в отчётах

### Конверсии (CR)
- **Всегда** показывать CR текущего и прошлого периода рядом (было → стало)
- Изменение CR — в процентных пунктах (pp), не в процентах

### Органика vs Платное
- **Переходы в карточку** (card_opens + clicks) — сопоставимые метрики, показывать доли
- **Рекламные показы** (impressions) — отдельная метрика, НЕ складывать с органикой
- При разбивке органики — отдельно по статусу товара (основные vs выводимые)

### Расчётная органика OZON
- `organic_orders = total_orders − ad_orders` (cap на 0 при отрицательных)
- Только уровень заказов — показы/переходы недоступны
- Если ad_orders > total_orders — добавить warning

### Аномалии по моделям
- Порог отклонения по умолчанию: 30%
- Для каждой аномалии — гипотеза причины + рекомендация
- Паттерны: CTR↓+заказы↓, CTR↑+заказы stable, органика↓+платное↑, CR корзина↓
