# Людмила — Agent Specification

## Статус
- **Версия**: 1.0
- **Готовность**: Production (60%)
- **Последнее обновление**: 2026-02-14

## Миссия
Офис-менеджер бренда Wookiee. Принимает задачи через Telegram, структурирует их, создаёт в Bitrix24.

## Архитектура

### Тип агента
LLM-assistant с intent detection (1-2 вызова LLM, не полноценный agent loop)

### Модели
- **Классификация/структурирование**: z.ai GLM-4.5-flash
- **Сложный анализ**: Claude Sonnet 4.5

### Компоненты
- **Интерфейс**: Telegram (aiogram 3.15) с FSM
- **Хранилище**:
  - Supabase (context, user profiles)
  - MemoryStorage (FSM state — **NOT persistent!**)
- **Цикл**: Intent detection → Structure → Create in Bitrix (linear, not loop)

## Tools

| Tool | Назначение | Статус |
|------|-----------|--------|
| `classify_intent` | Определить тип задачи (task/meeting/question) | ✅ Working |
| `structure_task` | Структурировать задачу из NL | ✅ Working |
| `create_bitrix_task` | Создать задачу в Bitrix24 | ✅ Working |
| `suggest_improvements` | Предложить улучшения к задаче | ✅ Working |
| `fetch_user_context` | Загрузить контекст из Supabase | ✅ Working |

## Feedback Loop

**Status**: ❌ NOT WORKING

### Проблемы
- Suggestion rejection не анализируется
- Нет обучения на отклонённых предложениях
- Отсутствует механизм адаптации промптов на основе обратной связи

## Известные проблемы

| # | Проблема | Влияние | Приоритет | Статус |
|---|---------|---------|-----------|--------|
| 1 | Промпт в `persona.py`, не в `playbook.md` | Не обновляется при feedback | High | Open |
| 2 | FSM в MemoryStorage | Перезагрузка бота = потеря всех диалогов | Critical | Open |
| 3 | Нет fallback при недоступности Supabase | Контекст пуст, работает "с нуля" | Medium | Open |
| 4 | Rejected suggestions не анализируются | Не учится что НЕ предлагать | Medium | Open |
