# analytics-agent-roma

**Рома** — ИИ финансовый и бизнес-аналитик компании Wookiee.

## Что это

Рома — аналитический агент, который генерирует бизнес-анализ дневных отчётов.
Вместо шаблонных гипотез ("confidence 0.72", "триангуляция") Рома строит
причинно-следственные цепочки на человеческом бизнес-языке.

## Как вызвать Рому

### Способ 1: Команда `/daily-report` (рекомендуемый)

В Claude Code набрать:
```
/daily-report 2026-02-09
```

Рома автоматически:
1. Запустит скрипт и соберёт данные
2. Прочитает свой playbook (правила анализа)
3. Напишет **краткую сводку** (для чата) + **подробный отчёт** (для Notion)
4. Сохранит подробный отчёт в файл и синхронизирует с Notion
5. Покажет краткую сводку — готовую для копирования в Telegram

### Способ 2: Свободный запрос

Просто написать в Claude Code:
```
Создай отчёт за 9 февраля
```
или
```
Что с маржой за вчера?
```

Рома поймёт и выполнит тот же workflow.

### Способ 3: Ручной запуск (по шагам)

```bash
# 1. Генерация данных
python3 scripts/daily_analytics.py --date 2026-02-09 --save --export-context

# 2. Рома (Claude Code) анализирует
# -> прочитать playbook.md + data_context.json + написать анализ

# 3. Notion sync
python3 scripts/notion_sync.py --file reports/2026-02-09_daily_analytics.md
```

## Что Рома выдаёт

### Краткая сводка (для чата)

30-40 строк, формат `[b]...[/b]` для Telegram. Содержит:
- Бренд / WB / OZON: маржа, продажи, заказы, реклама, ДРР
- Декомпозиция: «X% падения дал фактор Y. Если убрать Z → маржа была бы N руб (M%)»
- Причинно-следственная связь: 3-5 предложений
- 2-3 рекомендации с оценкой эффекта в рублях

Готова для копирования и отправки в рабочий чат.

### Подробный отчёт (Notion)

Полный Markdown-отчёт с:
- Таблицами метрик (скрипт)
- Причинно-следственными цепочками (Рома)
- Модельной декомпозицией (таблица по моделям с ДРР)
- 7-дневным контекстом (тренды, закономерности)
- Пронумерованными рекомендациями с рублёвой оценкой

Сохраняется в `reports/` и синхронизируется с Notion.

## Как взаимодействовать с Ромой

### Обратная связь (самое важное!)

После отчёта — скажите Роме что не так. Примеры:

```
"Декомпозиция по рекламе неточная — внешняя реклама была на Audrey, укажи это"
"Добавь правило: joy на WB — модель C-группы, не тратить на неё рекламу"
"Сводка слишком длинная, сократи до 25 строк"
"ДРР считай от продаж, не от выручки до СПП"
```

Рома:
1. Обновит `playbook.md` — добавит новое правило
2. Запишет в `feedback_log.md` — что сказали и что изменилось
3. Следующий отчёт автоматически учтёт правку

### Уточняющие вопросы

```
"Почему wendy просела на OZON?"
"Покажи динамику joy за неделю"
"Сравни эффективность рекламы за последние 5 дней"
```

### Изменение формата

```
"Сводку делай короче — максимум 20 строк"
"В подробном отчёте добавь секцию по трафику"
"ДРР показывай и от продаж, и от выручки до СПП"
```

## Как работает

### Фаза 1 (текущая): Claude Code = Рома

```
daily_analytics.py --export-context
         |
         v
   таблицы (Markdown) + data_context.json
         |
         v
   Рома читает playbook.md + data_context.json
         |
         v
   Рома пишет: краткая сводка + подробный отчёт
         |
         v
   Подробный отчёт → файл + Notion sync
   Краткая сводка → показать пользователю
```

### Фаза 2 (будущее): Telegram-бот = Рома

- Тот же `data_context.json` передаётся в Claude API
- Бот получает анализ автономно, без IDE
- Файл `llm_client.py` (будет добавлен) вызывает Claude API

## Как дообучается

Рома работает по циклу обратной связи:

```
Отчёт → Пользователь читает → Feedback → playbook.md обновлён → Следующий отчёт лучше
```

1. **playbook.md** — живой справочник правил. Читается ПЕРЕД каждым анализом.
2. После отчёта пользователь даёт обратную связь (правки, замечания).
3. Рома добавляет новое правило в `playbook.md` и запись в `feedback_log.md`.
4. Следующий отчёт автоматически учитывает обновлённые правила.

### Примеры дообучения

- "Выкуп% — лаговый показатель, нельзя использовать для дневного анализа" -> правило добавлено
- "ДРР нужно разбивать на внутреннюю и внешнюю рекламу" -> правило добавлено
- "Делай краткую сводку для чата + подробный отчёт для Notion" -> формат обновлён
